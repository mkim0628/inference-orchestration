# Evaluation Criteria

evaluator 에이전트는 이 파일의 모든 항목을 기준으로 구현을 평가한다.
항목 추가·수정은 사람이 직접 한다. 에이전트는 읽기만 한다.

---

## 1. 정확성

| 항목 | 기준 | 가중치 |
|------|------|--------|
| 단위 테스트 통과율 | 100% | 필수 |
| 통합 테스트 통과율 | 100% | 필수 |
| CacheStore 인터페이스 준수 | 모든 추상 메서드 구현 | 필수 |

## 2. 성능 — Cache Hit Rate

| 항목 | 기준 | 가중치 |
|------|------|--------|
| 베이스라인 대비 Cache Hit Rate | +5%p 이상 | 높음 |
| 비연속 세그먼트 히트율 | 전체 히트의 20% 이상이 비연속 구간에서 발생 | 높음 |

## 3. 성능 — 지연

| 항목 | 기준 | 가중치 |
|------|------|--------|
| TTFT p50 회귀 없음 | 베이스라인 대비 +10% 이내 | 필수 |
| TTFT p99 | 베이스라인 대비 +20% 이내 | 높음 |
| TBT 평균 | 베이스라인 대비 +5% 이내 | 중간 |

## 4. 메모리

| 항목 | 기준 | 가중치 |
|------|------|--------|
| KV Memory Footprint | 베이스라인 대비 +30% 이내 | 높음 |
| 캐시 퇴거 정책 | LRU 또는 동등 이상의 효율 | 중간 |

## 5. 코드 품질

| 항목 | 기준 | 가중치 |
|------|------|--------|
| 타입 힌트 | 모든 공개 함수·메서드에 존재 | 중간 |
| 불필요한 추상화 없음 | Spec.md에 없는 추상화 도입 금지 | 낮음 |
| 주석 과잉 없음 | WHY가 자명한 코드에 주석 없음 | 낮음 |

## 6. 재현성

| 항목 | 기준 | 가중치 |
|------|------|--------|
| 시드 고정 재현 | 동일 시드·설정으로 결과 동일 | 필수 |
| 설정 YAML 존재 | `configs/experiments/YYYY-MM-DD.yaml` 생성됨 | 필수 |

---

## 판정 규칙

- **필수** 항목 중 하나라도 Fail → 전체 Fail (루프 継続)
- 필수 항목 전부 Pass + **높음** 항목 70% 이상 Pass → 전체 Pass
- 그 외 → Partial (루프 継続, 단 3회차에서는 리포트 저장 후 종료)
