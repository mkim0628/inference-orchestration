# Evaluation Criteria

evaluator 에이전트는 이 파일의 모든 항목을 기준으로 구현을 평가한다.
항목 추가·수정은 사람이 직접 한다. 에이전트는 읽기만 한다.

구현된 Activity(A/B/C 또는 조합)에 따라 해당 섹션의 항목을 적용한다.
**섹션 0(공통)**은 항상 적용된다.

---

## 0. 공통 (항상 적용)

| 항목 | 기준 | 가중치 |
|------|------|--------|
| 단위 테스트 통과율 | 100% | 필수 |
| 통합 테스트 통과율 | 100% | 필수 |
| CacheStore 인터페이스 준수 | 모든 추상 메서드 구현 | 필수 |
| 시드 고정 재현성 | 동일 시드·설정으로 결과 동일 | 필수 |
| 설정 YAML 존재 | `configs/experiments/YYYY-MM-DD.yaml` 생성됨 | 필수 |
| 타입 힌트 | 모든 공개 함수·메서드에 존재 | 중간 |
| 불필요한 추상화 없음 | Spec.md에 없는 추상화 도입 금지 | 낮음 |

## 1. 추론 처리량 (Inference Throughput) — 핵심 목표

| 항목 | 기준 | 가중치 |
|------|------|--------|
| Throughput 향상 | 베이스라인 대비 tokens/sec +10% 이상 | 높음 |
| Throughput 목표치 | 베이스라인 대비 +20% (장기 목표) | 중간 |
| TTFT p50 | 베이스라인 대비 +5% 이내 | 필수 |
| TTFT p99 | 베이스라인 대비 +15% 이내 | 높음 |
| TBT 평균 | 베이스라인 대비 +5% 이내 | 중간 |

---

## 2. Activity A — KV Cache-aware Scheduling (해당 사이클에 적용)

| 항목 | 기준 | 가중치 |
|------|------|--------|
| 스케줄링 오버헤드 | TTFT p50 증가 +5% 이내 | 필수 |
| 캐시 히트율 향상 | 스케줄링 미적용 대비 히트율 +10%p 이상 | 높음 |
| 요청 처리 공정성 | 최대 대기 시간이 2× 초과하지 않음 | 높음 |
| 배치 활용률 | GPU 활용률 회귀 없음 | 중간 |

---

## 3. Activity B — Non-Contiguous KV Cache Reuse (해당 사이클에 적용)

| 항목 | 기준 | 가중치 |
|------|------|--------|
| 전체 Cache Hit Rate | 베이스라인 대비 +5%p 이상 | 높음 |
| 비연속 세그먼트 히트율 | 전체 히트의 30% 이상이 비연속 구간에서 발생 | 높음 |
| KV Memory Footprint | 베이스라인 대비 +20% 이내 | 높음 |
| 캐시 퇴거 정책 효율 | LRU 동등 이상 | 중간 |

---

## 4. Activity C — KV Cache Compression (해당 사이클에 적용)

| 항목 | 기준 | 가중치 |
|------|------|--------|
| **Accuracy 보존 (필수)** | perplexity 변화 ±1% 이내 | 필수 |
| **Accuracy 보존 — 태스크** | downstream 태스크 정확도 변화 ±1% 이내 | 필수 |
| KV Memory Reduction | 베이스라인 대비 −30% 이상 | 높음 |
| Effective Context Length | 동일 메모리 예산에서 2× 이상 | 높음 |
| 압축 오버헤드 | Encode/Decode 추가 지연 TTFT +10% 이내 | 높음 |
| Cache Eviction Rate | 베이스라인 대비 감소 | 중간 |

---

## 5. 크로스 Activity 조합 (해당 사이클에 적용)

| 항목 | 기준 | 가중치 |
|------|------|--------|
| 복합 Throughput 향상 | 단일 Activity 대비 추가 +5% 이상 | 높음 |
| 복합 Memory Reduction | 단일 Activity 대비 추가 −10% 이상 | 높음 |
| Accuracy 보존 (C 포함 시) | 복합 적용 후에도 ±1% 이내 | 필수 (C 포함 시) |
| 인터페이스 결합 정합성 | A/B/C 구현체가 공통 인터페이스를 통해 결합됨 | 중간 |

---

## 판정 규칙

- **필수** 항목 중 하나라도 Fail → 전체 Fail (루프 계속)
- 필수 항목 전부 Pass + 해당 Activity의 **높음** 항목 70% 이상 Pass → 전체 Pass
- 그 외 → Partial (루프 계속, 단 3회차에서는 리포트 저장 후 종료)
- Activity C를 포함하는 경우, Accuracy 보존 항목이 Fail이면 무조건 전체 Fail
