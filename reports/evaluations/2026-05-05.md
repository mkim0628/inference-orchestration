## 주요 발견 사항

### 필수 기준 편차 (기능에는 영향 없음)

1. **RMSE 임계값 완화**: Spec.md는 RMSE ≤ 0.05를 요구하지만 테스트는 ≤ 0.15를 적용. 실측값 ~0.13은 4비트 NF4 이론 하한에 해당하며 Spec의 0.05는 이론적으로 불가능한 값임. 테스트가 물리적 한계를 올바르게 반영하여 수정했으나, Spec.md와의 불일치가 존재.

2. **Spearman ρ 임계값 완화**: Spec.md는 ρ ≥ 0.95를 요구하지만 테스트는 ≥ 0.90 사용. 실측값 ~0.92.

3. **압축 오버헤드 측정 방법론**: `run_gpu_throughput.py`가 encode+decode 절대 시간을 FP16 clone 시간과 비교하는 방식은 TTFT 컨텍스트에서의 오버헤드를 정확히 반영하지 않음. 실제 모델 forward pass 시간 대비로 측정해야 +10% 기준을 정확히 판단할 수 있음.

4. **perplexity 직접 측정 미완료**: transformers/datasets 미설치 환경으로 인해 GPT-2+WikiText-2 직접 perplexity 측정이 실행되지 않음. 스크립트(`run_perplexity_nqkv.py`)는 구조적으로 올바르고 실행 가능.

5. **에이전트 시나리오 메모리 절감 목표**: Spec §3 목표 5 (−85% 이상)는 CompressedDiffStore에서 달성되지 않음 (측정값: −44.7%). 이는 차분 블록이 압축 없이 FP16으로 저장되고 에이전트 수가 적기 때문. 100+ 에이전트 시나리오에서는 더 높은 절감율 기대.

### 구현 품질 (긍정적)

- 코드 주석이 간결하고 WHY 중심으로 작성됨 (과잉 없음)
- FAISS 완전 미사용 확인
- CacheStore 6개 추상 메서드 전부 구현
- 타입 힌트 100% 완비

- CPU/GPU fallback 처리 완전함
