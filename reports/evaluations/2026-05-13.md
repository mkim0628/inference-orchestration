## 총평

Activity A+B+C 세 개를 한 사이클에서 통합 구현하고 855개 테스트 전원 통과를 달성했다. Activity C의 정확도 보존(±1% 필수 조건)이 모든 프록시 지표에서 여유 있게 통과했으며(상대 오차 0.65%, KL < 0.00002, 코사인 0.9999), 이론 메모리 감소율은 목표 −30% 대비 73.4%를 달성했다. 크로스 조합 파이프라인(AgenticChunkPreCachingPipeline)이 CacheStore 인터페이스를 완전 준수하고 A+B+C 세 구현체 간 통합 E2E 테스트까지 통과했다.

## 미해결 이슈

1. **실제 LLM 추론 엔진 미통합**: Throughput (tokens/sec), TTFT p50/p99, TBT 지표는 실제 `vLLM` 또는 `src/engine/runner.py` 연동 없이 시뮬레이션 기반으로만 검증됨. 이는 다음 사이클(vLLM 이식, Report ②)에서 해결될 예정이다.
2. **memory_reduction_ratio()의 이론-실측 차이**: INT8 내부 저장이므로 실측 감소율은 48.4%이나 이론 4-bit 기준은 73.4%로 보고됨. 이중 보고가 혼선을 유발할 수 있으나 Spec이 명시적으로 허용한 설계이므로 Fail 처리하지 않음. 다음 사이클에서 INT4 nibble-pack 실구현 또는 주석 명확화 고려.
3. **Effective Context Length 직접 측정 미실시**: 수식 기반 추정(73.4% 감소 → 3.76×)만 수행. NIAH 16K/64K/128K 실제 평가는 실 모델 없이 불가.

## 다음 사이클 제언

1. **vLLM 이식 우선**: `SRFTFusedINT4KVKernel`을 vLLM KV 캐시 저장 경로에 직접 주입(block_manager 또는 attention_backend 패치)하면 실제 메모리 감소 측정이 가능해진다.
2. **INT4 실구현 전환**: nibble-pack(INT4) 실구현 시 이론치와 실측치 간 차이가 제거되고 메모리 감소율이 73.4%로 직접 달성 가능하다. `torch.int8`로 저장하고 nibble-pack을 별도로 구현하는 방향이 검토 대상이다.
3. **NIAH 벤치마크 연결**: KVFoldAccumulativeRadixCache의 drift plateau 감지를 활용해 128K NIAH 100% 정확도를 실 평가 환경에서 검증하면 Activity B의 "훈련-프리 128K" 클레임을 확인할 수 있다.
4. **Throughput/TTFT 실측**: vLLM 이식 후 `results/2026-05-13/metrics.json`에 throughput(tokens/sec)과 TTFT p50/p99를 추가 기록할 것을 권장한다.
