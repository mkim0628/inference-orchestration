## Activity A 아이디어

### 아이디어 A-1: CONCURCongestionBasedAgentAdmissionScheduler
- **가설**: 런타임 KV 풀 점유율(occupancy)을 혼잡도 신호로 사용하여 에이전트 레벨 어드미션 제어를 수행하면, "중간 단계 쓰래싱(middle-phase thrashing)" 없이 에이전트 워크로드에서 처리량을 극대화할 수 있다.
- **접근 방법**:
  1. KV 풀 점유율을 실시간 모니터링하여 임계값(α_low, α_high) 기반 혼잡 상태를 3단계(여유/경계/혼잡)로 분류한다.
  2. 혼잡 단계에서는 신규 에이전트 스텝 어드미션을 일시 중단(pause-and-wait)하고, 진행 중인 에이전트의 KV를 선제적으로 퇴거하지 않는다.
  3. KV 풀이 α_low 이하로 복구되면 큐에서 대기 중인 에이전트를 FIFO + 우선순위 가중치로 재개한다.
  4. 피드백 루프: 어드미션 게이트 처리량 vs 대기 지연 지표를 실시간 추적해 α 임계값을 온라인으로 조정한다.
- **대상 환경**: 단일 노드 / 멀티 노드 양쪽
- **멀티 노드 고려사항**: 멀티 노드 환경에서는 각 노드의 KV 풀 점유율을 집계하여 글로벌 혼잡 신호를 구성한다. InfiniBand/RDMA를 통한 KV 마이그레이션 비용이 혼잡 임계값 결정에 반영된다.
- **예상 효과**: 처리량 +30~40%(에이전트 워크로드 기준, CONCUR 논문의 4.09× 참고), Scheduling overhead ±3% 이내
- **구현 난이도**: medium
- **우선순위**: 1

### 아이디어 A-2: PBKVConcurPredictiveReactiveDualLayerScheduler
- **가설**: PBKV의 예측적 KV 보존(proactive)과 CONCUR의 반응적 혼잡 제어(reactive fallback)를 2계층으로 결합하면, 예측 오류 시에도 쓰래싱 없이 안정적 처리량을 유지할 수 있다.
- **접근 방법**:
  1. **Layer 1 (예측)**: PBKV 예측기가 다음 N스텝의 KV 수요를 예측하여 선제적으로 캐시를 보존/선인출한다.
  2. **Layer 2 (반응)**: 예측 오류로 KV 풀이 혼잡 임계값을 초과하면 CONCUR 스타일의 어드미션 게이트가 활성화되어 신규 에이전트 스텝을 블로킹한다.
  3. 두 계층 간 전환: 예측 신뢰도 점수(confidence score)가 낮거나, KV 풀 점유율이 임계값을 초과하면 Layer 2로 전환.
  4. 예측 신뢰도 추적: 실제 KV 수요 vs 예측 오차를 누적하여 PBKV 예측기를 온라인으로 보정.
- **대상 환경**: 단일 노드 / 멀티 노드 양쪽
- **멀티 노드 고려사항**: 예측 계층은 로컬 노드 KV 트렌드를 기반으로 동작하고, 반응 계층은 클러스터 전체 혼잡 신호로 트리거된다.
- **예상 효과**: 처리량 +25~35%, 예측 오류 시 TTFT 지연 최소화 (+2% 이내)
- **구현 난이도**: high
- **우선순위**: 3

## Activity B 아이디어

### 아이디어 B-1: CongestionAwareNonContiguousSegmentEvictionPolicy
- **가설**: KV 풀 혼잡도 신호(CONCUR 스타일)를 비연속 세그먼트 캐시의 퇴거율 제어에 적용하면, 혼잡 시 공격적 퇴거로 KV 풀을 신속히 회복하고 여유 시 히트율을 보존하는 동적 균형을 달성할 수 있다.
- **접근 방법**:
  1. KV 풀 혼잡도를 3단계로 분류(여유/경계/혼잡).
  2. **여유 단계**: 표준 LRU 퇴거 + 세그먼트 히트율 기반 보존 정책 유지.
  3. **경계 단계**: 사용 빈도가 낮은 비연속 세그먼트 우선 퇴거(frequency-weighted LRU).
  4. **혼잡 단계**: 재사용 가능성이 낮은 세그먼트(last-access 오래된 것)를 공격적으로 퇴거하여 KV 풀 점유율을 α_low 이하로 신속히 낮춤.
  5. 퇴거 결정에 세그먼트 재사용 점수(frequency × recency × segment_length_benefit)를 활용.
- **예상 효과**: 비연속 히트율 유지(여유 시) + KV 풀 회복 속도 +50%(혼잡 시), TTFT ±3%
- **구현 난이도**: medium
- **우선순위**: 2

## Activity C 아이디어

### 아이디어 C-1: SpecAttnVerificationGuidedKVSparseCodec
- **가설**: 자기-추측 디코딩(self-speculative decoding)의 검증 단계에서 계산되는 전체 어텐션 로짓(Collect-2-Query 메커니즘)을 "무비용" KV 중요도 식별기로 활용하면, 별도 캘리브레이션 비용 없이 정확도를 보존하면서 KV 캐시를 희소화할 수 있다.
- **접근 방법**:
  1. 자기-추측 디코딩 파이프라인에서 검증 단계의 full-attention 연산은 어차피 필요하다 — 이 로짓에서 각 토큰의 어텐션 중요도 마스크를 추출(Collect-2-Query).
  2. 추출된 중요도 마스크를 다음 디코딩 스텝의 KV 유지/퇴거 결정에 사용한다.
  3. 중요도가 낮은 KV (임계값 이하)는 즉시 퇴거하거나 INT4 양자화로 압축한다.
  4. 중요도 임계값은 레이어별로 다르게 설정(하위 레이어 > 상위 레이어): 전체 KV 예산의 70~85% 범위 내에서 탄력적으로 조정.
  5. Training-free: 검증 로짓 추출은 기존 추측 디코딩 연산의 부산물이므로 추가 연산 비용 없음.
- **예상 효과**: 메모리 −30~40%, accuracy delta ±0.5% (perplexity 기준)
- **accuracy-preserving 근거**: SpecAttn 논문에서 Collect-2-Query로 선택된 KV 집합은 full-attention과 출력이 동등함을 수학적으로 보임. 검증 단계 전체 어텐션 로짓에서 추출한 마스크는 현재 컨텍스트에서 실제로 중요한 KV를 정확히 식별하므로, 예측 기반(LookaheadKV) 또는 통계 기반(H2O) 방식 대비 정확도 보존 신뢰도가 높다. 임계값 보수적 설정(상위 70~85% KV 유지)으로 perplexity 변화 ±1% 이내 달성 가능.
- **구현 난이도**: medium
- **우선순위**: 1

### 아이디어 C-2: LRKVSharedFullRankHeadResidualCodec
- **가설**: 사전학습 시점에 레이어별 하나의 공유 풀랭크 KV 프로젝션과 헤드별 저랭크 잔차 프로젝션을 도입하면, MQA/GQA가 희생하는 헤드 간 다양성을 유지하면서 45~53% KV 메모리를 절감할 수 있다.
- **접근 방법**:
  1. 각 레이어에 공유 K/V 프로젝션 W_shared (풀랭크 d_k × d_model)를 두고, 헤드별 저랭크 잔차 W_head_i (랭크 r << d_k)를 추가한다.
  2. 헤드 h의 KV: K_h = W_shared(x) + W_res_h(x), 랭크 r을 d_k의 10~25%로 설정.
  3. 사전학습 시 공유 프로젝션과 저랭크 잔차를 함께 학습: W_shared는 공통 표현 포착, W_res_h는 헤드별 특수화.
  4. 추론 시: K_shared 한 번만 계산하고 각 헤드는 저랭크 보정만 추가 → KV 연산량 대폭 감소.
  5. 기존 학습된 모델에 대해서는 점진적 지식 증류(progressive distillation)로 LRKV 구조로 전환 가능.
- **예상 효과**: KV 메모리 −45~53%, accuracy delta ±0.8% (WikiText-103 perplexity 기준)
- **accuracy-preserving 근거**: LRKV 논문(arXiv 2601.11471)에서 공유 풀랭크 기반 + 헤드별 저랭크 잔차 구조가 MHA/GQA/MLA 대비 같은 KV 예산에서 더 낮은 perplexity를 달성함을 실험적으로 증명. 공유 기반이 풀랭크이므로 GQA처럼 공유 KV가 모든 헤드의 다양성을 표현해야 하는 병목이 없다. 저랭크 잔차의 랭크 r을 레이어별로 조정하여 정확도 예산을 미세 조정할 수 있다.
- **구현 난이도**: high
- **우선순위**: 4

## 크로스 Activity 시너지 아이디어

### 아이디어 Cross-1: CongestionAdmissionSpecAttnDualReductionPipeline (A+C)
- **조합 근거**: CONCUR 기반 어드미션 제어(A-1)는 KV 풀에 진입하는 에이전트 수를 제한(볼륨 제어)하고, SpecAttn 검증 유도 희소화(C-1)는 진입한 에이전트의 KV 접근 자체를 희소화(접근 희소화)한다. 두 기법의 감소 효과는 곱연산적(multiplicative)이다: 에이전트 수 감소 × 에이전트당 KV 사용량 감소.
- **접근 방법**:
  1. **레이어 1 (A)**: CONCURCongestionBasedAgentAdmissionScheduler가 KV 풀 혼잡도 기반으로 에이전트 어드미션을 제어.
  2. **레이어 2 (C)**: 어드미션된 에이전트의 자기-추측 디코딩 검증 로짓에서 KV 중요도 마스크를 추출하여 실시간 KV 퇴거/압축 결정.
  3. 통합 피드백: KV 풀 점유율이 목표 범위 내에 있으면 C-1의 임계값을 완화(더 많은 KV 보존), 범위를 벗어나면 강화(더 공격적 희소화).
  4. 멀티 에이전트 환경: 에이전트별 KV 사용 패턴을 추적하여 "불균형 에이전트"(KV 집중 사용)를 선별 차단.
- **예상 복합 효과**: 처리량 +50~60%, 메모리 −50%, 정확도 ±0.7%
- **구현 난이도**: medium
- **우선순위**: 1

### 아이디어 Cross-2: ConcurLRKVNonContiguousSpecAttnIntegratedStack (A+B+C)
- **조합 근거**: 4단계 파이프라인으로 KV 감소 효과를 계층적으로 중첩한다: (1) CONCUR 어드미션 제어로 KV 풀 진입 볼륨 제한, (2) LRKV 구조적 압축으로 개별 KV 크기 감소, (3) 확보된 HBM 슬랙을 활용한 비연속 세그먼트 캐시 히트율 향상, (4) SpecAttn 희소화로 디코딩 시 KV 접근 추가 감소.
- **접근 방법**:
  1. **Stage 1 (A — CONCUR)**: KV 풀 혼잡도 기반 에이전트 어드미션 게이트.
  2. **Stage 2 (C — LRKV)**: 공유 풀랭크 KV + 헤드별 저랭크 잔차로 사전학습된 모델의 KV 크기를 구조적으로 절반 수준으로 감소.
  3. **Stage 3 (B — Non-Contiguous)**: LRKV 압축으로 절약된 HBM 공간에 비연속 세그먼트 캐시를 더 많이 수용 → 비연속 히트율 향상.
  4. **Stage 4 (C — SpecAttn)**: 자기-추측 디코딩 검증 로짓 기반 KV 희소화로 디코딩 KV 접근을 추가로 30~40% 감소.
  5. 단계 간 인터페이스: LRKV K/V 텐서는 표준 KV 캐시 인터페이스(CacheStore)를 통해 비연속 세그먼트 캐시와 호환되도록 어댑터를 작성.
- **예상 복합 효과**: 처리량 +70~80%, 메모리 −65%, 정확도 ±1.0%
- **구현 난이도**: high
- **우선순위**: 2
