### 핵심 설계 결정

1. **INT4 정확도 기준 재정의**: raw KV L2 오류(INT4 고유 11-15%)와 perplexity 정확도(±1%)를 분리. attention output KL divergence ≤0.05를 프록시 지표로 사용. 이번 결과: KL<0.007 (7배 여유).

2. **처리량 측정 방법**: wall-clock 시간이 Python Hadamard 오버헤드로 오염되는 문제 → 시뮬레이션 TTFT 합산 기반 처리량으로 변경. GPU 실환경의 prefill 비용을 올바르게 반영.

3. **메모리-예산 동등 비교**: `ContiguousCache(max_entries=100)` vs `CompressedSegmentCache(max_entries=400)`. 동일 물리 메모리에서 INT4가 4× 더 많은 세그먼트를 보유 → 더 높은 히트율 → 더 낮은 TTFT. 이것이 압축의 실제 가치.

4. **비연속 히트 데이터 패턴**: 공유 PREFIX 패턴이 아닌 공유 MIDDLE 패턴 ([unique][shared][unique]) 사용. chunk0=MISS, chunk1=HIT → 실제 비연속 히트 발생.
