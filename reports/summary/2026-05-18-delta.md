## Activity별 결과 요약

### Activity A — AMPDLazySegmentFetchSchedulerMixin
- AMPD(2602.14516) pull-on-demand 지연 읽기 원칙을 비연속 세그먼트 스케줄러에 최초 적용.
- 세그먼트 메타데이터(segment_id, position_range, source_node_id, approx_kv_size) 선행 전달 후 스케줄 확정 순간에만 KV pull.
- tier-based HBM/DDR/REMOTE 비용 차등화로 최적 소스 노드 자동 선택.
- unnecessary_transfer_ratio 지표 도입 (실측 0.90 — 목표 0.40 대비 개선 필요).
- vLLM 실측: p50 0.036ms (목표 5ms 대비 139배 여유), 메타데이터 등록 0.0073ms (<0.1ms).
- enable_multinode=True 멀티노드 KV 라우팅 지원 (로컬 0.01ms vs 원격 5.0ms 구분).
- issubclass(AMPDScheduler, Scheduler)=True vLLM 서브클래싱 확인.

### Activity B — AMPDAdapShotLazyLoadKVCacheManagerMixin
- AMPD 지연 로드 + AdapShot RoPE 재인코딩 3단계 비동기 파이프라인.
- SHA-256 위치-독립 콘텐츠 해시 키로 비연속 세그먼트 독립 조회.
- resolve_segments_b18 3단계(hit_metas + miss_chunk_indices 반환) 파이프라인 정상 동작.
- noncontiguous_hit_rate 66.7% (3청크 시나리오, chunk2만 저장 후 기준).
- vLLM Loop 2 수정: load_and_reencode_b18에 Union[int, List[int]] 배치 타입 안전성 추가. List 입력 배치 처리 확인, 길이 불일치 ValueError 정상 발생.
- issubclass(AMPDAdapShotVllmKVCacheManager, KVCacheManager)=True vLLM 서브클래싱 확인.

### Activity C — DPAttentionAwareCompressionAttentionHook (+ DPAttentionAwareCompressionSelector)
- SGLang DP Attention 상태 인식 환경별 압축 정책 선택기.
- vLLM Loop 1 문제: `(q.float() * scale).half()` FP16 재패킹으로 실측 KV Memory Reduction = 0%.
- vLLM Loop 2 수정: INT8 텐서 직접 저장 (k_int8: torch.int8, v_int8: torch.int8), scale을 _store_scale() 사이드채널 저장.
  - 메모리: FP16 32,768 bytes → INT8+scale 17,408 bytes = **46.9% 감소** (목표 ≥30% 달성).
  - Per-row scale: `x_2d.abs().max(dim=-1, keepdim=True)[0] / 127.0` → shape (n_rows, 1), outlier row 정밀도 저하 방지.
  - 정확도: attention relative error 0.005853 (<0.01), cosine 0.999982 (≥0.99), KL 0.0000167 (<0.015). 전부 MANDATORY Pass.
- read_from_cache()에서 항상 FP16 반환 보장.
- 독립 구현: KV Memory Reduction 68.75% (memory_reduction_ratio=0.6875), ~3.2× 컨텍스트 지원.

### Cross A+B+C — DPAttentionCrossABCCodec (AMPDPrefillShareNonContiguousStack)
- 세그먼트 히트율 + 비연속 히트율 + 압축 지표 통합.
