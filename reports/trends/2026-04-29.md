# 트렌드 리포트 — 2026-04-29

## 전체 요약

1. **Activity A (스케줄링)**: vLLM Q2 2026 로드맵에 KV 캐시 매니저 재설계·스케줄러 개선이 공식 등록됐고, llm-d의 분산 KV 라우팅이 프로덕션 적용 단계에 진입했다. Gateway API Inference Extension v1.3.1(2026-02 GA)이 KV 캐시 인식 스케줄링을 쿠버네티스 수준에서 표준화하고 있다.
2. **Activity B (비연속 재사용)**: KV Packet(arXiv 2604.13226, 2026-04-14)이 경량 trainable adapter로 재계산 없이 위치-독립 KV 재사용을 달성하는 새로운 접근법을 제시했다. CacheSlide(FAST26)는 에이전트 워크플로우에서 상대-위치 의존 캐싱(RPDC)으로 3~5× 처리량 향상을 달성했다.
3. **Activity C (압축)**: SAW-INT4(arXiv 2604.19157, Together AI)가 블록-대각 Hadamard 회전으로 INT4에서 near-BF16 정확도를 달성하며 추가 오버헤드 없이 SGLang에 통합됐다. INT8에서 INT4로의 전환이 실용화 단계에 진입했다.
4. **B+C 시너지 확장**: ChunkKV의 의미 단위 청킹과 SAW-INT4의 시스템-인식 양자화를 결합하면 이전 사이클의 위치-독립 해싱 기반 B+C 구현을 한 단계 발전시킬 수 있다.
5. **A+B+C 방향**: vLLM Q2 로드맵의 스케줄러 재설계와 llm-d 라우팅이 이번 연구의 Activity A 통합 타이밍으로 적합하다.

## Activity B — Non-Contiguous KV Cache Reuse

### 새로운 논문·구현체

#### 1. [KV Packet: Recomputation-Free Context-Independent KV Caching for LLMs](https://arxiv.org/abs/2604.13226)
- **출처**: arXiv (2026-04-14, Hugging Face Papers 주목)
- **핵심 아이디어**: 캐시된 문서를 불변 "패킷"으로 취급하고, self-supervised distillation로 학습된 경량 soft-token adapter가 컨텍스트 불연속성을 브릿징. CacheBlend·EPIC 대비 재계산 FLOPs ≈ 0, TTFT 추가 증가 없음.
- **연구 목표와의 관련성**: high
- **주목할 점**: 기존 위치-독립 캐싱의 핵심 병목(선택적 재계산 오버헤드)을 adapter 학습으로 우회. Activity B의 현재 구현(`CompressedSegmentCache`)에 adapter 레이어를 추가하면 히트 시 TTFT를 추가로 줄일 수 있음. Llama-3.1·Qwen2.5에서 검증됨.

#### 2. [CacheSlide: Unlocking Cross Position-Aware KV Cache Reuse for Accelerating LLM Serving](https://www.usenix.org/conference/fast26/presentation/liu-yang)
- **출처**: USENIX FAST26 (최고 시스템 학회)
- **핵심 아이디어**: 에이전트 워크플로우에서 세그먼트가 절대 위치는 달라도 상대 순서가 유지되는 "상대-위치 의존 캐싱(RPDC)" 패턴을 정의하고, Chunked Contextual Position Encoding + Weighted Correction Attention으로 이를 효율적으로 처리. vLLM KV 캐시 관리 확장으로 구현.
- **연구 목표와의 관련성**: high
- **주목할 점**: FAST26 채택으로 비연속 KV 재사용이 시스템 커뮤니티 주류 연구로 인정됨. 처리량 3.5~5.8×, 레이턴시 3.11~4.3× 개선. vLLM 확장 구현이 Activity B의 6단계 이식과 직접 연결.

#### 3. [KVLink: Accelerating Large Language Models via Efficient KV Cache Reuse](https://arxiv.org/abs/2502.16002)
- **출처**: arXiv (2025-02 → 2026-02 v2 업데이트)
- **핵심 아이디어**: 비연속 KV 세그먼트를 어텐션 전에 연결(link)하는 효율적 방법. InfoBlend의 KVLink 메커니즘의 기반 논문. 위치-독립 세그먼트 연결 시 최소 재계산만 수행.
- **연구 목표와의 관련성**: high
- **주목할 점**: v2 업데이트로 더 다양한 모델·시나리오 검증. Activity B의 segment concatenation 구현의 이론적 기반.

#### 4. [CacheClip: Accelerating RAG with Effective KV Cache Reuse](https://arxiv.org/abs/2510.10129)
- **출처**: arXiv (2025-10)
- **핵심 아이디어**: RAG 시나리오에서 검색된 문서 KV를 재사용하는 클리핑 전략. 문서별 KV를 독립적으로 캐싱하여 비연속 재사용 가능.
- **연구 목표와의 관련성**: medium
- **주목할 점**: Activity B의 실용 사례. RAG + 비연속 KV 재사용의 결합이 긴 문서 컨텍스트에서의 히트율 측정 기준을 제공.

## Activity C — KV Cache Compression

### 새로운 논문·구현체

#### 1. [SAW-INT4: System-Aware 4-Bit KV-Cache Quantization for Real-World LLM Serving](https://arxiv.org/abs/2604.19157)
- **출처**: arXiv (2026-04, Together AI)
- **핵심 아이디어**: 블록-대각 Hadamard 회전(BDR)을 KV 캐시 저장 직전에 적용한 후 token-wise INT4 양자화. 융합(fused) 회전-양자화 커널이 paged KV-cache 레이아웃에 직접 통합되어 추가 오버헤드 제로. SGLang 포크 구현, near-BF16 정확도 달성.
- **연구 목표와의 관련성**: high
- **주목할 점**: INT4에서 추가 오버헤드 없이 near-BF16 정확도는 Activity C의 "accuracy-preserving" 제약 달성의 새로운 벤치마크. 기존 INT8(−50%) → INT4(−75%) 메모리 절감으로 목표 −30%를 대폭 초과 달성 가능. 오픈소스(github.com/togethercomputer/saw-int4).

#### 2. [ChunkKV: Semantic-Preserving KV Cache Compression for Efficient Long-Context LLM Inference](https://arxiv.org/abs/2502.00299)
- **출처**: arXiv (2025-02 → 2026 OpenReview 채택)
- **핵심 아이디어**: 개별 토큰 대신 의미 청크(semantic chunk) 단위로 KV 퇴거 결정. 청크 내 어텐션 스코어 합산으로 청크 중요도를 계산하고 통째로 유지·퇴거. layer-wise 인덱스 재사용으로 추가 연산 26.5% 감소.
- **연구 목표와의 관련성**: high
- **주목할 점**: Activity B의 청크 단위 캐싱과 Activity C의 퇴거 결정을 동일 청크 경계에서 수행하면 B+C 통합의 일관성이 높아짐. 공격적 압축 비율에서 기존 방법 대비 10% 성능 향상.

#### 3. [TTKV: Temporal-Tiered KV Cache for Long-Context LLM Inference](https://arxiv.org/html/2604.19769)
- **출처**: arXiv (2026-04)
- **핵심 아이디어**: 시간 순서에 따라 KV 캐시를 계층화(tiering). 최근 토큰 KV는 GPU에 고정밀도, 오래된 토큰 KV는 CPU/압축으로 계층 이동.
- **연구 목표와의 관련성**: medium
- **주목할 점**: Activity C의 퇴거 정책과 계층 오프로딩을 결합하는 새로운 방향. 장기 메모리 지원 목표와 직결.

#### 4. [Don't Waste Bits! Adaptive KV-Cache Quantization for Lightweight On-Device LLMs](https://arxiv.org/html/2604.04722v1)
- **출처**: arXiv (2026-04)
- **핵심 아이디어**: 토큰 중요도에 비례해 비트폭을 동적 할당. 중요 토큰은 FP16, 비중요 토큰은 INT2~INT4. 메모리·레이턴시 기대값을 최소화하면서 정확도 유지.
- **연구 목표와의 관련성**: medium
- **주목할 점**: SAW-INT4(균일 회전 기반)와 달리 토큰별 적응 정밀도 전략. 온디바이스 포커스지만 비트폭 적응 원리는 서버 환경에도 적용 가능.

## Activity 간 시너지 포인트

- **B + C (기존 → 강화)**: 전날 구현(위치-독립 세그먼트 해시 + FP16/INT8 혼합 정밀도)에서 C 측을 SAW-INT4의 Hadamard 회전 기반 INT4로 업그레이드하면 메모리 절감이 −68.8% → −75% 이상으로 향상 가능. ChunkKV와 B의 청크 경계를 동기화하면 퇴거-재사용 의사결정이 일치하여 B+C 결합 효율 추가 향상.

- **A + B**: llm-d-kv-cache의 글로벌 KV 블록 인덱서를 Activity B의 비연속 세그먼트 인덱스와 연결하면, 멀티-노드에서 "부분 세그먼트 히트 정보를 인식한 라우팅"이 가능해짐.

- **B + C + 신기법**: KV Packet의 soft-token adapter를 SAW-INT4 압축 KV 위에 적용하면 "압축된 비연속 세그먼트 + 재계산 없는 컨텍스트 브릿징" 조합이 실현 가능. TTFT 절감 + 메모리 절감 동시 달성.

- **A + B + C 통합 방향**: vLLM Q2 2026 스케줄러 재설계 시점에 Activity A(llm-d 스타일 라우팅), B(CacheSlide/KV Packet 방식), C(SAW-INT4)를 단계적으로 통합하면 세 목표 지표를 동시에 달성하는 종합 시스템이 가능.

## idea-generator에게 전달할 포인트

### Activity A
- vLLM Q2 2026 로드맵의 스케줄러 재설계가 Activity A 구현 타이밍. 현재 `Scheduler` 서브클래싱 방식이 Q2 재설계 이전에 안정적으로 이식 가능.
- llm-d-kv-cache의 KVEvents 스트리밍이 멀티-노드 KV 상태를 실시간으로 추적하는 기성 인프라. Activity A 멀티-노드 라우팅 구현 시 참고 설계.
- PrefillShare의 공유 프리필 모듈이 Activity B의 비연속 세그먼트 공유와 자연스럽게 결합 — "공유 프리필 + 비연속 재사용" 조합이 새로운 A+B 시너지.

### Activity B
- KV Packet(2604.13226)이 재계산 없는 PIC의 새 패러다임. 현재 구현에 soft-token adapter를 추가하면 히트 시 TTFT 추가 감소 가능.
- CacheSlide의 RPDC 패턴 — 에이전트 워크플로우에서 상대 위치 유지 세그먼트에 대한 특수 처리를 Activity B에 추가하면 적용 범위 확대.
- 청크 경계를 ChunkKV(C)와 공유하면 B+C 통합에서 퇴거-재사용 의사결정 일관성 확보.

### Activity C
- SAW-INT4 (2604.19157): INT4 + Hadamard 회전이 near-BF16 정확도로 Activity C의 "accuracy ±1%" 제약을 INT4에서도 만족. FP8 → INT8 → INT4 전환 로드맵에서 INT4가 실용화.
- ChunkKV의 의미 청크 단위 퇴거가 Activity B의 청크 기반 캐싱과 경계를 공유 — B+C 통합 구현에서 청크 크기를 동기화하면 코드 복잡도 감소.
- 기존 −68.8% → SAW-INT4(−75%) + ChunkKV 퇴거 추가 시 −80%+ 달성 가능성. 목표 −30% 대비 2.5× 초과 달성이 현실적.
