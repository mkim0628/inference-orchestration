# 트렌드 리포트 — 2026-04-30

## 전체 요약

이번 주(2026년 4월 말) KV 캐시 연구의 핵심 흐름은 다음과 같다.
(1) P/D 분리(Disaggregated Prefill/Decode) 아키텍처가 크로스-데이터센터 수준으로 확장되고 있으며, 대역폭-인식 스케줄링이 핵심 과제로 부상했다.
(2) 비연속 KV 재사용 분야에서는 위치-독립 캐싱(Position-Independent Caching) 기법과 소프트-토큰 어댑터 기반 "KV Packet"이 등장해 컨텍스트 불연속성을 극복하려는 시도가 주목된다.
(3) 압축 분야에서는 INT4 회전(Hadamard rotation) 기반 양자화(SAW-INT4)와 어텐션 기반 적응형 3-상태(retain/compress/evict) 관리(ARKV)가 정확도 보존 측면에서 두각을 나타냈다.
(4) 실제 서빙 프레임워크(vLLM, SGLang, llm-d)에서 분산 KV 캐시 스케줄링을 직접 통합하는 움직임이 빨라지고 있다.
(5) 세 Activity 간 시너지로, 위치-독립 KV 세그먼트를 양자화하여 P/D 분리 환경에서 전송하는 통합 파이프라인이 차기 연구 주제로 수렴되고 있다.

## Activity A — KV Cache-aware Scheduling / Orchestration

### 새로운 논문·구현체

#### 1. [FlowKV: A Disaggregated Inference Framework with Low-Latency KV Cache Transfer and Load-Aware Scheduling](https://arxiv.org/abs/2504.03775)
- **출처**: arXiv 2504.03775 (April 2026)
- **핵심 아이디어**: Prefill(P)과 Decode(D) 단계를 완전히 분리하고, KV 캐시 전송 비용을 최소화하는 전용 통신 채널과 Load-Aware Scheduler를 결합한다. 노드 간 부하 불균형을 실시간 모니터링하여 요청 라우팅을 조정한다.
- **연구 목표와의 관련성**: high — 멀티노드 KV 라우팅 및 스케줄링 오버헤드 최소화가 Activity A의 핵심 과제와 직결된다.
- **주목할 점**: KV 전송 지연(latency)을 TTFT에 포함시켜 측정하며, 기존 DistServe 대비 전송 오버헤드를 크게 낮췄다고 보고한다.

#### 2. [Prefill-as-a-Service: KVCache of Next-Generation Models Could Go Cross-Datacenter](https://arxiv.org/abs/2604.15039)
- **출처**: arXiv 2604.15039 (April 2026)
- **핵심 아이디어**: 장문 컨텍스트 Prefill을 컴퓨팅 집약형 별도 클러스터에서 처리한 후, 생성된 KVCache를 이더넷을 통해 로컬 Decode 클러스터로 전송한다. 대역폭-인식(bandwidth-aware) 스케줄링과 캐시-인식 요청 배치(cache-aware placement)를 결합한다.
- **연구 목표와의 관련성**: high — 네트워크-인식 KV 배치와 크로스-노드 캐시 재사용이 Activity A의 멀티노드 시나리오에 직접 해당한다.
- **주목할 점**: 상용 이더넷 기반이므로 InfiniBand 없이도 P/D 분리가 가능함을 실증한다. KV 효율성 기법(모델 측)과 시스템 측 선택적 오프로딩을 함께 사용한다.

#### 3. [BanaServe: Unified KV Cache and Dynamic Module Migration](https://onlinelibrary.wiley.com/doi/10.1002/spe.70054)
- **출처**: Software: Practice and Experience (Wiley, 2026)
- **핵심 아이디어**: 레이어 단위 가중치 마이그레이션과 어텐션 레이어 단위 KV 캐시 마이그레이션을 결합한 글로벌 KV Cache Store를 운영한다. 계산·메모리 자원 모두를 지속적으로 재균형화한다.
- **연구 목표와의 관련성**: high — KV Migration 과 멀티노드 오케스트레이션 측면에서 Activity A의 핵심 과제다.
- **주목할 점**: 레이어-와이즈 오버랩 전송(layer-wise overlapped transmission)으로 재균형 비용을 숨긴다.

#### 4. [llm-d: KV-Cache Wins You Can See — Distributed Scheduling](https://llm-d.ai/blog/kvcache-wins-you-can-see)
- **출처**: llm-d 공식 블로그 (April 2026)
- **핵심 아이디어**: vLLM 기반 분산 KV 캐시 스케줄링 구현체. 접두사 인식 배치(prefix-aware batching)와 캐시 국소성 우선 스케줄링을 실제 서빙 환경에서 적용하고 히트율 개선을 정량적으로 보고한다.
- **연구 목표와의 관련성**: high — 실제 vLLM 이식 대상으로 직접 활용 가능하다.
- **주목할 점**: "캐시 히트율 향상 → TTFT 감소" 인과관계를 실측 데이터로 제시해 Activity A 벤치마킹 기준으로 활용 가능하다.

### 동향 변화 감지

| 기법 | 이전 상태 | 이번 주 변화 |
|------|---------|------------|
| Prefix-aware batching | vLLM/SGLang 단일 노드 중심 구현 | llm-d에서 분산 환경 실증, 히트율 정량 보고 |
| Cache-locality scheduling | 연구 수준 프로토타입 위주 | FlowKV·BanaServe에서 Load-Aware 스케줄러로 실용화 |
| Multi-node KV routing | InfiniBand/NVLink 전제 | 이더넷 기반 크로스-데이터센터(PrefillaaS)로 적용 범위 확장 |
| Disaggregated prefill | P/D 분리 후 단순 전송 | 전송 오버헤드를 TTFT에 포함시켜 통합 평가하는 방식 정착 |

## Activity B — Non-Contiguous KV Cache Reuse

### 새로운 논문·구현체

#### 1. [KV Packet: Recomputation-Free Context-Independent KV Caching for LLMs](https://arxiv.org/abs/2604.13226)
- **출처**: arXiv 2604.13226 (April 2026)
- **핵심 아이디어**: 캐시된 문서를 위치-독립적인 "패킷"으로 취급하고, 경량 학습 가능한 소프트-토큰 어댑터를 통해 컨텍스트 불연속성을 교정한다. 자기지도 증류(self-supervised distillation)로 어댑터를 학습해 재계산 없이 비연속 세그먼트를 조합한다.
- **연구 목표와의 관련성**: high — 비연속 KV 세그먼트를 정확도 손실 없이 재사용하는 것이 Activity B의 핵심 과제와 정확히 일치한다.
- **주목할 점**: 학습된 어댑터가 "정확도 보존"을 담보하므로 Activity C(압축 후 정확도)와 결합 가능성이 크다.

#### 2. [An Experimental Study of KV Cache Reuse Strategies in Chunk-Level Caching Systems](https://arxiv.org/abs/2603.20218)
- **출처**: arXiv 2603.20218 (March 2026)
- **핵심 아이디어**: 청크 수준 캐싱(Chunk-Level Caching, CLC)에서 미리 계산된 KV 캐시를 재사용할 때, 청크 간 교차-어텐션(cross-attention) 의존성 누락으로 인한 출력 품질 저하를 체계적으로 실험한다. 다양한 재사용 전략(완전 재사용, 부분 재계산, CacheBlend 등)의 트레이드오프를 정량화한다.
- **연구 목표와의 관련성**: high — 비연속 세그먼트 재사용 시 정확도 손실 원인과 완화 전략을 실증적으로 분석한다.
- **주목할 점**: CacheBlend 대비 각 전략의 정확도-지연 트레이드오프 매핑이 향후 실험 설계에 직접 활용 가능하다.

#### 3. [MEPIC: Memory Efficient Position Independent Caching for LLM Serving](https://arxiv.org/html/2512.16822)
- **출처**: arXiv 2512.16822 (December 2025, 참조용)
- **핵심 아이디어**: RoPE 인코딩을 적용하지 않은 상태(NoPE)로 KV를 저장하고, 실행 시점에 퓨즈드 RoPE 어텐션 커널에서 위치 정보를 주입한다. KV가 절대 토큰 위치와 분리되므로 청크를 프롬프트 내 임의 위치에 재사용할 수 있다.
- **연구 목표와의 관련성**: high — 위치-독립 캐싱은 비연속 재사용의 가장 근본적인 해결 방식이다.
- **주목할 점**: KV Packet(2604.13226)과 MEPIC은 상호 보완적이다: MEPIC은 저장 구조를 바꾸고, KV Packet은 어댑터로 컨텍스트 의존성을 보정한다.

#### 4. [vLLM Generalized KV Cache Reuse (Issue #25672)](https://github.com/vllm-project/vllm/issues/25672)
- **출처**: GitHub vllm-project/vllm (활성 이슈, 2026)
- **핵심 아이디어**: vLLM에서 접두사 일치뿐 아니라 비접두사 위치의 KV 세그먼트도 재사용하는 일반화된 캐시 재사용 메커니즘을 설계하는 논의.
- **연구 목표와의 관련성**: high — Activity B 구현체를 vLLM에 직접 이식할 때 참고해야 하는 핵심 이슈다.
- **주목할 점**: 현재 아키텍처의 제약(prefix-only RadixAttention)과 비연속 재사용을 위해 필요한 변경 사항이 상세히 논의된다.

### 동향 변화 감지

| 기법 | 이전 상태 | 이번 주 변화 |
|------|---------|------------|
| Segmented Hash / Radix | 접두사 일치 전용, 단일 노드 | vLLM Issue #25672에서 일반화 재사용 설계 논의 시작 |
| CacheBlend / Blending | 레이어 1 선택적 재계산으로 정확도 보정 | 실험 연구(2603.20218)로 트레이드오프 정량화 완료 |
| Position-independent caching | MEPIC(NoPE 저장) 제안 단계 | KV Packet 어댑터 방식이 추가 등장, 두 접근이 경쟁/보완 |

## Activity C — KV Cache Compression

### 새로운 논문·구현체

#### 1. [SAW-INT4: System-AWare 4-Bit KV-Cache Quantization for Real-World LLM Serving](https://arxiv.org/html/2604.19157)
- **출처**: arXiv 2604.19157 (April 2026)
- **핵심 아이디어**: 네 가지 양자화 전략(naive INT4, KMeans 벡터 양자화, Hessian-aware, Hadamard 회전 기반)을 토큰 단위로 체계적으로 비교한다. 회전 + 토큰 단위 INT4 조합이 정확도-효율 트레이드오프 최고점임을 실증한다.
- **연구 목표와의 관련성**: high — "정확도 보존" 조건 하 4비트 양자화가 Activity C의 핵심 목표와 일치하며, Qwen3/GLM-4 기반 최신 모델에서 검증됐다.
- **주목할 점**: Hadamard 회전이 이상치(outlier)를 제거해 INT4에서 정확도를 유지하는 핵심 기제임이 명확히 분석되어 구현 기준으로 활용 가능하다.

#### 2. [ARKV: Adaptive and Resource-Efficient KV Cache Management](https://arxiv.org/html/2603.08727)
- **출처**: arXiv 2603.08727 (March 2026)
- **핵심 아이디어**: KV 토큰을 retain / compress / evict 세 가지 상태로 분류하는 tri-state 캐싱 프레임워크. 어텐션 기반 heavy-hitter 점수와 레이어별 스케줄링으로 세밀하고 경량한 적응형 캐시 제어를 수행한다.
- **연구 목표와의 관련성**: high — 토큰 퇴거(eviction) + 압축(compression)을 동시에 관리하는 프레임워크로, "정확도 보존 KV 압축" 연구 목표와 직결된다.
- **주목할 점**: compress 상태에서 저비트 양자화를 적용하면 SAW-INT4와 결합한 계층적 압축 파이프라인을 구성할 수 있다.

#### 3. [Don't Waste Bits! Adaptive KV-Cache Quantization for Lightweight On-Device LLMs](https://arxiv.org/html/2604.04722)
- **출처**: arXiv 2604.04722 (April 2026)
- **핵심 아이디어**: 토큰 중요도를 기반으로 {2-bit, 4-bit, 8-bit, FP16} 중 비트폭을 동적으로 선택하는 학습된 정책(Huffman 코딩 영감). 토큰 빈도, 품질 점수, 어텐션 분산, 엔트로피 기반 불확실성 등 경량 특징을 사용한다.
- **연구 목표와의 관련성**: high — 온디바이스(제한된 메모리 예산) 시나리오에서 정확도를 유지하면서 메모리를 최소화하는 전략이다.
- **주목할 점**: 비트폭 선택 정책이 Activity B의 세그먼트 중요도 점수와 결합 가능하다(중요 세그먼트=높은 비트폭, 덜 중요한 세그먼트=낮은 비트폭).

#### 4. [GPU-Accelerated INT8 Quantization for KV Cache Compression](https://arxiv.org/html/2601.04719v1)
- **출처**: arXiv 2601.04719 (January 2026)
- **핵심 아이디어**: GPU 커널 수준에서 INT8 양자화를 가속하여 KV 캐시를 4배 압축하면서 정확도 손실을 최소화한다. vLLM 통합 구현을 제공한다.
- **연구 목표와의 관련성**: medium — INT8은 INT4보다 정확도 보존 측면에서 안전하나 압축률이 낮다. vLLM 통합 참고용으로 유용하다.
- **주목할 점**: vLLM 공식 문서에서 Quantized KV Cache 기능으로 지원 중이며, 실제 이식 작업(vllm-porter)의 기반이 된다.

### 동향 변화 감지

| 기법 | 이전 상태 | 이번 주 변화 |
|------|---------|------------|
| Quantization (INT4/FP8) | INT8 위주 실용화, INT4는 정확도 우려 | SAW-INT4에서 Hadamard 회전으로 INT4 정확도 문제 해소 실증 |
| Token eviction (H2O, SnapKV) | 단순 heavy-hitter 기반 퇴거 | ARKV의 tri-state (retain/compress/evict) 로 세분화 |
| Adaptive bit-width | 연구 제안 수준 | "Don't Waste Bits"에서 온디바이스 학습 정책으로 실용화 |
| Low-rank approximation | ThinKV 등 추론 모델 특화 | 이번 주 새 논문 없음, SAW-INT4·ARKV가 주류 흐름 |

## Activity 간 시너지 포인트

**A + B (스케줄링 + 비연속 재사용)**
- FlowKV의 Load-Aware Scheduler가 비연속 세그먼트 히트율을 메타데이터로 수집하면, 공통 청크를 가진 요청을 같은 P 노드로 라우팅하여 캐시 재사용률을 극대화할 수 있다.
- llm-d의 분산 스케줄러에 MEPIC(위치-독립 KV)을 결합하면 청크 위치에 무관하게 노드 간 KV를 공유할 수 있다.

**B + C (비연속 재사용 + 압축)**
- KV Packet의 소프트-토큰 어댑터를 SAW-INT4로 양자화하면, 압축된 비연속 KV를 정확도 보존 조건 하에서 재사용하는 파이프라인이 가능하다.
- ARKV의 tri-state 프레임워크에서 "compress" 상태 토큰을 청크 단위로 묶어 청크 수준 재사용의 우선 후보로 활용할 수 있다.

**A + C (스케줄링 + 압축)**
- PrefillaaS(크로스-데이터센터 KV 전송)에서 네트워크 비용이 높을 때, KV를 INT4로 압축 후 전송하면 대역폭 요구를 줄일 수 있다. "전송 전 압축" 정책이 스케줄러 결정 요소가 된다.
- ARKV의 레이어별 compress/evict 결정이 스케줄러의 메모리 압력 신호와 연동되면, 동적 메모리 예산 조절이 가능하다.

**A + B + C (통합)**
