# 트렌드 리포트 — 2026-05-04

## 전체 요약

1. **Activity A (스케줄링)**: HPDC '26 채택 SAGA가 워크플로우 DAG를 스케줄러에 명시적으로 노출해 오프라인 최적 Bélády 정책에 근접하는 온라인 KV 캐시 스케줄링을 실현했다. Pythia는 에이전트 워크플로우 예측 가능성을 활용해 프리픽스 캐시 히트율과 자원 경쟁을 동시에 개선하며 2.9× 처리량 향상을 달성했다. llm-d 0.5(2026-04-03 출시)는 계층형 KV 오프로딩과 LoRA-인식 캐시 라우팅을 정식 탑재하고 B200 16×16 토폴로지에서 50k tok/s, TTFT를 라운드-로빈 대비 한 자릿수 감소시켰다. Harvest는 NVLink P2P GPU 메모리를 중간 KV 계층으로 활용해 CPU 오프로딩 대비 KV 전송 지연 5.65× 단축을 달성했다.
2. **Activity B (비연속 재사용)**: ATC 2025 채택 "KVCache Cache in the Wild"가 클라우드 실측 데이터로 단일·멀티턴 재사용 패턴을 최초로 계량화해 캐시 설계의 실증 기반을 제공했다. SideQuest는 추론 모델이 에이전트 컨텍스트를 스스로 압축하는 "자기-지시형 KV 퇴거"로 피크 토큰 사용량 65% 감소를 달성하며 에이전틱 비연속 재사용 관리의 새 방향을 제시했다.
3. **Activity C (압축)**: NVIDIA의 DMS(Dynamic Memory Sparsification)가 NeurIPS 2025에서 공식 채택되고 오픈소스 KVPress 라이브러리로 배포됐다. 1K 스텝 파인튜닝으로 8× KV 캐시 압축을 달성하면서 AIME 24에서 기준 모델 대비 +12점을 기록해 "추론-시간 스케일링 + KV 압축"의 새 패러다임을 확립했다. ForesightKV는 강화학습 기반 KV 퇴거 정책으로 절반 캐시 예산에서 기존 방법을 일관 초과한다. R-KV(NeurIPS 2025)는 추론 모델에서 10% KV만으로 100% 성능 유지 + 6.6× 처리량 향상을 달성했다.
4. **A+B 시너지**: SAGA의 워크플로우 DAG 인식이 StepCache·KVShare 등 비연속 재사용 시스템의 세그먼트 히트 예측 가능성을 향상시킬 수 있다.
5. **B+C 시너지**: SideQuest의 자기-지시형 모델-주도 퇴거가 FreeKV 투기적 검색과 결합되면 에이전틱 장문 컨텍스트에서 압축-인식 비연속 KV 검색 파이프라인이 가능하다.

## Activity A — KV Cache-aware Scheduling / Orchestration

### 새로운 논문·구현체

#### 1. [SAGA: Workflow-Atomic Scheduling for AI Agent Inference on GPU Clusters](https://arxiv.org/abs/2605.00528)
- **출처**: arXiv 2605.00528 / HPDC '26 (2026-05-01 공개, 2026-07-13~16 Cleveland OH)
- **핵심 아이디어**: AI 에이전트 추론에서 워크플로우 DAG(유향 비순환 그래프) 구조를 스케줄러에 명시적으로 노출하면 온라인 KV 캐시 관리가 오프라인 최적 Bélády 정책에 근접할 수 있다는 핵심 주장을 제시한다. 워크플로우-원자적(workflow-atomic) 스케줄링으로 에이전트 단위 GPU 클러스터 스케줄링을 수행하고, 온라인 스케줄러가 달성 가능한 상한(upper bound)을 최초로 실증적으로 정량화한다. 단순 투기 실행(speculative execution)과 달리 낭비 연산 없이 알려진 워크플로우 구조를 활용한다.
- **연구 목표와의 관련성**: high
- **주목할 점**: 프로덕션 트레이스에서 1.31× 처리량 향상. 워크플로우 DAG가 관찰 가능해지면 KV 캐시 퇴거·배치 결정의 이론적 최적성 보장이 가능하다는 새로운 접근. Activity A의 "에이전틱 스케줄링 + KV 캐시 인식"을 형식적으로 정의하는 최초의 HPDC 채택 논문.

#### 2. [Pythia: Toward Predictability-Driven Agent-Native LLM Serving](https://arxiv.org/abs/2604.25899)
- **출처**: arXiv 2604.25899 (2026-04-24)
- **핵심 아이디어**: 에이전트 워크플로우의 예측 가능성(predictability)을 서빙 레이어에서 캡처하여 저프리픽스 캐시 히트율, 장문 컨텍스트 자원 경쟁, 큐잉 지연 세 가지 병목을 동시에 해소한다. 게이트웨이 프로파일러가 워크플로우 그래프와 에이전트별 출력 길이를 추론하고, 룩어헤드 요청 스케줄러가 예측 출력 길이와 워크플로우 구조로 이종 워크로드를 균형화하며, 페이즈-적응형 오토스케일러가 구조적 워크로드 변화를 선제적으로 처리한다.
- **연구 목표와의 관련성**: high
- **주목할 점**: 최대 2.9× 처리량 향상. 통합에 모델 아키텍처 변경이 불필요하며 전역 API 인터셉터만 설정하면 된다. Activity A의 "스케줄링 오버헤드 TTFT p50 +5% 이내" 목표와 직결되며 프리픽스 캐시 히트율 향상이 Activity B 비연속 재사용과 직접 시너지를 낸다.

#### 3. [llm-d 0.5: Sustaining Performance at Scale](https://llm-d.ai/blog/llm-d-v0.5-sustaining-performance-at-scale)
- **출처**: llm-d 공식 블로그 (2026-04-03 릴리즈, 2026-04-21 블로그 공개)
- **핵심 아이디어**: 계층형 KV 오프로딩(GPU HBM → CPU DRAM → 파일시스템), LoRA-인식 캐시 라우팅(프리픽스 캐시 인식 스케줄링으로 LoRA 커널 중복 실행 방지), UCCL 기반 회복탄력성 네트워킹, 스케일-투-제로 오토스케일링을 통합 탑재했다. KV 캐시 용량을 GPU 메모리에서 분리하는 네이티브 계층 아키텍처가 핵심.
- **연구 목표와의 관련성**: high
- **주목할 점**: B200 16×16 P/D 분리 토폴로지에서 50k 출력 tok/s, TTFT 라운드-로빈 대비 한 자릿수 감소 달성. LoRA-인식 KV 캐싱이 다중 어댑터 시나리오에서 Activity A의 캐시-인식 라우팅을 확장한다. vLLM v1 기반으로 Activity A 구현체의 이식 타깃이 갱신됐다.

#### 4. [Harvest: Opportunistic Peer-to-Peer GPU Caching for LLM Inference](https://arxiv.org/abs/2602.00328)
- **출처**: arXiv 2602.00328 (2026-01-30)
- **핵심 아이디어**: NVLink로 연결된 피어 GPU의 미사용 HBM을 KV 캐시와 MoE 전문가 가중치의 중간 계층으로 활용한다. 피어 GPU 메모리를 임시 캐시 계층으로 노출하는 배치 모델, 동적 가용성에 적응하는 런타임 시스템, 모델 코드 변경 없이 동작하는 세 기여를 결합한다. CPU 오프로딩의 PCIe 대역폭 병목을 NVLink 대역폭으로 우회한다.
- **연구 목표와의 관련성**: high
- **주목할 점**: KV 캐시 전송 지연 CPU 오프로딩 대비 5.65× 단축, 처리량 1.5~2.0× 향상(Qwen, Phi-3.5). Activity A의 NVLink 기반 멀티노드 KV 라우팅 설계에서 "피어 GPU 메모리를 LRU 캐시로 사용"하는 구체적 구현 경로를 제공한다. TraCT(CXL 기반, 이미 보고)의 대안으로 기존 NVLink 클러스터에서 즉시 적용 가능.

### 동향 변화 감지

| 기법 | 이전 상태 | 이번 주 변화 |
|------|---------|------------|
| Prefix-aware batching | DualMap 이중 해시로 캐시 친화성+부하 균형 동시 달성 이론화 | Pythia가 워크플로우 예측 가능성으로 프리픽스 캐시 히트율을 구조적으로 향상 |
| Cache-locality scheduling | NVIDIA Dynamo 1.0 프로덕션 OS로 격상 | SAGA HPDC '26 채택으로 워크플로우-인식 캐시 스케줄링의 이론적 최적성 상한 최초 정량화 |
| Multi-node KV routing | DualPath RDMA 스토리지-to-decode 경로로 NIC 포화 해소 | Harvest NVLink P2P GPU 캐시 계층이 PCIe/RDMA 없이 KV 전송 지연 5.65× 단축하는 인트라-클러스터 대안 제시 |
| Disaggregated prefill | TraCT CXL 기반 랙-규모 KV 전송 시연 | llm-d 0.5에서 계층형 KV 오프로딩이 프로덕션 쿠버네티스 환경에 공식 통합 |

## Activity B — Non-Contiguous KV Cache Reuse

### 새로운 논문·구현체

#### 1. [KVCache Cache in the Wild: Characterizing and Optimizing KVCache Cache at a Large Cloud Provider](https://arxiv.org/abs/2506.02634)
- **출처**: arXiv 2506.02634 / USENIX ATC 2025 (Shanghai Jiao Tong University / Alibaba Group)
- **핵심 아이디어**: 대형 클라우드 LLM 서비스 제공자의 실제 KV 캐시 워크로드 패턴을 최초로 체계적으로 특성화한다. 주요 발견: (1) 단일 턴 요청 간 KV 재사용이 멀티 턴 요청과 동등하게 중요하다. (2) 요청 카테고리별 재사용 시간·확률이 예측 가능하다. (3) 이상적 캐시 히트율을 위한 전체 캐시 크기는 moderate하다. 이 관찰을 기반으로 워크로드-인식 캐시 퇴거 정책을 제안한다.
- **연구 목표와의 관련성**: high
- **주목할 점**: LRU/FIFO 대비 캐시 히트율 23.9% 향상, 지연 42% 감소. 단일 턴 비연속 재사용이 실제로도 매우 중요하다는 산업 실증 데이터가 Activity B의 비연속 세그먼트 재사용 연구의 실용적 정당성을 강화한다. 워크로드 특성화 결과가 Activity A의 캐시-인식 스케줄러 설계의 데이터 기반을 제공한다.

#### 2. [SideQuest: Model-Driven KV Cache Management for Long-Horizon Agentic Reasoning](https://arxiv.org/abs/2602.22603)
- **출처**: arXiv 2602.22603 (2026-02-26, Cornell / Georgia Tech)
- **핵심 아이디어**: 추론 모델(LRM) 자체를 KV 캐시 압축의 주체로 활용한다. 에이전틱 장문 추론에서 외부 검색 토큰이 컨텍스트를 지배하는 문제를 해결하기 위해, KV 캐시 압축을 보조 작업(auxiliary task)으로 정의하고 메인 추론과 병렬로 실행한다. 215개 학습 샘플만으로 파인튜닝된 모델이 작업 상태에 대한 의미 이해를 활용해 구식 토큰을 자기-지시적으로 퇴거한다.
- **연구 목표와의 관련성**: high
- **주목할 점**: 피크 토큰 사용량 65% 감소, 휴리스틱 방법 대비 정확도 유지하면서 우위. Activity B의 핵심 문제인 "에이전틱 파이프라인에서 비연속 세그먼트를 선택적으로 유지"하는 문제를 "모델이 스스로 결정"하는 새로운 패러다임. FreeKV(이미 보고)의 투기적 검색과 결합하면 모델-주도 선택 + 크리티컬 패스 외 검색의 완전한 B+C 파이프라인이 가능하다.

### 동향 변화 감지

| 기법 | 이전 상태 | 이번 주 변화 |
|------|---------|------------|
| Segmented Hash / Radix | StepCache가 출력-레벨 스텝 단위 재사용으로 Activity B 적용 범위 확장 | KVCache in the Wild가 단일 턴 비연속 재사용이 실제로도 중요하다는 최초 산업 실증 데이터 제공 |
| CacheBlend / Blending / KVLink | KVShare DHD 알고리즘이 의미 기반 KV 선택 재계산으로 정확도 향상 | SideQuest 자기-지시형 모델-주도 퇴거가 에이전틱 비연속 컨텍스트 관리의 새 패러다임 제시 |
| Position-independent caching | FreeKV 투기적 검색이 KV 검색 지연을 크리티컬 패스에서 제거 | 변화 없음 — 이번 주 신규 PIC 논문 없음 |

## Activity C — KV Cache Compression

### 새로운 논문·구현체

#### 1. [Inference-Time Hyper-Scaling with KV Cache Compression (DMS)](https://arxiv.org/abs/2506.05345)
- **출처**: arXiv 2506.05345 / NeurIPS 2025 (NVIDIA / University of Edinburgh; KVPress 오픈소스 배포)
- **핵심 아이디어**: KV 캐시 압축으로 동일 메모리 예산에서 더 많은 토큰을 생성해 추론-시간 스케일링 정확도를 향상시키는 "추론-시간 하이퍼스케일링" 개념을 도입한다. Dynamic Memory Sparsification(DMS)은 "지연 퇴거(delayed eviction)" 메커니즘을 사용해 퇴거 표시된 토큰을 짧은 윈도우 동안 유지하여 필요한 정보를 컨텍스트에 합산 후 퇴거한다. 1K 훈련 스텝만 필요하며 표준 FlashAttention 커널과 완전 호환된다.
- **연구 목표와의 관련성**: high
- **주목할 점**: 8× KV 캐시 압축으로 훈련-무료 희소 어텐션 대비 우수한 정확도 유지. Qwen-R1 32B에서 AIME 24 +12.0점, GPQA +7.6점, LiveCodeBench +9.6점. Activity C의 "동일 메모리에서 2× 이상 유효 컨텍스트 지원" 목표를 8×로 대폭 초과. NeurIPS 2025 채택 + NVIDIA KVPress 라이브러리로 즉시 배포 가능. Qwen3-8B-DMS-8x HuggingFace 모델 공개.

#### 2. [R-KV: Redundancy-aware KV Cache Compression for Reasoning Models](https://arxiv.org/abs/2505.24133)
- **출처**: arXiv 2505.24133 / NeurIPS 2025 (GitHub: Zefan-Cai/R-KV)
- **핵심 아이디어**: 추론 모델(CoT)의 KV 캐시에는 반복적 자기 성찰(self-reflection) 등 고도로 중복적인 토큰이 포함됨을 관찰한다. 어텐션 기반 중요도 점수와 동적 중복성 점수를 결합해 중요도와 중복성을 동시에 균형화하는 이중 스코어 시스템으로 퇴거 토큰을 결정한다. 훈련-무료, 모델-비종속적 방법이다.
- **연구 목표와의 관련성**: high
- **주목할 점**: 10% KV만으로 전체 KV 성능의 ~100% 유지, 90% 메모리 절감, 6.6× 처리량 향상. 기존 방법들이 같은 예산에서 60% 성능에 그치는 것과 대비된다. Activity C의 "메모리 −30% 이상" 목표를 −90%로 극단적으로 초과하면서 "정확도 ±1% 이내" 조건도 만족. 추론 모델 특화 Activity C 방향으로 중요.

#### 3. [ForesightKV: Optimizing KV Cache Eviction for Reasoning Models by Learning Long-Term Contribution](https://arxiv.org/abs/2602.03203)
- **출처**: arXiv 2602.03203 / OpenReview 채택 (2026-02)
- **핵심 아이디어**: KV 캐시 퇴거를 마르코프 결정 과정(MDP)으로 정의하고 강화학습(GRPO 알고리즘)으로 퇴거 정책을 학습한다. Golden Eviction 알고리즘이 미래 어텐션 스코어를 이용해 최적 퇴거 토큰을 레이블링하고, 이를 지도학습 후 RL로 정제하는 2단계 학습 패러다임을 사용한다. 저-엔트로피 토큰에서의 언어 모델링 손실 증가를 GRPO로 완화한다.
- **연구 목표와의 관련성**: high
- **주목할 점**: AIME2024/2025 세 추론 모델에서 절반 캐시 예산 대비 기존 방법 일관 초과. Activity C의 "정확도 보존" 제약을 RL 기반으로 달성하는 첫 방법. ForesightKV의 장기 기여도 학습이 Activity B의 세그먼트 중요도 점수에 RL 기반 학습을 적용하는 영감을 제공한다.

#### 4. [ScoutAttention: Efficient KV Cache Offloading via Layer-Ahead CPU Pre-computation](https://arxiv.org/abs/2603.27138)
- **출처**: arXiv 2603.27138 (2026-03-28)
- **핵심 아이디어**: KV 캐시 오프로딩에서 CPU가 GPU보다 한 레이어 앞서 어텐션 계산을 시작하는 "레이어-선행 CPU 선계산" 알고리즘으로 GPU-CPU 파이프라인 레이턴시를 숨긴다. 인접 디코딩 토큰의 어텐션 패턴이 강한 시간적 국소성을 보인다는 관찰로, CPU 계산을 GPU 캐시에 없는 상위-k 블록에만 한정한다. GPU-CPU 협력 블록-와이즈 희소 어텐션.
- **연구 목표와의 관련성**: high
- **주목할 점**: 기준 대비 정확도 저하 2.1% 이내, 기존 오프로딩 방법 대비 처리량 2.1×, 전체 어텐션 대비 5.1× 처리량. Activity C의 "동일 메모리에서 더 긴 컨텍스트 지원"과 직결되며 GPU-CPU 경계를 투명하게 관리한다. IceCache(이미 보고)의 의미 클러스터링과 결합하면 클러스터링된 블록을 CPU에 선계산해 오프로딩 효율을 추가 향상시킬 수 있다.

#### 5. [PackKV: Reducing KV Cache Memory Footprint through LLM-Aware Lossy Compression](https://arxiv.org/abs/2512.24449)
- **출처**: arXiv 2512.24449 / IPDPS 2026 (GitHub: BoJiang03/PackKV)
- **핵심 아이디어**: 오류-제어 양자화, KV 캐시 특화 무손실 압축 체계, 캐시-상주 디컴프레션(cache-resident decompression)을 통합하는 LLM-인식 손실 압축 프레임워크. 압축 파이프라인을 시스템 수준 실행과 공동 설계(co-design)해 압축 해제 데이터를 GPU 공유 메모리·레지스터 내에서 직접 소비함으로써 전역 메모리 쓰기 되돌림을 제거한다.
- **연구 목표와의 관련성**: high
- **주목할 점**: K 캐시 153.2%, V 캐시 179.6% 높은 압축률(기존 SOTA 양자화 대비, 동등 정확도 기준). K 처리량 75.6%, V 처리량 171.6% 향상. IPDPS 2026 채택으로 시스템 커뮤니티 인정. Activity C에서 "양자화 + 시스템 공동 설계"라는 새로운 레이어를 추가하며, 기존 VQKV·TurboQuant와 구분되는 하드웨어-인식 구현 방향.

### 동향 변화 감지

## Activity 간 시너지 포인트

**A + B**
- SAGA의 워크플로우 DAG 인식이 StepCache(이미 보고)의 스텝-레벨 KV 재사용 패턴을 스케줄러에서 미리 예측하는 구조와 자연스럽게 결합된다. 워크플로우 DAG에서 특정 에이전트 노드가 이전 노드의 KV를 재사용할 것을 알면, 스케줄러가 그 노드를 KV를 보유한 GPU로 사전에 라우팅할 수 있다.
- Pythia의 룩어헤드 스케줄러가 워크플로우 출력 길이를 예측하므로, KVCache in the Wild의 "요청 카테고리별 재사용 시간·확률"을 결합하면 재사용 예상 시간 기반 선제 KV 보존 스케줄링이 가능하다.

**B + C**
- SideQuest의 자기-지시형 모델-주도 퇴거 결정과 DMS의 지연 퇴거 메커니즘을 결합하면, "모델이 결정한 퇴거 후보를 짧은 유예 기간 동안 접근 가능하게 유지"하는 완충 퇴거 파이프라인이 가능하다. 에이전틱 장문 컨텍스트에서 모델 의미 이해 + 정확도 보존의 이중 보장.
- R-KV의 중복성 스코어를 Activity B의 세그먼트 중요도 메트릭에 적용하면, 동일 내용이 반복되는 비연속 세그먼트(RAG에서 문서가 반복 등장하는 경우)를 중복성 기준으로 퇴거 우선화할 수 있다.

**A + C**
- Harvest의 NVLink P2P GPU 캐시 계층에 DMS 8× 압축 KV를 저장하면, 동일 피어 GPU HBM에 8배 더 많은 KV를 캐싱할 수 있어 Activity A의 클러스터 전체 캐시 용량을 대폭 확장할 수 있다.
- PackKV의 캐시-상주 디컴프레션이 GPU 공유 메모리에서 직접 작동하므로, llm-d 0.5의 계층형 KV 오프로딩과 결합하면 오프로딩된 KV를 GPU로 불러올 때 압축 해제 오버헤드를 숨길 수 있다.

**A + B + C**
- 통합 파이프라인 갱신: SAGA/Pythia 워크플로우-인식 스케줄링(A) → SideQuest 자기-지시형 비연속 세그먼트 선택(B) → DMS 8× KV 압축 + PackKV 캐시-상주 디컴프레션(C) → Harvest NVLink P2P GPU 계층 또는 llm-d 0.5 계층형 오프로딩(A). 워크플로우 DAG 기반 예측으로 압축된 비연속 KV를 최적 GPU 계층에 사전 배치하는 완전한 통합 경로.
- KVCache in the Wild의 실측 데이터(단일 턴 재사용 중요성)가 이 파이프라인의 실제 워크로드에서의 유효성을 뒷받침한다.

## idea-generator에게 전달할 포인트

### Activity A
- **SAGA의 교훈**: 워크플로우 DAG를 스케줄러에 노출하면 Bélády 정책에 근접 가능하다. Activity A 구현에서 에이전트 파이프라인의 DAG 구조(도구 호출 순서, 에이전트 의존 관계)를 스케줄러 메타데이터로 수집하는 인터페이스를 추가할 것.
- **Pythia의 교훈**: 워크플로우 예측 가능성을 활용한 룩어헤드 스케줄링이 프리픽스 캐시 히트율과 자원 경쟁을 동시에 해소한다. 215개 샘플로 파인튜닝된 SideQuest처럼, 소량 프로파일링으로 워크플로우 출력 길이를 예측하는 경량 모듈을 Activity A에 추가하면 스케줄링 정밀도가 향상된다.
- **llm-d 0.5**: LoRA-인식 캐시 라우팅이 Activity A의 다중 어댑터 서빙 시나리오로 적용 범위를 확장한다. vLLM v1 기반이므로 6단계 이식 타깃 검토 필요.

### Activity B
- **KVCache in the Wild**: 단일 턴 요청 간 KV 재사용이 멀티 턴과 동등하게 중요하다는 실측 데이터는 Activity B의 "비연속 세그먼트 재사용"이 실제 서비스에서도 의미 있음을 확인한다. 카테고리별 예측 가능한 재사용 패턴은 Activity A+B 통합 시 스케줄링 힌트로 활용 가능하다.
- **SideQuest의 접근**: 모델이 스스로 컨텍스트를 관리하는 자기-지시형 KV 퇴거는 Activity B의 "비연속 세그먼트 중 어느 것을 재사용하고 어느 것을 퇴거할 것인가"를 외부 휴리스틱이 아닌 모델 의미 이해로 결정하는 새로운 방향이다. 215개 샘플 파인튜닝으로 구현 가능하다는 점에서 실용적 구현 후보.

### Activity C
- **DMS(NeurIPS 2025 + KVPress 배포)**: 8× KV 압축 + 추론-시간 스케일링 정확도 향상이라는 Activity C 목표의 이중 달성. 1K 스텝 파인튜닝과 FlashAttention 호환성이 6단계 vLLM 이식 후보로 최우선화된다. KVPress 라이브러리를 통한 즉시 활용 가능.
- **R-KV(NeurIPS 2025)**: 추론 모델 전문화가 Activity C의 새 세그먼트. 10% KV로 100% 성능 달성은 "CoT 중복 토큰"이 KV 압축의 안전 여유(safety margin)를 제공한다는 것을 의미한다. Activity B의 세그먼트 중요도 점수에 중복성 차원을 추가하면 B+C 정책의 정확도 보존 margin이 확대된다.
- **PackKV(IPDPS 2026)**: 시스템-레벨 공동 설계(GPU 공유 메모리 내 디컴프레션)가 Activity C 구현에서 "압축 비용을 숨기는" 핵심 설계 원칙을 제공한다. vLLM PagedAttention과의 통합 구현 가능성 탐색 가치 있음.
