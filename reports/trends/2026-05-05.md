# 트렌드 리포트 — 2026-05-05

## 전체 요약

1. **Activity A (스케줄링)**: 에이전틱 워크플로우 전용 스케줄링이 하나의 독립 분야로 정착했다. Helium(쿼리 플랜 기반 캐시-인식 스케줄러), Continuum(멀티턴 에이전트 KV TTL 핀닝), TokenCake(DAG-기반 KV 공간·시간 스케줄러), Hexgen-Flow(Text2SQL 계층형 스케줄러)가 모두 에이전트 간 KV 의존성 및 도구 호출 중단 문제에 특화된 해법을 제시한다. 기존 "단일 요청 최적화"에서 "워크플로우 전체 KV 수명 관리"로 패러다임이 이동 중이다.
2. **Activity B (비연속 재사용)**: TokenDance가 멀티에이전트 All-Gather 패턴에서 KV 차이(diff)만 저장하는 Diff-Aware Storage로 17.5× 메모리 절감을 달성했다. LRAgent는 멀티-LoRA 아키텍처에서 기저(base) KV와 어댑터 KV를 분리 캐싱해 중복 연산을 구조적으로 제거한다. 두 논문 모두 "비연속 세그먼트의 차분(differential) 재사용"이라는 새로운 방향을 열었다.
3. **Activity C (압축)**: FireQ(INT4+FP8 혼합 커널, RoPE-인식 아웃라이어 평활화), NQKV(정규분포 특성 기반 블록-양자화), TailorKV(레이어별 오프로딩+양자화 하이브리드) 등 하드웨어-인식 INT4/FP4 압축이 주류로 올라섰다. Lookahead Q-Cache(EMNLP 2025)는 사전 생성된 pseudo query로 프리필-디코딩 단계 간 어텐션 불일치를 해소한다.
4. **A+B 시너지**: Helium의 워크플로우 쿼리 플랜과 TokenDance의 diff-aware KV 공유를 결합하면 에이전트 단위 "증분적(incremental) KV 전파"가 가능하다.
5. **B+C 시너지**: LRAgent의 LoRA-분리 KV 구조에 FireQ INT4 압축을 적용하면 어댑터 KV만 고정밀도 유지, 기저 KV는 INT4 압축하는 비대칭 계층 압축 파이프라인이 실현 가능하다.

## Activity A — KV Cache-aware Scheduling / Orchestration

### 새로운 논문·구현체

#### 1. [Helium: Efficient LLM Serving for Agentic Workflows (A Data Systems Perspective)](https://arxiv.org/abs/2603.16104)
- **출처**: arXiv 2603.16104 (2026-03-17, UC San Diego)
- **핵심 아이디어**: 에이전틱 워크플로우를 "쿼리 플랜(query plan)"으로 모델링하고 LLM 호출을 일급 연산자(first-class operator)로 취급한다. 데이터베이스 쿼리 최적화 원리를 LLM 서빙에 이식해, 프리픽스 KV 상태·워크플로우 간 중복 KV를 선제적으로 캐싱하는 캐시-인식 스케줄러(proactive caching + cache-aware scheduling)를 구현한다.
- **연구 목표와의 관련성**: high
- **주목할 점**: 최대 1.56× 속도향상. 쿼리 플랜 기반 설계가 SAGA(워크플로우 DAG, 이미 보고)의 이론적 접근을 실용 시스템으로 구체화한 직접적 구현체. Activity A의 "에이전틱 스케줄링 + KV 재사용" 목표에 대한 가장 완성된 시스템 구현 레퍼런스.

#### 2. [Continuum: Efficient and Robust Multi-Turn LLM Agent Scheduling with KV Cache Time-to-Live](https://arxiv.org/abs/2511.02230)
- **출처**: arXiv 2511.02230 (2025-11, Hugging Face Papers 주목)
- **핵심 아이디어**: 멀티턴 에이전트 워크로드에서 도구 호출(tool call) 중 발생하는 KV 캐시 퇴거 문제를 해결한다. 도구 호출 완료 후 재진입을 예측하고, KV 캐시에 TTL(Time-to-Live) 값을 설정해 GPU 메모리에 선택적으로 핀(pin)한다. TTL은 KV 재로드 비용과 순서 보존 이익을 함께 고려해 결정된다.
- **연구 목표와의 관련성**: high
- **주목할 점**: SWE-Bench 및 BFCL 실 워크로드에서 평균 작업 완료 시간 유의미하게 단축. Activity A의 "에이전틱 파이프라인 KV 수명 관리" 문제를 TTL 기반으로 형식화한 최초 시스템. Pythia(이미 보고)의 룩어헤드 스케줄러와 결합하면 TTL 예측 정밀도 향상 가능.

#### 3. [TokenCake: A KV-Cache-Centric Serving Framework for Multi-Agent Applications](https://arxiv.org/abs/2510.18586)
- **출처**: arXiv 2510.18586 (2025-10)
- **핵심 아이디어**: DAG 기반 에이전트 의존성 그래프를 파싱해 공간 스케줄러(Space Scheduler)와 시간 스케줄러(Time Scheduler)로 분리 제어한다. 공간 스케줄러는 동적 메모리 파티셔닝으로 중요 에이전트 캐시를 경쟁으로부터 보호하고, 시간 스케줄러는 도구 호출 지연 동안 GPU 메모리를 선제적 오프로딩·예측적 업로드로 재사용한다.
- **연구 목표와의 관련성**: high
- **주목할 점**: Continuum(TTL 핀닝)과 보완적 설계: TokenCake는 DAG 전체 KV 생명주기를 관리하는 반면 Continuum은 단일 턴의 재진입을 처리. Activity A의 "캐시 퇴거율 최소화" 목표에 직접 기여. SAGA의 DAG 인식과 통합 시 오프라인 최적 Bélády 수준의 온라인 근사 가능.

#### 4. [Hexgen-Flow: Optimizing LLM Inference Request Scheduling for Agentic Text-to-SQL](https://arxiv.org/abs/2505.05286)
- **출처**: arXiv 2505.05286 (2025-05, Hugging Face Daily Papers)
- **핵심 아이디어**: 멀티스테이지 Text-to-SQL 에이전틱 워크플로우를 이기종 GPU 클러스터에서 스케줄링한다. 전역 워크로드-균형 태스크 디스패치와 로컬 긴급도-기반 우선순위 결정의 계층형 스케줄링을 결합한다. 주요 스케줄링 하이퍼파라미터 튜닝을 경량 시뮬레이션 기반 탐색으로 수행한다.
- **연구 목표와의 관련성**: medium
- **주목할 점**: vLLM 대비 레이턴시 최대 1.67× 감소, 처리량 최대 1.75× 향상. 에이전틱 워크로드의 "멀티스테이지 의존성 + SLO 제약" 스케줄링 문제를 명시적으로 정형화. Text2SQL이라는 특수 도메인에 집중하지만 계층형 스케줄링 원리는 Activity A의 일반적 에이전틱 스케줄러 설계에 전용 가능.

### 동향 변화 감지

| 기법 | 이전 상태 | 이번 주 변화 |
|------|---------|------------|
| Prefix-aware batching | Pythia 룩어헤드 스케줄러로 워크플로우 예측 캐시 히트율 향상 | Helium이 쿼리-플랜 기반 연산자 추상화로 멀티-워크플로우 간 KV 공유 최적화 |

## Activity B — Non-Contiguous KV Cache Reuse

### 새로운 논문·구현체

#### 1. [TokenDance: Scaling Multi-Agent LLM Serving via Collective KV Cache Sharing](https://arxiv.org/abs/2604.03143)
- **출처**: arXiv 2604.03143 (2026-04-03)
- **핵심 아이디어**: 멀티에이전트 All-Gather 패턴에서 매 라운드마다 모든 에이전트가 동일한 공유 출력 블록을 프롬프트에 포함해 KV 캐시가 폭발적으로 중복되는 문제를 해결한다. KV Collector가 라운드 전체에 걸쳐 공유 KV를 단일 집합 단계로 재사용하고, Diff-Aware Storage가 형제 캐시(sibling caches)를 마스터 복사본에 대한 블록-희소 차분(block-sparse diff)으로 인코딩한다.
- **연구 목표와의 관련성**: high
- **주목할 점**: 2.7× 더 많은 동시 에이전트 지원, 에이전트별 KV 저장 최대 17.5× 절감, 요청별 위치-독립 캐싱 대비 1.9× 프리필 속도향상. **차분 기반 비연속 재사용**이 핵심 — 세그먼트를 독립적으로 해시하는 대신 마스터 복사본과의 차이만 저장함으로써 Activity B의 비연속 세그먼트 재사용 효율을 에이전트 집합 레벨로 확장한다.

#### 2. [LRAgent: Efficient KV Cache Sharing for Multi-LoRA LLM Agents](https://arxiv.org/abs/2602.01053)
- **출처**: arXiv 2602.01053 (2026-02, GitHub: hjeon2k/LRAgent)
- **핵심 아이디어**: 멀티-LoRA 에이전트 시스템에서 각 에이전트가 동일한 프롬프트에 대해 독립적으로 KV를 생성하는 낭비를 제거한다. KV를 공유 사전학습 기저 성분(base)과 어댑터 의존 성분(adapter-dependent)으로 분해한다. 기저 성분은 에이전트 간 공유하고, 어댑터 성분은 저랭크 형태로 저장한다. Flash-LoRA-Attention 커널이 저랭크 캐시를 전체 차원으로 구체화하지 않고 런타임에 어댑터 기여를 재구성한다.
- **연구 목표와의 관련성**: high
- **주목할 점**: Activity B의 "공통 기저 KV 공유 + 어댑터별 차분 관리"가 TokenDance의 마스터+diff 패턴과 설계 원리를 공유하면서 LoRA 아키텍처 특화. llm-d 0.5(이미 보고)의 LoRA-인식 캐시 라우팅과 직접 결합 가능. 완전 공유 대비 정확도를 유지하면서 비공유 대비 처리량·TTFT를 대폭 개선.

#### 3. [FASTLIBRA: Improving Multi-LoRA Serving Performance via Efficient LoRA and KV Cache Management](https://arxiv.org/abs/2505.03756)
- **출처**: arXiv 2505.03756 (2025-05, Hugging Face Papers 수록)
- **핵심 아이디어**: 다중 LoRA 추론 시스템에서 LoRA와 KV 캐시 간 사용 의존성(usage dependency)을 명시적으로 추적한다. 의존성-인식 캐시 매니저가 LoRA-KV 쌍을 통합 캐시 풀로 관리하고, 성능-주도 캐시 스왑퍼가 HBM 유휴 시 스왑-인, 혼잡 시 스왑-아웃을 결정하는 통합 비용 모델을 사용한다.
- **연구 목표와의 관련성**: medium
- **주목할 점**: vLLM이 46.5%의 유효하지 않은 KV 캐시(invalid KV)를 발생시키는 문제를 실증적으로 측정해 보고. Activity B의 비연속 세그먼트 재사용에서 LoRA 어댑터별 캐시 분리 관리가 필요한 실용 이유를 정량화. LRAgent(기저/어댑터 분해)와 함께 멀티-LoRA 비연속 재사용 생태계를 형성.

## Activity C — KV Cache Compression

### 새로운 논문·구현체

#### 1. [FireQ: Fast INT4-FP8 Kernel and RoPE-aware Quantization for LLM Inference Acceleration](https://arxiv.org/abs/2505.20839)
- **출처**: arXiv 2505.20839 (2025-05, GitHub: llm-fireq/fireq)
- **핵심 아이디어**: 선형 레이어 가중치와 KV 캐시를 INT4로, 활성화와 쿼리를 FP8로 양자화하는 혼합 정밀도 공동 설계 프레임워크. RoPE 인식 아웃라이어 평활화를 pre-RoPE 채널 쌍 정규화(안정 쌍 처리)와 post-RoPE 표적 스케일링(잔여 아웃라이어 처리)의 2단계로 수행한다. FlashAttention-3 커널을 수정한 3단계 프리필 파이프라이닝으로 TTFT 감소.
- **연구 목표와의 관련성**: high
- **주목할 점**: Llama2-7B FFN 레이어 1.68×, Llama3-8B 프리필 1.26× 가속(QServe 대비), 정확도 손실 무시 수준. **RoPE와 INT4의 충돌 문제를 2단계 아웃라이어 평활화로 해소**하는 것이 핵심 novelty. SAW-INT4(Hadamard 회전)의 대안적 INT4 접근으로, Hopper GPU 특화 GEMM 커널을 오픈소스로 공개하여 Activity C의 INT4 구현 후보로 직접 활용 가능.

#### 2. [NQKV: A KV Cache Quantization Scheme Based on Normal Distribution Characteristics](https://arxiv.org/abs/2505.16210)
- **출처**: arXiv 2505.16210 (2025-05)
- **핵심 아이디어**: KV 캐시 내 각 블록의 원소들이 정규분포를 따른다는 관찰에 기반해, 블록-분위수 양자화(per-block quantile quantization)를 적용한다. 정보이론적으로 최적 양자화 오류를 달성하며, 계산 데이터 타입이 아닌 Normal Float 저장 타입을 사용해 표현 공간을 효율화한다.
- **연구 목표와의 관련성**: medium
- **주목할 점**: 4비트 KV 캐시에서 OPT 모델 기준 2× 배치 크기 또는 4× 컨텍스트 길이 지원, 9.3× 처리량 향상. 정확도 저하 무시 가능(제로-샷 태스크 기준). 블록-분위수 방식이 SAW-INT4(회전 기반)·TurboQuant(PolarQuant)와 달리 추가 변환 없이 분포 적합(distribution fitting)만으로 INT4 정확도를 달성. 이론적 최적성이 뒷받침되는 경량 대안.

#### 3. [TailorKV: A Hybrid Framework for Long-Context Inference via Tailored KV Cache Optimization](https://arxiv.org/abs/2505.19586)
- **출처**: arXiv 2505.19586 / ACL Findings 2025 (GitHub: ydyhello/TailorKV)
- **핵심 아이디어**: 레이어별 특성에 따라 KV 오프로딩과 양자화를 혼합 적용한다. "전역 정보를 유지해야 하는 레이어"는 선택적 토큰 로딩(dominant token loading)으로 처리하고, "소수 토큰에 집중하는 레이어"는 모든 토큰 1~2비트 양자화로 처리한다. 두 전략이 서로 보완(complementary)하므로 레이어별로 교대 적용 가능.
- **연구 목표와의 관련성**: high
- **주목할 점**: Llama-3.1-8B 128K 컨텍스트를 단일 RTX 3090 GPU(24GB)에서 82ms/token 디코딩 속도로 처리. ACL Findings 2025 채택. Activity C의 "동일 메모리에서 2× 이상 유효 컨텍스트 지원" 목표를 단일 소비자급 GPU에서 달성하는 실증. ScoutAttention(이미 보고)의 레이어-선행 CPU 선계산과 결합하면 오프로딩 처리량 추가 향상 가능.

#### 4. [Lookahead Q-Cache: Achieving More Consistent KV Cache Eviction via Pseudo Query](https://arxiv.org/abs/2505.20334)
- **출처**: arXiv 2505.20334 / EMNLP 2025 (GitHub: noforit/Lookahead_Q-Cache)
- **핵심 아이디어**: 기존 KV 퇴거 방법들이 프리필 단계의 어텐션 스코어를 사용해 실제 디코딩 쿼리와 불일치(inconsistency)하는 문제를 해결한다. 응답 접두사를 사전에 낮은 품질로 생성한 "pseudo query"를 관측 창(observation window)으로 사용해, 실제 디코딩 단계에서의 중요 토큰을 더 정확하게 예측한다.
- **연구 목표와의 관련성**: high
- **주목할 점**: EMNLP 2025 채택으로 학계 인정. LookaheadKV(이미 보고, ICLR 2026)와 같은 "미래 엿보기" 방향이지만, 접근 방식이 다름 — LookaheadKV는 파라미터-효율 모듈로 미래 응답을 예측하는 반면, Lookahead Q-Cache는 실제 응답 접두사를 경량 생성하여 사용. 두 방법의 정밀도-비용 트레이드오프 비교가 Activity C 구현 방향 선택에 직접 유용.

#### 5. [CodeComp: Structural KV Cache Compression for Agentic Coding](https://arxiv.org/abs/2604.10235)
- **출처**: arXiv 2604.10235 (2026-04, SGLang 통합)
- **핵심 아이디어**: 에이전틱 코드 태스크(버그 탐지, 패치 생성)에서 어텐션 신호만으로는 구조적으로 중요한 토큰(호출 지점, 조건 분기, 할당)을 보존하지 못하는 문제를 해결한다. Joern이 추출한 코드 속성 그래프(Code Property Graph, CPG)를 압축 결정에 통합한다. 스팬-레벨 구조 보호(structurally critical token eviction 방지)와 구조-인식 예산 할당(chunk 중요도에 따른 압축 예산 배분)을 수행한다.
- **연구 목표와의 관련성**: medium
- **주목할 점**: SGLang에 seamless 통합. 버그 탐지·코드 생성 벤치마크에서 동일 메모리 예산에서 어텐션 전용 압축보다 일관적 우위. Activity C에 "도메인-지식(코드 구조) 기반 KV 압축"이라는 새로운 방향을 추가. Activity B의 세그먼트 경계를 코드 AST 노드 단위로 설정하면 B+C 통합에서 코드 특화 세그먼테이션 전략으로 활용 가능.

## Activity 간 시너지 포인트

**A + B**
- Helium의 쿼리-플랜 기반 캐시-인식 스케줄러와 TokenDance의 Diff-Aware Storage를 결합하면 "워크플로우 쿼리 플랜이 에이전트 간 KV 차분(diff)을 미리 계산해 전파하는" 증분적 KV 전파 시스템이 가능하다. Helium이 어떤 에이전트가 어떤 KV를 공유할지를 쿼리 플랜 수준에서 예측하고, TokenDance가 차분 저장으로 전송 비용을 최소화한다.
- TokenCake의 시간 스케줄러(도구 호출 중 선제 오프로딩)와 Continuum의 TTL 핀닝을 결합하면 "도구 호출이 짧으면 TTL 핀, 길면 선제 오프로딩"을 자동 선택하는 하이브리드 에이전틱 KV 수명 정책이 가능하다.

**B + C**
- LRAgent의 기저(base) KV와 어댑터 KV 분해 구조에 FireQ의 INT4+FP8 혼합 압축을 적용하면, 공유 기저 KV는 INT4로 고압축, 어댑터 KV는 FP8 고정밀도로 유지하는 비대칭 계층 압축 파이프라인이 실현 가능하다. 메모리 절감(-75% 이상)과 어댑터 정확도 보존을 동시에 달성.
- TokenDance의 마스터 복사본에 TailorKV의 하이브리드 오프로딩+양자화를 적용하고, diff 성분은 원본 정밀도로 유지하면 "마스터는 고압축, 차분은 고정밀도" 구조의 차분-인식 압축 저장소가 가능하다.

**A + C**
- TailorKV의 레이어별 압축 결정을 Helium의 쿼리 플랜 연산자가 인식하면, 특정 에이전트 노드가 이후에 해당 KV를 재사용할 것으로 예측될 때 압축률을 낮추는 "재사용 예측 기반 선택적 압축"이 가능하다.
- vLLM v0.9의 비동기 오프로딩 커넥터에 FireQ INT4 압축을 연결하면 오프로딩 시 자동으로 INT4 압축을 수행하는 투명한 압축-오프로딩 파이프라인이 vLLM 메인라인에서 실현 가능하다.

**A + B + C**
- 통합 파이프라인 (에이전틱 특화): Helium 쿼리-플랜 스케줄러(A) → TokenDance diff-aware 비연속 공유(B) → LRAgent 기저/어댑터 분해 + FireQ INT4 압축(C) → TokenCake 공간/시간 스케줄러(A). 워크플로우 DAG 기반으로 에이전트 간 KV 차분을 INT4로 압축·공유하고, 도구 호출 중 선제 오프로딩하는 완전한 에이전틱 통합 경로.
- FASTLIBRA의 LoRA-KV 의존성 추적이 이 파이프라인에서 어떤 KV 블록이 유효(valid)인지를 런타임에 관리하는 메타데이터 레이어로 작동할 수 있다.

## idea-generator에게 전달할 포인트

### Activity A
- **Helium의 교훈**: 데이터베이스 쿼리 플랜 최적화 원리가 LLM 워크플로우 스케줄링에 효과적으로 이식된다. Activity A 구현에서 "스케줄러가 워크플로우 DAG를 쿼리 플랜으로 파싱하고 LLM 호출을 연산자로 취급"하는 추상화 레이어를 설계할 것. Pythia(이미 보고) + SAGA(이미 보고) + Helium의 세 접근을 비교하면 Activity A의 에이전틱 스케줄러 설계 공간이 명확해진다.
- **Continuum + TokenCake 조합**: TTL 핀닝(Continuum)과 선제 오프로딩(TokenCake)의 자동 선택 정책이 Activity A의 "캐시 퇴거율 최소화" 목표와 "스케줄링 오버헤드 TTFT +5% 이내" 목표를 동시에 달성하는 실용적 경로.
- **vLLM v0.9 비동기 KV 오프로딩 커넥터**: 비동기 API가 이제 표준으로 올라왔으므로, Activity A의 6단계 이식 시 v0.9의 비동기 KV Connector 인터페이스를 활용해 스케줄링 로직을 KV 저장/로드와 비블록 방식으로 통합할 것.

### Activity B
- **TokenDance의 차분 저장 패러다임**: "공통 세그먼트는 마스터, 차이는 diff로 저장"하는 설계가 Activity B의 비연속 세그먼트 캐시에서 저장 공간을 획기적으로 줄이는 핵심 아이디어. 현재 segmented hash cache 구현에 diff-aware 레이어를 추가하면 멀티에이전트 시나리오에서 추가 메모리 절감 가능.
- **LRAgent의 기저/어댑터 분해**: 멀티-LoRA 배포 시나리오에서 Activity B의 "비연속 KV 재사용"을 어댑터별로 분리하는 구조적 접근. llm-d 0.5의 LoRA-인식 캐시 라우팅과 결합해 Activity A+B 통합 구현에서 LoRA 차원을 지원할 것.
- **FASTLIBRA의 유효 KV 추적**: vLLM의 46.5% 무효 KV 문제는 Activity B 구현에서도 동일하게 발생할 수 있으므로, 세그먼트별 유효성 메타데이터를 명시적으로 관리하는 것이 중요.

### Activity C
- **FireQ (2505.20839) vs SAW-INT4 선택**: FireQ는 RoPE-인식 2단계 아웃라이어 평활화 + Hopper GPU 특화 GEMM 커널을 오픈소스로 제공하므로 현재 하드웨어(Hopper/Blackwell)에서 즉시 적용 가능한 INT4 구현 후보. SAW-INT4는 Hadamard 회전 기반이라 커널 수정 없이 기존 GEMM에 통합 용이. 두 방법의 실제 처리량-정확도 비교 실험이 Activity C 구현 방향 결정에 필수.
- **TailorKV (ACL 2025)**: 레이어별 오프로딩+양자화 하이브리드가 Activity C의 "유효 컨텍스트 2× 이상" 목표를 단일 소비자급 GPU에서도 달성함을 증명. ScoutAttention(이미 보고)과 TailorKV를 결합하면 ScoutAttention이 CPU에서 레이어-선행 계산을 수행하고 TailorKV가 레이어별 압축 전략을 결정하는 계층적 파이프라인 구성 가능.
- **Lookahead Q-Cache (EMNLP 2025)**: 프리필 어텐션과 실제 디코딩 쿼리 간 불일치를 pseudo query 생성으로 해소하는 접근이 Activity B의 세그먼트 중요도 점수 산출에 적용 가능 — 프리필 시 pseudo query를 생성해 디코딩 단계를 미리 시뮬레이션하고, 이를 세그먼트 중요도 평가에 활용하면 B+C 통합의 퇴거 정확도 향상.
