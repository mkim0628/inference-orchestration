# 트렌드 리포트 — 2026-05-06

## 전체 요약

1. **Activity A (스케줄링)**: 이론적 최적성 보장이 실용화 단계로 전환되고 있다. Online Scheduling(MIT, 2502.07115)이 KV 캐시 메모리 제약 하에서 최초의 준-온라인 최적 다항시간 알고리즘과 완전-온라인 상수 후회(constant regret) 알고리즘을 제시하며, 기존 경험적 스케줄러(Helium, SAGA 등)의 이론적 보완재로 등장했다. ContiguousKV(2601.13631)는 오프로딩 시스템과 프루닝 알고리즘 간 세분화 불일치(granularity mismatch) 문제를 ContiguousChunk 단위로 해소해 Re-Prefill 단계를 3.85× 가속한다.

2. **Activity B (비연속 재사용)**: RAG 비연속 KV 재사용에서 "쿼리 중심(query-centric)" 접근이 새로운 주류로 자리잡고 있다. ProphetKV(2602.02579)는 사용자 쿼리와 의미 관련성을 기준으로 선택적 재계산 예산을 배분해 20% 재계산으로 96~101% 정확도 보존을 달성한다. QCFuse(2604.08585)는 의미 요약 앵커로 쿼리 표현을 강화해 기존 캐시 융합 기준선 대비 TTFT를 40% 추가 감소시킨다. InfoFlowKV(2603.05353)는 정보 흐름 관점에서 선택적 재계산을 형식화하고 청크 재순서화 전략까지 포함한다.

3. **Activity C (압축)**: TriAttention(2604.04921)이 pre-RoPE 공간의 삼각함수 시리즈로 RoPE 기반 방법의 고질적 문제(포지션 회전으로 대표 쿼리가 불안정해지는 현상)를 해소하며 AIME25 32K CoT에서 전체 어텐션과 동등한 정확도를 유지하면서 10.7× KV 감소를 달성했다. KVSculpt(2603.27819)는 KV 캐시 압축을 증류(distillation) 문제로 재정의해 L-BFGS 최적화로 연속 임베딩 공간에서 최적 KV를 찾는 방향을 개척했다.

4. **B+C 시너지**: ProphetKV의 쿼리 중심 재계산 선택과 TriAttention의 pre-RoPE 중요도 추정을 결합하면 "쿼리 관련성 + pre-RoPE 안정성 기반 이중 필터링" 세그먼트 선택이 가능하다.

5. **A+B 시너지**: ContiguousKV의 청크-단위 I/O-프루닝 정렬(alignment) 원리가 Activity B의 세그먼트 해시 캐시에서 세그먼트 경계를 I/O 블록 경계와 일치시키는 시스템 설계 원칙으로 직접 응용 가능하다.

## Activity A — KV Cache-aware Scheduling / Orchestration

### 새로운 논문·구현체

#### 1. [Online Scheduling for LLM Inference with KV Cache Constraints](https://arxiv.org/abs/2502.07115)
- **출처**: arXiv 2502.07115 (2025-02, v3 업데이트 2026-04, MIT — Patrick Jaillet, Jiashuo Jiang, Konstantina Mellou 외)
- **핵심 아이디어**: 응답 길이를 사전에 알 수 없는 온라인 환경에서 KV 캐시 메모리 예산 제약 하에 평균 레이턴시를 최소화하는 첫 번째 이론 기반 일괄처리(batching) 및 스케줄링 알고리즘을 제안한다. 준-온라인(semi-online) 프롬프트 도착 모델에서 평균 레이턴시에 대해 다항시간 내 정확히 최적인 알고리즘을 도출하고, 완전 온라인(fully online) 확률적 도착 모델에서는 상수 후회(constant regret)를 보장하는 효율적 온라인 알고리즘을 제시한다.
- **연구 목표와의 관련성**: high
- **주목할 점**: 기존 GSA(2601.22996, 이미 보고)가 메모리 예산 위반 없는 상수 경쟁비 보장에 집중한 반면, 이 논문은 평균 레이턴시 최적화와 KV 메모리 예산을 동시에 형식화한다. Activity A의 "스케줄링 오버헤드 TTFT p50 +5% 이내" 목표를 수학적으로 뒷받침하는 이론 기반 스케줄러다. 실측 요청 트레이스에서도 강건한 성능을 실험으로 확인했다.

#### 2. [ContiguousKV: Accelerating LLM Prefill with Granularity-Aligned KV Cache Management](https://arxiv.org/abs/2601.13631)
- **출처**: arXiv 2601.13631 (2026-01-20, Jing Zou 외 — 홍콩 City University)
- **핵심 아이디어**: KV 캐시 오프로딩 시스템과 의미-인식 프루닝 알고리즘 사이에 세분화 불일치(granularity mismatch) 문제가 있음을 지적한다. 프루닝 알고리즘은 중요한 토큰을 세밀하게 선택하는 반면, 시스템 I/O는 고정 크기 블록 단위로 동작해 심각한 읽기 증폭(read amplification)이 발생한다. ContiguousChunk(16토큰 단위)라는 통합 데이터 관리 세분성을 도입해 KV 캐시 프루닝과 I/O 연산을 정렬한다. 인접 레이어 간 중요 청크 인덱스의 유사성을 이용한 레이어 내/간 비동기 프리패칭으로 I/O와 컴퓨팅 간 순차 의존성을 제거한다.
- **연구 목표와의 관련성**: high
- **주목할 점**: Qwen2.5 시리즈에서 SOTA 오프로딩 시스템(IMPRESS) 대비 Re-Prefill 단계 3.85× 가속. Activity B의 세그먼트 해시 캐시 구현에서 "세그먼트 크기 = I/O 블록 크기"로 정렬하면 읽기 증폭 없는 비연속 세그먼트 로드가 가능함을 암시한다. Activity A의 KV 오프로딩 스케줄러가 ContiguousChunk 단위를 인식하면 오프로딩 중 I/O-컴퓨팅 오버랩을 추가로 달성할 수 있다.

### 동향 변화 감지

|------|---------|------------|
| Prefix-aware batching | Pythia 룩어헤드 + Helium 쿼리-플랜 기반 스케줄러로 에이전틱 워크플로우 KV 히트율 향상 | 변화 없음 — 이번 주 신규 prefix-aware batching 논문 없음 |
| Cache-locality scheduling | SAGA HPDC '26, GSA 상수 경쟁비로 이론적 최적성 상한 정량화 | Online Scheduling(2502.07115)이 준-온라인/완전-온라인 모델 모두에서 평균 레이턴시 최적화와 KV 메모리 예산 제약을 동시 형식화 |
| Multi-node KV routing | DualPath RDMA 우회, Harvest NVLink P2P GPU 계층 | 변화 없음 — 이번 주 신규 멀티노드 라우팅 논문 없음 |

## Activity B — Non-Contiguous KV Cache Reuse

### 새로운 논문·구현체

#### 1. [ProphetKV: User-Query-Driven Selective Recomputation for Efficient KV Cache Reuse in Retrieval-Augmented Generation](https://arxiv.org/abs/2602.02579)
- **출처**: arXiv 2602.02579 (2026-02-05)
- **핵심 아이디어**: RAG 시나리오에서 기존 토큰 선택 기준(전역 중요도 기반)이 "쿼리 비관련 고현저성(salient-but-irrelevant) 토큰"으로 제한된 재계산 예산을 낭비하는 "밀어내기 효과(crowding-out effect)"를 발견한다. 사용자 쿼리와의 의미 관련성을 기준으로 재계산 대상 토큰을 우선화하고, 레이어별 어텐션 메트릭을 융합하는 이중 단계 재계산 파이프라인(dual-stage recomputation pipeline)으로 쿼리-타깃 어텐션 복원을 집중 수행한다.
- **연구 목표와의 관련성**: high
- **주목할 점**: 20% 재계산만으로 전체 프리필 정확도의 96~101% 유지. RULER +8.8~24.9%, LongBench +18.6~50.9% 향상 (CacheBlend, EPIC, KVShare 대비). Activity B의 "비연속 세그먼트 중 어느 것을 재계산하고 어느 것을 재사용할 것인가"를 사용자 쿼리 기반으로 결정하는 직접적 구현 참조. "쿼리-타깃 어텐션 복원"이라는 새로운 비연속 재사용 패러다임을 확립.

#### 2. [QCFuse: Query-Centric Cache Fusion for Efficient RAG Inference](https://arxiv.org/abs/2604.08585)
- **출처**: arXiv 2604.08585 (2026-03-30, Jianxin Yan 외)
- **핵심 아이디어**: CacheBlend/EPIC 계열의 캐시 융합(cache fusion) 방법이 로컬 관점의 토큰 선택에 의존해 사용자 쿼리의 전역 맥락을 활용하지 못하는 문제를 해결한다. 의미 요약 앵커(semantic summary anchors)로 쿼리 표현을 강화하고, 가장 중요한 트랜스포머 레이어의 어텐션 분포를 기준으로 쿼리 관련 토큰만 선택 재계산한다. 고비용 컨텍스트-인식 쿼리 표현 생성과 파이프라인 제약을 동시에 극복한다.
- **연구 목표와의 관련성**: high
- **주목할 점**: 다중 LLM 및 멀티-홉 QA 벤치마크에서 전체 재계산 대비 최대 2× TTFT 가속, 기존 캐시 융합 기준선 대비 40% 추가 레이턴시 감소. ProphetKV(쿼리 중심 재계산)와 설계 철학을 공유하면서 융합 파이프라인 제약 내에서 효율성을 극대화. Activity B의 CacheBlend 계열 발전 방향으로, "전역 쿼리 맥락 + 선택적 재계산" 조합이 이번 주 B 분야의 핵심 흐름.

#### 3. [InfoFlow KV: Information-Flow-Aware KV Recomputation for Long Context](https://arxiv.org/abs/2603.05353)
- **출처**: arXiv 2603.05353 (2026-03-05, Xin Teng 외)
- **핵심 아이디어**: 선택적 KV 재계산을 정보 흐름(information flow) 문제로 재해석한다. 어텐션-정규(attention-norm) 신호를 쿼리로부터 계산해 "의미적으로 관련되고 정보 전파에 구조적으로 유리한 위치에 있는" 토큰을 신뢰성 있게 식별한다. 이 신호는 추론-일관 RoPE 기하학 하에서 계산되어 위치 편향을 제거한다. 또한 검색된 청크의 전역 위치 할당을 재구성하고 정보 흐름 기반 청크 재순서화(chunk reordering) 전략을 도입한다.
- **연구 목표와의 관련성**: high
- **주목할 점**: RAG 장문 QA에서 기존 방법들이 휴리스틱 또는 표현 불일치에 의존하는 문제를 정보 흐름 이론으로 해소. 청크 재순서화가 Activity B의 비연속 세그먼트를 단순히 "어디서 가져올 것인가"를 넘어 "어떤 순서로 배치할 것인가"까지 포괄하는 새로운 차원을 추가. ProphetKV·QCFuse와 함께 "쿼리 중심 비연속 재사용 3부작"을 형성.

## Activity C — KV Cache Compression

### 새로운 논문·구현체

#### 1. [TriAttention: Efficient Long Reasoning with Trigonometric KV Compression](https://arxiv.org/abs/2604.04921)
- **출처**: arXiv 2604.04921 (2026-04, GitHub: WeianMao/triattention; vLLM·SGLang 통합 포함)
- **핵심 아이디어**: 기존 KV 중요도 추정이 post-RoPE 쿼리를 사용해 토큰 위치에 따라 쿼리 벡터가 회전하므로 대표 쿼리 수가 제한돼 불안정한 KV 선택이 발생하는 근본 문제를 해결한다. pre-RoPE 공간에서 Q와 K 벡터가 고정된 비제로 중심 주변에 집중됨을 관찰하고, 이 집중이 모델이 특정 거리 범위의 키를 우선 어텐션하도록 유도함을 보인다. 중심 벡터가 결정하는 선호 거리를 삼각함수 시리즈로 표현해 key 중요도를 위치-안정적으로 추정한다. 128 생성 토큰마다 프루닝을 트리거하는 윈도우 기반 방식.
- **연구 목표와의 관련성**: high
- **주목할 점**: AIME25 32K CoT에서 전체 어텐션과 동등한 추론 정확도를 유지하면서 2.5× 처리량 향상 또는 10.7× KV 메모리 감소 달성. 기존 기준선들이 같은 효율 예산에서 약 50% 정확도에 그치는 것과 대비. vLLM·SGLang 통합 구현 오픈소스 제공으로 6단계 vLLM 이식 후보. Activity C의 "정확도 ±1% 이내 + 메모리 감소" 목표를 추론 모델에서 달성하는 새로운 기준선.

#### 2. [KVSculpt: KV Cache Compression as Distillation](https://arxiv.org/abs/2603.27819)
- **출처**: arXiv 2603.27819 (2026-03-29, Bo Jiang & Sian Jin — Temple University)
- **핵심 아이디어**: 기존 KV 압축 방법들이 원본 KV 쌍을 선택하거나 결합하는 반면, KVSculpt는 연속 임베딩 공간에서 **제약 없는 최적 KV 쌍**을 직접 최적화한다. 각 레이어의 어텐션 동작을 보존하도록 Key를 L-BFGS로 최적화하고 Value를 최소자승으로 닫힌 형태로 풀며, 두 단계를 교대 반복한다. 적응 예산 할당(adaptive budget allocation)이 저렴한 파일럿 압축 실행으로 레이어·헤드별 압축 난이도를 추정해 예산을 재분배한다.
- **연구 목표와의 관련성**: high
- **주목할 점**: Qwen2.5-1.5B-Instruct 2048토큰 컨텍스트에서 KL 발산을 어텐션 스코어 퇴거+최소자승 대비 3.5~4.1× 감소. 적응 할당이 추론 비용 없이 1.3× KL 추가 감소. "단일 원본 토큰이 표현할 수 없는 위치"를 찾아 여러 원본 키를 요약하는 증류 키(distilled key)라는 개념이 Activity B의 비연속 세그먼트 표현에 응용 가능 — 다수의 비연속 세그먼트를 소수의 증류 KV로 요약하면 스토리지 효율과 정확도를 동시에 달성.

#### 3. [KV-CoRE: Benchmarking Data-Dependent Low-Rank Compressibility of KV-Caches in LLMs](https://arxiv.org/abs/2602.05929)
- **출처**: arXiv 2602.05929 (2026-02-07, OpenReview 게재; Hugging Face Papers 수록)
- **핵심 아이디어**: SVD 기반 방법으로 KV 캐시의 데이터 의존적 저랭크 압축 가능성(low-rank compressibility)을 정량화하는 벤치마크 프레임워크. 경사도-없는(gradient-free) 증분(incremental) 방식으로 모델, 데이터셋(5개 영어 도메인, 16개 언어), 레이어별로 체계적 분석을 수행한다. 주요 발견: (1) Key가 Value보다 일관되게 압축 가능성이 높다. (2) 교차-언어 변동이 교차-도메인 변동보다 크다. (3) 훈련 데이터 다양성과 언어 커버리지가 압축 가능성에 아키텍처보다 큰 영향.
- **연구 목표와의 관련성**: medium
- **주목할 점**: 기존 저랭크 압축 연구(ReCalKV, TurboQuant 등)가 실험 설계에서 임의 선택한 압축 비율이 데이터/언어에 따라 얼마나 실효성이 다른지를 최초로 체계적으로 정량화. Activity C의 압축 기법 선택 및 하이퍼파라미터 설정에서 "언어별 최적 압축 비율"이 다름을 고려한 적응형 설계 필요성을 실증.

#### 4. [HybridKV: Hybrid KV Cache Compression for Efficient Multimodal Large Language Model Inference](https://arxiv.org/abs/2604.05887)
- **출처**: arXiv 2604.05887 (2026-04, RichlyAI)
- **핵심 아이디어**: 멀티모달 LLM에서 각 시각 입력이 수천 토큰으로 팽창해 KV 캐시가 급증하는 문제를 해결한다. 어텐션 헤드를 텍스트-중심 어텐션(text-centric attention)을 이용해 정적(static) 헤드와 동적(dynamic) 헤드로 분류하고, 각 유형에 맞춤형 압축 방법을 적용한다. 정적 헤드는 텍스트-사전 프루닝(text-prior pruning), 동적 헤드는 청크-단위 검색(chunk-wise retrieval)을 사용하며, 계층적 예산 할당(hierarchical budget allocation)으로 K/V 쌍에 독립 예산을 설정한다.
- **연구 목표와의 관련성**: medium
- **주목할 점**: KV 캐시 메모리 최대 7.9× 감소, 디코딩 1.52× 가속, 전체 캐시 MLLM과 동등하거나 우수한 성능. 멀티모달 시나리오에서 비연속 세그먼트(시각 토큰 청크)에 유형별 차별 압축을 적용하는 B+C 통합 방향. HybridKV의 청크-단위 검색 방식이 Activity B의 세그먼트 해시 캐시 구조와 직접 통합 가능.

**B + C**

**A + C**

**A + B + C**
- 통합 파이프라인 (RAG 특화): Online Scheduling(A) + ContiguousKV Re-Prefill 가속(A) → ProphetKV 쿼리 중심 선택적 재계산(B) + InfoFlowKV 청크 재순서화(B) → TriAttention pre-RoPE 압축 + KVSculpt 증류 KV 저장(C). RAG 요청에서 쿼리가 도착하면 스케줄러가 메모리 예산 내 최적 배치를 결정하고, ProphetKV가 20% 재계산 예산을 쿼리 관련 토큰에 집중하며, KVSculpt가 나머지 캐시된 세그먼트를 증류 KV로 압축 저장하는 완전한 통합 경로.
