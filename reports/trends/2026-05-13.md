# 트렌드 리포트 — 2026-05-13

## 전체 요약

1. **Activity A (스케줄링)**: 에이전틱 워크플로우에 특화된 예측 기반 KV 캐시 관리가 새로운 실용 방향으로 확립됐다. PBKV(2605.06472)는 미래 에이전트 호출을 예측해 고재사용 KV를 GPU에 선점 유지하고, 동적 워크플로우에서 평균 레이턴시를 최대 1.39× 단축한다. SGLang Q2 2026 로드맵에서 "에이전틱 워크로드용 분산 KVCache 시스템" 항목이 공식 등재되며 에이전트-KV 인프라가 프레임워크 레벨 우선순위로 격상됐다.

2. **Activity B (비연속 재사용)**: KV 캐시를 누산기(accumulator)로 삼는 함수형 청크 반복 패러다임 두 편이 등장했다. KV-Fold(2605.12471)는 훈련·아키텍처 수정 없이 프리트레인드 트랜스포머를 재귀 장문 모델로 전환하며, 128K 컨텍스트 NIAH에서 100% 정확 검색을 단일 40GB GPU에서 달성한다. Key-Value Means/KVM(2605.09877)은 블록-재귀 압축 메모리로 증가형 또는 고정-크기 KV 상태를 지원하며, O(N) 청크-병렬 훈련을 가능하게 한다.

3. **Activity C (압축)**: RL 후훈련 환경에서 KV 압축이 야기하는 정책 편향 문제를 최초로 체계적으로 해결하는 Shadow Mask Distillation(2605.06850)이 등장했다. 1B~7B 모델에서 압축 Dense 베이스라인과 동등하거나 초과하는 RL 학습 성능을 달성한다. 하드웨어-특화 방향에서는 When Quantization Is Free(2605.05699)가 단일 퓨즈드 Metal 커널로 Apple Silicon에서 INT4 KV 캐시가 FP16보다 빠른 조건을 실증한다.

4. **A+B 시너지**: PBKV의 예측 기반 에이전트 호출 예측(A)과 KV-Fold의 청크 누산 방식(B)이 결합되면, "미래 에이전트가 필요로 할 KV 청크를 사전에 접힌(folded) 상태로 누산 유지"하는 에이전틱 청크-재귀 캐시 관리가 가능하다.

5. **C 독자 흐름**: Shadow Mask Distillation은 Activity C의 압축-정확도 연구가 "추론 시 정확도"에서 "학습 루프 안정성"으로 확장되는 새로운 연구 전선을 열었다. RL 기반 정렬에서 KV 압축을 기본 기능으로 통합하는 방향이 2026년 하반기 중요 과제로 부상했다.

## Activity A — KV Cache-aware Scheduling / Orchestration

### 새로운 논문·구현체

#### 1. [Efficient Serving for Dynamic Agent Workflows with Prediction-based KV-Cache Management (PBKV)](https://arxiv.org/abs/2605.06472)
- **출처**: arXiv 2605.06472 (2026-05-07, Wuhan University / Dameng Database / Shanghai Jiao Tong University / Macquarie University / HKUST)
- **핵심 아이디어**: LLM 기반 워크플로우에서 에이전트들이 방대한 컨텍스트를 공유해 KV 캐시 재사용 기회가 높지만, 기존 에이전트-레벨 캐시 관리는 워크플로우 내 재사용 기회를 놓치고, 워크플로우-레벨 캐시 관리는 정적 에이전트 시퀀스를 가정해 실제 동적 워크플로우에 적합하지 않다는 문제를 해결한다. PBKV는 세 구성 요소로 동적 워크플로우를 처리한다: (1) **예측기(Predictor)** — 과거 워크플로우 이력과 현재 워크플로우 컨텍스트를 결합해 미래 수 스텝의 에이전트 호출을 다단계 예측, (2) **KV 캐시 관리 정책** — 예측 기반 재사용 잠재력 추정으로 고잠재력 캐시 엔트리를 GPU 메모리에 보존하고 저잠재력 엔트리를 퇴거, (3) **2계층 KV 스토리지** — GPU 메모리 상 Radix 트리와 호스트 메모리 상 HiCache로 구성. 위계적 퇴거(hierarchical eviction)와 보수적 프리페치(conservative prefetching) 두 원칙이 고가치 캐시 보존과 저가치 캐시 회수를 균형 있게 조정한다. 예측기가 부정확할 때도 성능이 Lipschitz-연속적으로 저하되어 안정적이다.
- **연구 목표와의 관련성**: high
- **주목할 점**: 현실적인 동적 워크플로우에서 평균 워크플로우 레이턴시를 최대 1.39× 단축. 예측기의 성능이 오차에 연속적으로 의존(Lipschitz-연속)해 예측 실패 시 급격한 성능 저하 없음. Activity A의 "에이전틱 워크로드 캐시-인식 스케줄링" 방향에서 SAGA(이미 보고)의 워크플로우-원자적 스케줄링이 그래프 기반 단계 추정을 제공했다면, PBKV는 미래 호출 예측을 기반으로 캐시 관리 정책을 직접 구동하는 실용적 통합을 제공한다. Helium(이미 보고)의 쿼리-플랜 스케줄러가 사전 정의된 구조를 활용했다면 PBKV는 완전 동적 구조에서 학습된 예측으로 작동하는 상보적 접근. Continuum(이미 보고)의 TTL 기반 KV 핀닝과 결합 시 "예측 기반 선제적 보존 + TTL 기반 반응적 해제"의 이중 계층 에이전트 KV 관리가 가능하다.

|------|---------|------------|

## Activity B — Non-Contiguous KV Cache Reuse

### 새로운 논문·구현체

#### 1. [KV-Fold: One-Step KV-Cache Recurrence for Long-Context Inference](https://arxiv.org/abs/2605.12471)
- **출처**: arXiv 2605.12471 (2026-05-12, Alireza Nadali, Patrick Cooper, Ashutosh Trivedi, Alvaro Velasquez)
- **핵심 아이디어**: 프리트레인드 트랜스포머를 아키텍처 수정·파인튜닝·특수 메모리 토큰 없이 재귀적 장문 컨텍스트 모델로 전환하는 훈련-프리 프로토콜. KV 캐시를 함수형 프로그래밍의 `foldl` 연산의 누산기(accumulator)로 취급한다: 각 스텝에서 모델이 다음 청크를 누산된 KV 캐시를 접두사로 참조하며 처리하고, 새로 생성된 K·V를 캐시에 추가한 후 확장된 캐시를 다음 스텝으로 전달한다. 이 단일 업데이트 규칙을 반복해 임의 깊이의 체인을 구성한다. 핵심 발견은 이 재귀가 안정적(stable)이라는 점이다 — 스텝당 드리프트가 초기에 상승했다가 평탄한 플래토(plateau)에 수렴하며, 플래토는 수치 정밀도의 10,000배 변화에도, 청크 크기에도, 모델 패밀리에도 무감하다.
- **연구 목표와의 관련성**: high
- **주목할 점**: NIAH(Needle-in-a-Haystack) 벤치마크에서 16K~128K 컨텍스트, 체인 깊이 최대 511에 걸친 152개 시험 전체에서 100% 정확 검색을 단일 40GB GPU(Llama-3.1-8B)로 달성. 스트리밍 방법(StreamingLLM 등)이 메모리 경계를 위해 충실도를 희생하는 반면, KV-Fold는 장거리 정보를 완전 보존하면서 단일 GPU 메모리 한계 내에서 동작. Activity B의 핵심 문제인 "비연속 청크의 순차적 누산 후 참조" — 즉 이전 청크들의 KV를 접두사로 유지하면서 새 청크를 처리하는 형태 — 를 가장 단순한 형태로 구현한다. 기존 PagedAttention / RadixAttention이 해시 기반 세그먼트 재사용을 추구했다면, KV-Fold는 순서 의존적 누산 재사용이라는 상보적 방향을 제시한다. 훈련-프리 특성이 Activity B 구현에서 src/cache/ 레이어에 즉시 통합 가능한 실용 경로.

#### 2. [Key-Value Means: Transformers with Expandable Block-Recurrent Compressed Memory (KVM)](https://arxiv.org/abs/2605.09877)
- **출처**: arXiv 2605.09877 (2026-05-11, Daniel Goldstein, Eugene Cheah — Recursal AI / EleutherAI)
- **핵심 아이디어**: Ouro 등 루프드(looped) 트랜스포머 아키텍처가 반복 추론을 활성화 공간에서 중간 토큰 생성 없이 수행할 수 있지만, 루프 반복마다 표준 KV 캐시를 누적해 메모리가 추론 깊이에 비례해 선형 증가하는 문제를 해결한다. KVM은 새로운 블록-재귀 어텐션으로, 레이어당 학습 가능한 게이팅 메커니즘으로 시간에 따라 갱신되는 단일 공유 KV 캐시를 유지한다. 고정-크기 KVM(bounded memory)과 증가형 KVM(expandable memory, 서브이차 증가) 두 변형을 모두 지원하며, 표준 연산만으로 커스텀 CUDA 커널 없이 구현 가능하고 청크-병렬 훈련 및 프리필을 지원한다.
- **연구 목표와의 관련성**: high
- **주목할 점**: 고정-크기 KVM이 비슷한 크기의 표준 트랜스포머 대비 추론 벤치마크에서 경쟁력 있는 성능을 유지하면서 메모리 사용량을 반복 루프 깊이와 무관하게 고정. 증가형 KVM은 서브이차 상태 증가로 장문 컨텍스트 테스트에서 경쟁력 있는 성능 달성. Activity B의 "비연속 KV 세그먼트를 압축 형태로 유지"하는 방향에서 MELT(이번 주 Activity C에 등재)와 쌍을 이루는 아키텍처 수준 접근. KVM의 게이팅 메커니즘이 비연속 세그먼트를 선택적으로 압축 상태에 통합하는 "학습 가능한 비연속 세그먼트 합산기"로 해석 가능하다. 기존 RetentiveKV(이미 보고)의 SSM 기반 연속 기억이 GPU KV 압축에 집중한 반면, KVM은 아키텍처 수준에서 압축 메모리 상태를 학습하는 구조적 차이.

### 동향 변화 감지

| 기법 | 이전 상태 | 이번 주 변화 |
|------|---------|------------|
| Segmented Hash / Radix | Louver(2605.06763) 반공간 범위 탐색으로 제로 거짓 음성 KV 검색 최초 이론 보장 | 변화 없음 — 이번 주 신규 Radix/Segmented Hash 논문 없음 |
| CacheBlend / Blending | AdapShot(2605.03644) RoPE 위치 재인코딩으로 비연속 KV 세그먼트 무손실 재순서화 | KV-Fold(2605.12471)가 KV 캐시를 foldl 누산기로 취급하는 훈련-프리 청크 재귀로 128K 컨텍스트 100% NIAH 달성 — "블렌딩"에서 "누산적 재귀"로 비연속 재사용의 새 패러다임 추가 |
| Position-independent caching | AdapShot RoPE 회전 기반 위치 재인코딩이 비연속 세그먼트 위치 오류 무손실 해소 | KVM(2605.09877)이 학습 가능한 게이팅으로 압축 블록-재귀 KV 상태를 유지하는 아키텍처 수준 위치-독립 비연속 메모리 추가; 훈련-프리(KV-Fold)와 훈련-기반(KVM) 두 경로가 동시 확립 |
| Chunked-prefix reuse | KV-Fold 이전: AdapShot(RoPE 재인코딩), WiCER(도메인 지식 KV 증류) | KV-Fold가 완전히 새로운 패러다임 — 청크를 순차적 foldl로 처리하는 훈련-프리 장문 재귀 프로토콜. 동일 GPU 메모리 예산(40GB)에서 StreamingLLM의 유한 슬라이딩 윈도우 대비 장거리 정보를 완전 보존 |

## Activity C — KV Cache Compression

### 새로운 논문·구현체

#### 1. [How to Compress KV Cache in RL Post-Training? Shadow Mask Distillation for Memory-Efficient Alignment](https://arxiv.org/abs/2605.06850)
- **출처**: arXiv 2605.06850 (2026-05-07)
- **핵심 아이디어**: RL 후훈련(post-training)의 롤아웃(rollout) 단계에서 장문 추론 태스크의 KV 캐시가 과도한 메모리를 요구하는 "메모리 벽(memory wall)" 문제를 해결한다. KV 압축 적용으로 메모리를 줄이면 샘플러(압축 컨텍스트 사용)와 학습기(전체 컨텍스트 사용)가 서로 다른 분포에서 행동해 "오프-폴리시 편향(off-policy bias)"이 발생하며, 이는 표준 추론에서 근-무손실인 KV 압축도 RL 최적화의 불안정성에 의해 급격히 증폭된다. Shadow Mask Distillation(SMD)는 이중 트랙 프레임워크로 이 편향을 구조적으로 제거한다: (1) **Shadow mask** — 압축 KV로 롤아웃을 생성하되, 동일 패스에서 "그림자" 마스크로 전체 KV 연산도 병렬 실행해 정책 그래디언트를 완전 KV 기반 로그확률로 보정, (2) **Distillation** — 압축 모델이 완전 모델의 소프트 타겟을 모방하도록 증류. 이 설계가 완벽한 그래디언트 정렬(perfect gradient alignment)과 제로 데이터 낭비를 보장한다. 고수준 텐서 슬라이싱의 "Not-In-Place Allocation Spike"(OOM 위험)를 우회하는 프레임워크-독립적 구현을 포함한다.
- **연구 목표와의 관련성**: high
- **주목할 점**: 1B~7B 모델, MATH500·AIME24·HotpotQA·TL;DR 등 다양한 태스크에서 거의 모든 설정에서 SOTA 달성 — HotpotQA·TL;DR에서는 비압축 Dense 베이스라인을 초과(적절한 KV 압축이 정책이 단기적 RL 보상에 과적합하는 것을 방지하는 정보 병목 효과 작동). Activity C의 압축-정확도 연구가 "추론 시 정확도 유지"에서 "학습 루프 안정성 보장"으로 새로운 평가 차원이 추가됐음을 최초로 지적하는 중요 논문. LaProx(이미 보고)·LKV(이미 보고)의 추론-시 퇴거 연구와 달리, SMD는 RL 최적화 루프 내부에서 KV 압축을 운영하는 최초의 체계적 접근. vLLM/SGLang 기반 RL 학습 파이프라인(GRPO, PPO 등)에 통합 시 장문 추론 모델 훈련의 배치 크기 제약을 완화하는 직접적 응용.

#### 2. [When Quantization Is Free: An int4 KV Cache That Outruns fp16 on Apple Silicon](https://arxiv.org/abs/2605.05699)
- **출처**: arXiv 2605.05699 (2026-05-07, Mohamed Amine Bergach)
- **핵심 아이디어**: Apple Silicon(Metal 백엔드) 환경에서 INT4 KV 캐시 양자화가 메모리 절감뿐 아니라 실제 추론 속도도 FP16보다 빠를 수 있음을 최초로 실증한다. 단일 퓨즈드 Metal 커널에 네 가지 연산을 결합한다: (1) **부호-무작위화 FFT(SRFT)** — 아웃라이어 채널을 Gaussianize하여 양자화 오류 최소화, (2) **채널별 스케일링(λ)**, (3) **그룹별 abs-max 양자화**, (4) **INT4 nibble 패킹**. 커널 오버헤드가 약 25 ns/vec으로 3× 대역폭 절감 아래 유지된다. HuggingFace Cache 서브클래스로 노출되어 기존 코드베이스에 바로 통합 가능하다.
- **연구 목표와의 관련성**: high
- **주목할 점**: Gemma-3 1B에서 256~4096 토큰 컨텍스트 전 구간에서 INT4가 FP16보다 빠름(−3~−8 ms/tok). Qwen2.5-1.5B는 단문 컨텍스트(1K 이하)에서 빠름. 3× 영속 메모리 압축 + 품질 유지. SRFT와 SRHT가 통계적으로 동등한 Gaussianization 효과를 제공함을 입증해, 향후 하드웨어에서 둘 중 더 빠른 변환을 선택하는 실용적 가이드라인 제공. Activity C의 "정확도 ±1% 이내" 제약과 "메모리 −30% 이상" 목표를 에지(Apple Silicon) 환경에서 속도 향상까지 동반해 달성하는 최초의 하드웨어-특화 구현. RateQuant(이미 보고)의 헤드별 최적 비트 할당과 결합 시 헤드별로 INT4/FP8를 RateQuant가 결정하고 SRFT 커널이 최적 비트폭으로 양자화하는 "최적-비트 퓨즈드 Metal 커널"이 에지 배포에서 가능하다. vLLM의 FP8 KV 캐시(이미 보고)가 서버 GPU에서 FP8의 속도 우위를 실증했다면, 이 논문은 에지 Apple Silicon에서 INT4의 속도 우위를 실증한다.

#### 3. [Memory-Efficient Looped Transformer: Decoupling Compute from Memory in Looped Language Models (MELT)](https://arxiv.org/abs/2605.07721)
- **출처**: arXiv 2605.07721 (2026-05-08, Victor Conchello Vendrell 외 — Qualcomm AI Research)
- **핵심 아이디어**: Ouro 등 루프드(looped) 트랜스포머가 표준 KV 캐시를 반복 루프마다 누적해 메모리가 추론 깊이(루프 수)에 선형 비례해 증가하는 문제를 해결한다. MELT는 레이어당 단일 KV 캐시를 공유하되, 학습 가능한 게이팅 메커니즘으로 루프에 걸쳐 시간적으로 갱신한다. 이 설계가 표준 어텐션을 완전히 보존하면서 루프 깊이가 증가해도 메모리 사용량을 고정시킨다. 계산(추론 깊이)과 메모리(KV 캐시 크기)를 분리함으로써 추론 깊이를 제한 없이 확장할 수 있게 된다.
- **연구 목표와의 관련성**: high
- **주목할 점**: 추론 벤치마크에서 비슷한 크기의 표준 트랜스포머를 초과하는 성능. 출발점 LoopLM의 성능을 유지하면서 루프 누적 KV 캐시 대비 메모리를 "극적으로" 절감. 계산 깊이(루프 수) 확장 시 메모리가 증가하지 않는 첫 루프드 트랜스포머 설계. Activity C의 "동일 메모리 예산에서 2× 이상 유효 컨텍스트" 목표를 아키텍처 수준에서 달성하는 새로운 방향 — 토큰 퇴거나 양자화가 아닌 루프 상태 공유 자체로 메모리를 절감. RetentiveKV(이미 보고)의 SSM 기반 연속 기억 진화와 MELT의 게이팅 KV 공유가 "퇴거된 정보를 어떤 형태로 보존할 것인가"라는 동일 문제를 서로 다른 기제로 해결하는 보완 관계. Activity B의 KVM(이번 주 등재)과 쌍을 이루는 Activity C 방향 — KVM이 비연속 KV 누산(B)을, MELT가 루프 KV 압축 공유(C)를 담당하는 구조.

**A + B**

**B + C**
- KV-Fold(B)의 foldl 누산 KV와 MELT(C)의 단일 공유 게이팅 KV가 상보적 설계를 가리킨다. KV-Fold가 "기존 프리트레인드 모델에서 훈련-프리로 청크를 순차 누산"한다면, MELT는 "루프드 아키텍처에서 학습 가능한 게이팅으로 공유 KV를 유지"한다. 두 접근을 결합하면 훈련-프리 단계에서 KV-Fold로 초기 누산 후, 파인튜닝 단계에서 MELT 게이팅으로 압축 정제하는 2단계 파이프라인이 가능하다.
- KVM(B)의 고정-크기 압축 블록-재귀 KV 상태에 When Quantization Is Free(C)의 SRFT + INT4 양자화를 적용하면, KVM의 메모리-고정 KV 상태를 추가로 3× 압축해 에지 Apple Silicon 기기에서 KVM을 실용화하는 경로가 열린다.
- Shadow Mask Distillation(C)을 Activity B의 비연속 KV 재사용 기반 모델 파인튜닝에 적용하면, 비연속 세그먼트 압축을 포함하는 RL 정렬 루프에서 오프-폴리시 편향 없이 학습하는 "비연속 KV 재사용 인식 RL 파인튜닝"이 가능하다.

**A + C**
- SMD(C)의 RL 후훈련에서 KV 압축이 확립되면, Activity A의 서빙 스케줄러가 훈련과 서빙 단계 모두에서 동일 KV 압축 정책을 사용하는 "훈련-서빙 KV 압축 일관성" 시스템이 가능하다. 서빙 스케줄러가 압축 비율을 조절할 때 SMD로 훈련된 모델이 압축 정책 변화에 더 강건하게 반응한다.
- PBKV(A)의 예측 기반 KV 관리가 When Quantization Is Free(C)의 INT4 양자화 결정과 결합되면, 재사용 확률이 낮은 에이전트 KV는 INT4로 압축해 GPU에 저비용으로 유지하고 재사용 확률이 높은 KV는 FP16으로 유지하는 "예측-기반 적응 비트폭 에이전트 KV 스토리지"가 가능하다.

**A + B + C**
- 통합 파이프라인 (에이전틱 장문 추론 + RL 정렬): PBKV 예측 기반 에이전트 KV 관리(A) → KV-Fold foldl 청크 누산(B) → SMD RL 학습 오프-폴리시 편향 제거(C) → MELT 루프 공유 KV(C) → When Quantization Is Free INT4 에지 압축(C). PBKV가 어떤 청크를 보존할지 결정하고, KV-Fold가 청크를 누산하며, SMD가 RL 파인튜닝 중 압축 편향을 제거하고, MELT가 루프 연산에서 메모리를 고정하며, INT4 양자화가 에지 배포를 실용화하는 완전한 파이프라인.
