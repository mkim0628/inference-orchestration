## Activity A — KV Cache-aware Scheduling / Orchestration

### 새로운 논문·구현체

#### 1. [KV-RM: Regularizing KV-Cache Movement for Static-Graph LLM Serving](https://arxiv.org/abs/2605.09735)
- **출처**: arXiv 2605.09735 (2026-05-10, Zhiqing Zhong, Zhijing Ye, Jian Zhang, Weijian Zheng, Bolun Sun, Xiaodong Yu)
- **핵심 아이디어**: 정적-그래프(static-graph) LLM 디코더는 예측 가능한 커널 론치, 고정 텐서 형상, 낮은 제출 오버헤드를 제공하지만, 온라인 디코딩은 요청 길이 차이, 비동기 EOS 이벤트, 논리적 KV 이력의 시간적 단편화(fragmentation)라는 불규칙한 KV 캐시 이동을 유발해 정적 그래프 실행 모델과 근본적으로 충돌한다. KV-RM은 세 가지 핵심 설계로 이를 체계화한다: (1) **논리적 KV 이력과 물리적 스토리지의 분리** — 블록 페이저(block pager)로 활성 KV 상태를 추적하고, 각 디코딩 스텝을 단일 commit으로 구체화해 정적 그래프 텐서에 맞게 KV 레이아웃을 정규화, (2) **병합-단계 전송 경로(merge-staged transport path)** — 비연속 KV 매핑을 대형 전송 그룹으로 합체(coalesce)해 랜덤 I/O를 대용량 순차 전송으로 변환하는 내부 결속(intra-commit coalescing), (3) **정적 그래프와의 정합** — CUDA Graph나 Triton 정적 스케줄러가 예상하는 고정 텐서 형상을 유지하면서 동적 KV 이력 변화를 수용.
- **연구 목표와의 관련성**: high
- **주목할 점**: 정적 그래프 LLM 서빙 환경에서 KV 캐시의 논리-물리 분리와 비연속 KV 합체 전송을 최초로 체계화한 논문. Activity A의 "스케줄링 오버헤드 TTFT p50 +5% 이내" 목표를 정적-그래프 실행 환경에서도 달성하는 새로운 경로를 제시한다. Tutti(이미 보고)의 GPU-centric KV 오브젝트 스토어가 SSD→GPU 전송을 GPU io_uring으로 가속했다면, KV-RM은 CPU-side 논리-물리 분리와 비연속 합체로 정적 그래프 제약 안에서 KV 이동을 정규화한다 — 두 접근의 결합이 "논리-물리 분리(KV-RM) + GPU-native I/O(Tutti)"로 완전한 KV 이동 최적화 스택이 가능하다. Activity B의 비연속 세그먼트 재조합에서 "비연속 KV 매핑을 병합 그룹으로 합체"하는 KV-RM 원칙이 세그먼트 재조합 시 단편화된 블록들을 단일 전송 그룹으로 묶는 "세그먼트 코얼레싱 전송기"로 직접 활용 가능하다.

#### 2. [Not All Thoughts Need HBM: Semantics-Aware Memory Hierarchy for LLM Reasoning](https://arxiv.org/abs/2605.09490)
- **출처**: arXiv 2605.09490 (2026-05-10, Aojie Yuan, Tianqi Shen, Dajun Zhang — University of Southern California, University of Wisconsin–Madison)
- **핵심 아이디어**: 추론(reasoning) LLM이 생성하는 수천 개의 Chain-of-Thought(CoT) 토큰의 KV 캐시가 희소한 GPU HBM을 점령하는 문제를 해결한다. 기존 KV 퇴거 방법은 추론 태스크에서 캐시의 절반만 제거해도 정확도가 0~2.5%로 붕괴하는 반면, "Not All Thoughts Need HBM"(NAtH)은 토큰을 중요도에 따라 네 티어로 분류한다: (1) **HBM** — 최고 중요도 토큰 즉시 접근, (2) **DDR(CPU 메모리)** — 중간 중요도 토큰 오프로딩, 어텐션 스텝 전 전체 정밀도로 프리페치, (3) **압축** — 저중요도 토큰 압축 보존, (4) **퇴거** — 매우 낮은 중요도 토큰만 영구 제거. 누적 어텐션 점수(cumulative attention score)로 티어를 결정하며, DDR 오프로딩은 "제로 근사 오류 오프로딩(zero-approximation-error offloading)"으로 정형화한다 — CPU에서 프리페치된 토큰이 HBM에 있었을 경우와 정확히 동일한 어텐션 연산에 기여한다. 핵심 이론 결과: 정확도는 HBM 잔류 비율이 아니라 **퇴거 비율(eviction ratio)** 에만 의존한다.
- **연구 목표와의 관련성**: high
- **주목할 점**: GSM8K에서 3% 퇴거 시 전체 캐시 정확도의 91%, MATH-500에서 71% 유지 — 추론 태스크에서 극단적으로 낮은 퇴거 비율로 높은 정확도를 보존하는 최초 시스템. "정확도는 퇴거 비율에만 의존한다"는 이론적 발견이 Activity A의 "캐시 퇴거율 최소화" 목표에서 "최소 퇴거(예: 3%)만으로도 추론 정확도를 90% 이상 보존 가능"함을 정형화한다. Activity A의 스케줄러가 메모리 압박 시 어느 토큰을 DDR로 보낼지(퇴거 없음) vs 영구 퇴거할지를 NAtH 누적 어텐션 점수로 결정하는 "퇴거-최소화 DDR 오프로딩 스케줄러"가 가능하다. Tutti(이미 보고)의 GPU io_uring 비동기 전송이 SSD 오프로딩을 담당한다면, NAtH는 CPU-DDR 오프로딩을 추론 정확도 보존 제약 하에서 수행하는 상보적 계층이다. TTKV(이미 보고)의 시간 기반 HBM/DRAM 계층화에 NAtH의 누적 어텐션 중요도 신호를 결합하면 "시간적 최근성 + 의미적 중요도" 이중 기준 4-티어 메모리 계층이 완성된다. Activity C의 "Effective Context Length 2× 이상" 목표를 추론 워크로드에서 DDR 오프로딩으로 달성하는 가장 실용적인 경로 중 하나다.

## Activity C — KV Cache Compression

### 새로운 논문·구현체

#### 1. [Make Each Token Count: Towards Improving Long-Context Performance with KV Cache Eviction](https://arxiv.org/abs/2605.09649)
- **출처**: arXiv 2605.09649 (2026-05-10, Ngoc Bui, Hieu Trung Nguyen, Arman Cohan, Rex Ying — Yale University, The Chinese University of Hong Kong)
- **핵심 아이디어**: 기존 KV 퇴거 방법들의 암묵적 가정인 "전체 캐시 어텐션이 최적"이 긴 컨텍스트에서 틀렸음을 주장한다. 핵심 통찰: 긴 컨텍스트에서 관련 없는 토큰들이 유용한 증거로부터 어텐션을 희석하므로, 선택적·학습 가능한 퇴거가 전체 캐시를 초과하는 성능을 달성할 수 있다. 이를 위해 **전역 리텐션 기반 KV 퇴거(global retention-based KV eviction)** 방법을 도입한다: (1) **경량 리텐션 게이트** — 각 KV 항목에 미래 효용 점수를 할당하는 학습 가능한 게이트, (2) **공유 최종 점수 투영** — 모든 레이어와 헤드에 걸쳐 리텐션 게이트 점수를 보정하는 공유 투영으로 단일 전역 퇴거 정책 실현. 이 단일 전역 정책 하에서 서로 다른 레이어·헤드·모달리티의 토큰들이 동일 메모리 예산 안에서 직접 경쟁해 최적 퇴거 집합을 결정한다.
- **연구 목표와의 관련성**: high
- **주목할 점**: 다양한 장문 컨텍스트 언어·비전-언어 추론 및 다중 대화 벤치마크에서 KV 메모리를 대폭 줄이면서 전체 캐시 추론을 일치하거나 **초과하는** 성능 달성 — 선택적 퇴거가 전체 캐시보다 우월할 수 있음을 실증한 최초 논문 중 하나. Activity C의 "정확도 ±1% 이내" 제약을 "퇴거로 오히려 정확도를 높인다"는 새로운 각도에서 달성하는 방향. LKV(이미 보고, 15% KV 유지 near-lossless)가 예산과 선택을 엔드-투-엔드 학습으로 최적화한 반면, "Make Each Token Count"는 공유 투영으로 전역 경쟁 정책을 실현해 모달리티 간 통합 퇴거를 처음으로 다룬다. LookaheadKV(이미 보고)의 미래 어텐션 패턴 예측 기반 퇴거와 결합 시 "LookaheadKV가 미래 유용성을 예측 + 전역 리텐션 게이트가 글로벌 경쟁 집합 결정"하는 이중 최적화 전역 퇴거가 가능하다. LaProx(이미 보고)의 레이어 단위 출력-인식 중요도 점수 + "Make Each Token Count"의 전역 경쟁 리텐션 게이트 = "레이어별 중요도 계산 + 전역 예산 내 경쟁 퇴거"의 두 단계 전역 퇴거 파이프라인 후보.

#### 2. [Self-Pruned Key-Value Attention: Learning When to Write by Predicting Future Utility (SP-KV)](https://arxiv.org/abs/2605.14037)
- **출처**: arXiv 2605.14037 (2026-05-13, Gergely Szilvasy, Manuel Faysse, Maria Lomeli, Matthijs Douze, Pierre-Emmanuel Mazaré, Loïc Cabannes, Wen-tau Yih, Hervé Jégou — Meta FAIR)
- **핵심 아이디어**: KV 캐시 압축의 기존 접근들은 이미 기록된(written) KV를 사후에 퇴거하는 반면, SP-KV는 **KV를 캐시에 기록할지 여부를 사전에(at write time) 결정**하는 근본적으로 다른 패러다임을 도입한다. 세 가지 핵심 설계: (1) **로컬 윈도우** — 최근 KV는 항상 접근 가능하도록 보장하는 고정 슬라이딩 윈도우, (2) **경량 효용 예측기(lightweight utility predictor)** — 각 KV 쌍에 점수를 매겨 예측 효용이 임계값을 초과하는 경우에만 장기 KV 캐시에 기록, (3) **엔드-투-엔드 공동 훈련** — LLM과 효용 예측기를 다음-토큰 예측 손실만으로 공동 학습해 추가 감독 신호 불필요. 고정 압축 비율을 강제하지 않고 입력에 적응하는 **동적 희소화(dynamic sparsification)** 로 작동한다. 학습 완료 후 레이어별·헤드별 구조적 희소성 패턴(structured sparsity pattern)이 자연스럽게 드러나며, 이를 분석해 하이브리드 로컬-글로벌 어텐션 아키텍처 설계에 활용 가능하다.
- **연구 목표와의 관련성**: high
- **주목할 점**: 긴 시퀀스일수록 더 압축 가능해 KV 캐시 크기를 3~10× 동적으로 절감하며, 검증 손실과 다양한 다운스트림 태스크 성능의 저하가 거의 없음. Meta FAIR의 대형 연구 그룹(8명)이 발표한 논문으로 실용화 기반이 탄탄하다. Activity C의 "메모리 −30% 이상" 목표를 3~10× 절감으로 대폭 초과하면서 "정확도 ±1% 이내" 제약을 엔드-투-엔드 학습으로 보장하는 새로운 아키텍처 수준 접근. 기존 퇴거 방법(H2O, SnapKV, LaProx, LKV, LookaheadKV)이 모두 "이미 기록된 KV를 언제 버릴 것인가"를 다루는 반면, SP-KV는 "언제 기록할 것인가"라는 상보적 관점으로 접근 — 두 방향의 결합("SP-KV로 쓰기 선택적 + LookaheadKV로 퇴거 정밀화")이 최대 KV 절감을 달성한다. 레이어·헤드별 구조적 희소성 패턴 분석 결과가 Activity B의 비연속 세그먼트 재사용에서 "어떤 헤드의 KV가 재사용 가능하고 어떤 헤드는 재계산이 필요한가"를 결정하는 헤드-인식 비연속 재사용 정책의 이론적 기반을 제공한다.

#### 3. [A First Comprehensive Study of TurboQuant: Accuracy and Performance](https://vllm.ai/blog/2026-05-11-turboquant)
- **출처**: vLLM 공식 블로그 (2026-05-11, vLLM 팀)
- **핵심 아이디어**: vLLM v0.20.0에 통합된 TurboQuant 2비트 KV 캐시의 정확도·성능에 대한 최초 포괄 연구를 발표한다. TurboQuant는 KV 캐시 스토리지를 3~4비트로 압축하고 어텐션 계산 시 BF16으로 역양자화하는 방식으로, FP8 KV(스토리지+계산 모두 FP8 Tensor Core 사용)와 구조적으로 다르다. 4개 모델(30B~200B+, 덴스·MoE 포함), 5개 벤치마크(프리필-헤비 장문 컨텍스트 검색 + 디코드-헤비 추론 태스크)에서 TurboQuant 변형 및 FP8 대비 정확도·지연·처리량을 정량 비교한다.
- **연구 목표와의 관련성**: high
- **주목할 점**: **FP8(--kv-cache-dtype fp8)가 KV 캐시 양자화의 최선 기본값으로 확립** — 2× 용량 + 거의 없는 정확도 손실 + BF16 대비 대부분 지표에서 동등 또는 초과. TurboQuant 4bit-nc가 가장 실용적인 TurboQuant 변형이지만 정확도·지연·처리량 비용이 수반됨: Qwen3-30B에서 ~10~60%, Llama-3.3-70B에서 ~10~68% 지연 오버헤드. Llama-3.3-70B 4xH100에서 FP8가 BF16 대비 2.6× 버스트 처리량 + 2× KV 용량의 파레토 우위 실증. Activity C의 "정확도 ±1% 이내" 제약에서 TurboQuant 2비트(모든 변형)는 이 제약을 충족하기 어려운 반면, FP8는 충족하면서도 2× 용량을 제공하는 실용 선택임을 vLLM 공식 연구가 확인. eOptShrinkQ(이미 보고)의 저랭크+TurboQuant 잔차 2.2비트 조합이 이 블로그 결과와 상반된다는 점이 주목 — TurboQuant를 스탠드얼론 사용할 때와 저랭크 구조 제거 후 잔차에 적용할 때의 정확도-효율 차이가 Activity C 구현의 중요한 설계 선택이다. Activity C의 6단계 vLLM 이식 시 "FP8 기본 + 용량 압박 시 TurboQuant 4bit-nc"의 2단계 적응 양자화 정책을 기준으로 설정하는 것이 이 연구가 시사하는 실용 가이드라인이다.

## Activity 간 시너지 포인트

**A + C**
- NAtH(A/C)의 DDR 오프로딩 계층과 "Make Each Token Count"(C)의 전역 리텐션 게이트를 결합하면, 리텐션 게이트가 낮은 미래 효용 점수를 가진 토큰을 영구 퇴거 대신 DDR 오프로딩 후보로 분류하는 "리텐션-기반 오프로딩 티어 결정기"가 가능하다. 정확도는 퇴거 비율에만 의존한다는 NAtH의 이론적 발견이 "리텐션 게이트 임계값을 높여 오프로딩 비율을 높이되 퇴거 비율을 최소화"하는 최적 정책 설계의 수학적 기반이 된다.

**B + C**
- SP-KV(C)의 레이어·헤드별 구조적 희소성 패턴 분석이 Activity B의 비연속 세그먼트 재사용에서 "어느 레이어·헤드의 KV가 구조적으로 희소하므로 재사용보다 재계산이 저렴한가, 어느 레이어·헤드의 KV가 고밀도이므로 재사용이 필수적인가"를 사전에 결정하는 "SP-KV 희소성 프로파일 기반 비연속 재사용 정책"이 가능하다. LaProx(이미 보고)의 레이어 단위 중요도 점수와 SP-KV의 구조적 희소성 패턴을 앙상블하면 이 정책의 정확도가 더욱 향상된다.
- "Make Each Token Count"(C)의 전역 리텐션 게이트를 Activity B의 세그먼트-레벨에 적용하면, 서로 다른 비연속 세그먼트들이 공유 메모리 예산 안에서 직접 경쟁해 "어떤 세그먼트를 GPU에 유지하고 어떤 것을 DDR로 보낼 것인가"를 동적으로 결정하는 "세그먼트-레벨 전역 리텐션 게이트"가 가능하다.

**A + B + C**
- 통합 파이프라인 (추론 LLM 에이전틱 비연속 KV 최적화): KV-RM 비연속 KV 합체 전송(A) → AdapShot RoPE 재인코딩으로 비연속 세그먼트 재조합(B) → SP-KV 구조적 희소성 기반 재사용 정책(B+C) → "Make Each Token Count" 전역 리텐션 게이트 퇴거(C) → NAtH DDR 오프로딩으로 3% 이하 퇴거 유지(A/C). KV-RM이 비연속 블록을 효율 전송하고, AdapShot이 재조합 위치를 보정하며, SP-KV가 헤드별 재사용 가능 레이어를 식별하고, "Make Each Token Count"가 전역 예산 내 최적 퇴거를 결정하며, NAtH가 퇴거 대신 DDR 보존으로 추론 정확도를 지키는 완전한 비연속-추론 KV 최적화 스택.

- **SP-KV(2605.14037, Meta FAIR) — Write-time KV 희소화**: Write-time KV 기록 결정이라는 새로운 압축 패러다임이 Activity C 구현에서 기존 사후 퇴거와 보완적으로 결합 가능하다. "SP-KV가 불필요한 KV를 처음부터 쓰지 않고, LookaheadKV가 이미 쓰인 KV를 미래 패턴으로 퇴거"하는 이중 파이프라인이 Activity C의 최대 KV 절감 목표에 접근하는 가장 강력한 후보다. SP-KV를 LLM 파인튜닝 단계에서 훈련하는 비용이 수반되므로 6단계 vLLM 이식과 7단계 평가에서 훈련 비용 대비 KV 절감 효과를 정량 평가할 것.
- **vLLM TurboQuant 연구(2026-05-11) — FP8 기본값 확립**: Activity C의 vLLM 이식(6단계)에서 KV 양자화 기준선을 "FP8 기본 + 메모리 극한 시 TurboQuant 4bit-nc"로 설정하는 것이 이 포괄 연구가 시사하는 실용 가이드라인이다. Activity C의 자체 구현(저랭크+양자화 등)을 vLLM v0.20의 TurboQuant 기준선과 비교하는 평가 실험을 추가해 상대적 장점을 정량화할 것.
