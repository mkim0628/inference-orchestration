# 트렌드 리포트 — 2026-05-17

## 전체 요약

1. **Activity A (스케줄링)**: 오늘 주목할 핵심 변화는 vLLM v0.21.0 정식 릴리즈다. KV 오프로딩 서브시스템이 Hybrid Memory Allocator(HMA)와 통합되어 스케줄러 측 슬라이딩 윈도우 그룹 지원, 멀티-커넥터 HMA, 오프로딩 커넥터 DCP/PCP 지원 등이 한꺼번에 포함됐다. 이는 Activity A의 "KV 오프로딩 + CPU/디스크 계층화" 방향에서 vLLM 공식 구현이 처음으로 통합된 HMA 기반 아키텍처를 제공하는 이정표다. 순수 학술 논문 측면에서는 이번 주 신규 Activity A 논문은 발견되지 않았다.

2. **Activity B (비연속 재사용)**: 이번 주 Activity B에 해당하는 완전히 새로운 논문은 확인되지 않았다. 직전 주까지 보고된 RelayCaching, ICaRus, QKVShare, KV-Fold, KVM, AdapShot, Louver가 현재 최전선이다.

3. **Activity C (압축)**: 이번 주 Activity C에 해당하는 완전히 새로운 논문도 발견되지 않았다. vLLM Q2 2026 로드맵에서 INT8 동적 퍼-토큰 KV 캐시 양자화를 시작으로 NVFP4 등 동적 KV 압축 지원이 공식화됐다. 기존 최전선(SP-KV, "Make Each Token Count", LookaheadKV, FibQuant, AB-Sparse, Fluxion)이 유효하다.

4. **A+C 시너지 심화**: vLLM v0.21.0의 HMA 통합이 Activity A의 스케줄러-HMA 공동 설계와 Activity C의 vLLM 이식(6단계)을 연결하는 실용 기반을 제공한다. v0.20에서 TurboQuant 2-bit이 추가됐다면, v0.21에서는 오프로딩 커넥터가 HMA와 통합되어 CPU/디스크 계층 KV를 스케줄러가 직접 제어하는 구조가 완성됐다.

5. **전반적 관찰**: 5월 둘째 주까지 급증하던 KV 캐시 논문 양이 이번 주에는 소강 상태다. 주요 연구 그룹들이 NeurIPS 2026 제출 마감(6월)을 앞두고 준비 중인 것으로 추정된다. 다음 주부터 arXiv 제출량이 다시 급증할 가능성이 높다.

## Activity A — KV Cache-aware Scheduling / Orchestration

### 새로운 논문·구현체

#### 1. [vLLM v0.21.0 — KV Offloading + Hybrid Memory Allocator 통합 릴리즈](https://github.com/vllm-project/vllm/releases/tag/v0.21.0)
- **출처**: vLLM GitHub 릴리즈 (2026-05-12 전후, vLLM 팀)
- **핵심 아이디어**: vLLM v0.21.0은 KV 오프로딩 서브시스템과 Hybrid Memory Allocator(HMA)를 공식 통합했다. 주요 KV 캐시 관련 변경: (1) **KV 오프로딩 + HMA 통합** — 스케줄러 측 슬라이딩 윈도우 그룹 지원(scheduler-side sliding window group support)과 full HMA enablement로 CPU-DRAM 및 디스크 계층을 스케줄러가 직접 제어, (2) **멀티-커넥터 HMA** — 단일 HMA 인스턴스가 복수의 KV 커넥터(예: LMCache, Mooncake)를 동시 지원하는 멀티-커넥터 아키텍처, (3) **오프로딩 커넥터 DCP/PCP 지원** — OffloadingConnector에서 per-job store completion(DCP)과 per-job prefetch completion(PCP)을 지원해 오프로딩 요청별 세밀한 완료 추적 가능, (4) **TOKENSPEED_MLA 어텐션 백엔드** — DeepSeek-R1/Kimi-K25 모델의 Blackwell GPU 프리필+디코딩을 위한 새 어텐션 백엔드 추가, (5) **CUDA 13.0 기본 PyPI 휠** — 새 GPU 아키텍처 지원 기반 확장. 추가로 투기적 디코딩이 추론/사고 예산을 올바르게 준수하도록 개선됐다.
- **연구 목표와의 관련성**: high
- **주목할 점**: v0.20.0에서 TurboQuant 2-bit KV(4× 용량)가 추가됐다면, v0.21.0에서는 KV 오프로딩과 HMA가 스케줄러 레벨에서 통합돼 "스케줄러가 CPU/디스크 계층 KV를 직접 제어"하는 구조가 완성됐다. Activity A의 "KV 오프로딩 스케줄러 설계" 목표에서 6단계 vLLM 이식이 이 HMA 통합 아키텍처를 기반으로 구현돼야 한다는 것을 시사한다. NAtH(2605.09490, 이미 보고)의 DDR 오프로딩 정책과 v0.21 HMA 통합이 맞물리면, "NAtH 누적 어텐션 점수 기반 오프로딩 결정 → vLLM v0.21 HMA로 실행"하는 스케줄러 오프로딩 파이프라인의 구현이 즉시 가능하다. Tutti(2605.03375, 이미 보고)가 GPU-centric SSD 오프로딩을 제안했다면, v0.21 HMA는 프레임워크 내장 CPU/디스크 오프로딩 인프라로 Tutti 원칙의 절반을 공식 채택한 셈이다. vLLM Q2 2026 로드맵의 "KV cache manager rethink for complex KV cache layout + CPU 오프로딩 + Disk + overall connector API" 항목이 v0.21에서 HMA 통합으로 구체화됐다. Activity C 6단계 이식에서 v0.21의 멀티-커넥터 HMA를 활용하면 자체 압축 정책을 독립 커넥터로 플러그인하는 깔끔한 아키텍처가 가능하다.

## Activity 간 시너지 포인트

**A + C (vLLM v0.21.0 HMA 통합 중심)**
- vLLM v0.21.0의 HMA 통합(A)이 Activity C의 KV 압축 기법들을 vLLM에 이식하는 새로운 플러그인 경로를 제공한다. 멀티-커넥터 HMA 아키텍처를 활용하면 Activity C의 각 압축 기법(FibQuant VQ 압축, SP-KV write-time 희소화, "Make Each Token Count" 전역 리텐션 퇴거)을 독립 HMA 커넥터로 구현하고 스케줄러와 직접 통합하는 "압축 정책 플러그인" 아키텍처가 가능하다.
- vLLM Q2 2026 로드맵의 INT8→NVFP4 동적 KV 양자화 순차 지원이 Activity C의 "정확도 ±1% 이내" 제약을 vLLM 공식 동적 양자화로 검증하는 평가 경로를 제공한다. RateQuant(이미 보고)의 헤드별 최적 비트 할당 + vLLM INT8/NVFP4 동적 양자화 백엔드의 결합이 Activity C의 양자화 파이프라인 vLLM 이식의 가장 명확한 경로다.

**A + B (vLLM KV cache manager rethink 중심)**
- vLLM Q2 2026 로드맵의 "KV cache manager rethink for complex KV cache layout"이 Activity B의 비연속 KV 레이아웃을 vLLM 스케줄러가 직접 인식하는 구조를 준비하는 중이다. AdapShot(B)의 RoPE 위치 재인코딩 + Louver(B)의 제로 거짓 음성 검색 + KV-Fold(B)의 foldl 청크 누산이 새로운 vLLM KV cache manager와 통합되는 시점이 Activity B의 vLLM 이식 기회가 된다.

**B + C**
- 이번 주 신규 발견 없음. 직전 주의 "SP-KV(C) 구조적 희소성 패턴 → B 헤드-인식 재사용 정책", "FibQuant(C) 랜덤-접근 VQ → B 비연속 세그먼트 독립 압축" 연결이 여전히 가장 유효한 B+C 시너지다.

**A + B + C (통합 관점)**
- vLLM v0.21.0 HMA(A) + KV cache manager rethink 진행(A/B) + INT8/NVFP4 동적 양자화 로드맵(C)이 한 사이클 안에 공식화되면서, 본 연구의 "비연속 KV 재사용 + 압축 + 스케줄러 통합"을 vLLM 공식 아키텍처에 이식하는 시점이 Q2-Q3 2026로 압축됐다. Activity B+C의 자체 구현 완성과 vLLM v0.21 이후 버전의 KV cache manager 변경을 동시 모니터링해 이식 타이밍을 최적화해야 한다.
