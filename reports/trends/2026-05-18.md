# 트렌드 리포트 — 2026-05-18

## 전체 요약

1. **Activity A (스케줄링)**: 멀티-라운드 및 멀티-모델 분산 서빙에서 KV 캐시를 효율적으로 라우팅·재사용하는 방향이 두 편의 새 논문으로 구체화됐다. AMPD(2602.14516)는 멀티-라운드 LLM 워크플로우에서 P/D 분리 서빙의 적응형 배치 관리를 제시하며 최대 339.74% SLO 달성률 향상을 보였고, PrefillShare(2602.12029)는 멀티-모델 분산 서빙에서 프리필 단계를 여러 모델이 공유하는 아키텍처로 p95 지연 4.5×, 처리량 3.9× 향상을 달성했다. SGLang v0.5.11(2026-05-05 LMSYS 블로그)은 대규모 전문가 병렬 P/D 분리 스택으로 DeepSeek V3/R1을 64-H200 클러스터에서 단일 노드 대비 3.8× 처리량·3.5× TTFT 단축을 실증했다.

2. **Activity B (비연속 재사용)**: 이번 주(2026-05-11 ~ 2026-05-18) 기간에 Activity B에 해당하는 완전히 새로운 논문은 확인되지 않았다. PrefillShare(Activity A 등재)는 ICaRus(이미 보고)의 크로스-모델 KV 공유 방향을 분산 서빙 레이어까지 확장하는 상보적 기여를 담고 있어 Activity B 관점에서도 주목해야 한다. 현재 최전선(RelayCaching, ICaRus, QKVShare, KV-Fold, KVM, AdapShot, Louver)이 유효하다.

3. **Activity C (압축)**: 이번 주 Activity C에 해당하는 완전히 새로운 논문도 확인되지 않았다. SGLang v0.5.11 블로그가 FP8 KV 기본 정밀도 채택을 실운영에서 재확인하는 맥락을 제공했고, SGLang의 DP Attention이 멀티-GPU 환경에서 KV 캐시 중복을 제거하는 방향을 제시했다. 기존 최전선(SP-KV, Make Each Token Count, LookaheadKV, FibQuant, AB-Sparse, Fluxion)이 유효하다.

4. **A+B 시너지**: AMPD의 "어디서 멀티-라운드 프리필을 수행할 것인가" 적응형 결정과 PrefillShare의 공유 프리필 모듈이 결합되면, 멀티-라운드 세션에서 공유 KV를 분산 풀에 유지하면서 각 라운드의 증분 프리필만 적응 노드에서 수행하는 완전한 멀티-라운드 비연속 KV 재사용 아키텍처가 가능하다.

5. **전반적 관찰**: 5월 셋째 주는 NeurIPS 2026 마감(6월)을 앞둔 소강 상태가 지속되고 있다. 한편 지난주에 보고된 vLLM v0.21.0 HMA 통합 이후 분산 서빙 프레임워크 수준에서의 대규모 실증(SGLang v0.5.11 LMSYS 64-GPU 실험)이 동시에 나타나 "이론 논문" 소강과 "시스템 검증" 활성화가 교차하는 구조다.

## Activity A — KV Cache-aware Scheduling / Orchestration

### 새로운 논문·구현체

#### 1. [AMPD: Efficient Multi-round LLM Inference over Disaggregated Serving](https://arxiv.org/abs/2602.14516)
- **출처**: arXiv 2602.14516 (2026-02-16, Wenhao He, Youhe Jiang, Penghao Zhao, Quanqing Xu, Eiko Yoneki, Bin Cui, Fangcheng Fu — Cambridge / Peking University / NVIDIA)
- **핵심 아이디어**: 자율 에이전트나 반복 검색 같은 멀티-라운드 LLM 워크플로우가 확산되면서, P/D(Prefill-Decode) 분리 서빙 패러다임이 라운드 간 인터리브 프리필-디코딩 패턴을 제대로 처리하지 못하는 구조적 한계가 드러났다. AMPD는 세 단계로 이를 해결한다: (1) **적응형 배치 계획(offline planning)** — 두 단계의 최적 자원 할당과 병렬 전략을 사전 도출하는 계획 알고리즘, (2) **온라인 적응 스케줄링** — 실시간 워크로드에 따라 증분 프리필을 어디서(로컬 디코딩 노드 vs 원격 프리필 노드) 실행할지 동적 결정, (3) **KV 캐시 지연 읽기(lazy read)** — 라우팅 시 KV 캐시 자체 대신 메타데이터만 프리필 워커에 전달하고, 실제 실행 스케줄이 확정될 때 비로소 이전 디코딩 노드에서 KV를 읽어 전송·계산을 오버랩함으로써 불필요한 KV 전송 최소화. NVIDIA Dynamo 위에 구축된 AMPD 시스템으로 구현했다.
- **연구 목표와의 관련성**: high
- **주목할 점**: 다양한 LLM 모델·멀티-라운드 워크로드에서 Dynamo 대비 SLO 달성률 67.29~967.54%, vLLM 대비 평균 339.74% 향상. "KV 캐시 지연 읽기"는 Activity A의 "스케줄링 오버헤드 최소화" 원칙을 분산 환경으로 확장한 구현이다 — 스케줄러가 KV를 미리 이동하는 대신 실행 스케줄 확정 순간에만 읽는 pull-on-demand 방식으로 불필요한 네트워크 전송을 원천 제거한다. PBKV(이미 보고)의 에이전트 호출 예측이 "어떤 KV를 보존할 것인가"를 다루는 반면, AMPD는 "증분 프리필을 어느 노드에서 수행하고 KV를 언제 가져올 것인가"를 결정하는 상보적 문제를 해결한다. vLLM x Mooncake Store(이미 보고)가 클러스터-와이드 KV 풀에서 히트율을 극대화한다면, AMPD는 멀티-라운드 세션에서 라운드별 증분 프리필을 로컬/원격 중 최적 경로로 배분해 KV 전송 비용을 최소화하는 다른 수준의 최적화다. Activity B의 비연속 KV 재사용 관점에서 "이전 라운드의 디코딩 KV를 다음 라운드 프리필에 직접 재사용"하는 지연 읽기 패턴이 RelayCaching(이미 보고)의 에이전트 간 KV 재사용과 근본 원리를 공유한다.

#### 2. [PrefillShare: A Shared Prefill Module for KV Reuse in Multi-LLM Disaggregated Serving](https://arxiv.org/abs/2602.12029)
- **출처**: arXiv 2602.12029 (2026-02-12, Sunghyeon Woo 외 10인 — NAVER Cloud / 다수 소속)
- **핵심 아이디어**: 멀티-에이전트 시스템에서 여러 전문화 모델이 동일 프롬프트를 반복 프리필하고 각자 KV 캐시를 유지하는 이중 낭비를 해결한다. PrefillShare는 모델을 공유 프리필 모듈(freezed)과 태스크-특화 디코딩 모듈로 분리하고, 여러 모델이 하나의 프리필 모듈이 생성한 KV 캐시를 공유한다. 세 가지 핵심 기여: (1) **캐시-컨디셔닝 파인튜닝** — 디코딩 모듈만 기반 모델의 KV 캐시를 조건으로 파인튜닝해 공유 프리필과의 호환성 보장, (2) **vLLM 기반 이종 라우팅** — 다양한 태스크-특화 모델들이 동일 프리필 모듈을 공유하는 분산 서빙 라우팅 메커니즘, (3) **disaggregated 서빙 통합** — KV 캐시 핸드오프 메커니즘으로 프리필-디코딩 분리 환경에서 공유 KV를 여러 디코딩 워커에 효율 배포.
- **연구 목표와의 관련성**: high
- **주목할 점**: 다양한 태스크·모델에서 full fine-tuning과 동등한 정확도를 유지하면서 멀티-모델 에이전트 워크로드에서 p95 지연 4.5×, 처리량 3.9× 향상. ICaRus(이미 보고, 2603.13281)가 "모든 레이어에 걸쳐 동일 KV를 공유하는 아키텍처 수준 재설계"로 8 모델·3.8× 처리량을 달성한 반면, PrefillShare는 "분산 서빙 스택 레이어에서 프리필 노드를 공유하는 시스템 수준 설계"로 유사한 효과를 추가 아키텍처 변경 없이 달성한다 — 두 접근이 알고리즘(ICaRus, Activity B)과 시스템(PrefillShare, Activity A) 수준에서 상보적이다. Activity A의 "멀티-노드 KV 마이그레이션 라우팅" 목표에서 단일 프리필 노드의 KV를 N개 디코딩 노드에 배포하는 "1-to-N KV 배포 라우팅"이라는 새로운 패턴을 정의한다. CacheFlow(이미 보고)의 3D 병렬 KV 복원이 "이미 생성된 KV를 어떻게 빨리 복원할 것인가"를 다루는 반면, PrefillShare는 "공통 KV를 한 번만 생성하고 다수 모델에 배포"하는 근본적으로 다른 KV 절감 전략이다.

#### 3. [SGLang v0.5.11 — 대규모 전문가 병렬 P/D 분리 실증](https://www.lmsys.org/blog/2025-05-05-large-scale-ep/)
- **출처**: LMSYS 공식 블로그 (2026-05-05, SGLang 팀 / LMSYS Org)
- **핵심 아이디어**: SGLang v0.5.11 릴리즈와 함께 DeepSeek V3/R1 모델을 64-H200 클러스터에서 대규모 전문가 병렬(Expert Parallelism, EP) + P/D 분리 스택으로 서빙하는 최초 오픈소스 대규모 실증을 발표했다. 핵심 기술 요소: (1) **DP Attention** — 멀티-GPU 환경에서 KV 캐시 중복을 제거하는 데이터 병렬 어텐션으로 메모리 오버헤드 대폭 감소 (SGLang v0.4부터 도입, v0.5에서 하이브리드 DP+TP로 확장), (2) **P/D 분리 + decode-radix 캐시** — 프리필에서 생성된 KV 상태를 decode-radix 캐시에 저장해 디코딩 단계에서 재사용, (3) **RDMA 기반 모델 가중치 전송** — Mooncake TransferEngine + RDMA P2P로 1T 파라미터 모델(Kimi-K2)의 가중치를 7× 빠르게(53s → 7.2s) 업데이트하는 RL 학습 환경 지원.
- **연구 목표와의 관련성**: high
- **주목할 점**: 64-H200 클러스터에서 단일 노드 대비 3.8× 처리량·3.5× TTFT 단축을 오픈소스로 최초 실증. DeepSeek 공식 블로그 대비 거의 동등한 throughput(52.3K 입력 tokens/s 수준) 달성 → 대규모 MoE LLM 서빙의 오픈소스 기준선이 됐다. Activity A의 "멀티-노드 KV 라우팅"과 "disaggregated prefill" 방향 모두에서 오픈소스 최대 규모 실증이 등장했다는 점에서 중요하다. vLLM v0.21.0 HMA 통합(이미 보고)이 단일 인스턴스 내 KV 계층화를 완성했다면, SGLang v0.5.11은 클러스터 수준 P/D 분리를 오픈소스로 최대 규모 검증한 이정표다. DP Attention의 KV 중복 제거 원리가 Activity A의 "멀티-노드 KV 관리" 설계에서 레이어별·헤드별 KV 복사본 수를 최소화하는 구조로 참조 가능하다.

## Activity B — Non-Contiguous KV Cache Reuse

### 새로운 논문·구현체

이번 주(2026-05-11 ~ 2026-05-18) 기간에 Activity B 범주의 완전히 새로운 논문·구현체는 확인되지 않았다. 직전 주까지 보고된 RelayCaching(2603.13289), ICaRus(2603.13281), QKVShare(2605.03884), KV-Fold(2605.12471), KVM(2605.09877), AdapShot(2605.03644), Louver(2605.06763)가 현재 Activity B의 최전선이다.

Activity A에 등재된 PrefillShare(2602.12029)는 공유 프리필 모듈이 생성한 KV를 여러 모델의 디코딩 워커가 직접 소비하는 구조로, "동일 프롬프트 KV를 비연속 디코딩 경로에서 재사용"하는 Activity B의 크로스-모델 비연속 재사용 문제의 분산 서빙 레이어 구현이기도 하다. ICaRus(이미 보고)가 아키텍처 수준에서 크로스-모델 KV 공유를 실현했다면, PrefillShare는 기존 모델 아키텍처를 수정하지 않고 시스템 수준에서 동일 목표를 달성하는 보완적 접근이다.

## Activity 간 시너지 포인트

**A + B (AMPD + RelayCaching + PrefillShare)**
- AMPD(A)의 지연 읽기 KV 재사용은 멀티-라운드 세션에서 이전 디코딩 노드의 KV를 다음 라운드 프리필이 필요로 할 때만 읽는 pull-on-demand 방식이다. RelayCaching(B)의 "이전 에이전트 디코딩 KV → 다음 에이전트 프리필 직접 재사용"과 동일한 원리를 분산 서빙 인프라 레이어에서 시스템으로 구현한 것이다. AMPD가 "언제 어느 노드에서 KV를 읽을 것인가"를 결정하고, RelayCaching이 "그 KV 중 어느 레이어의 어느 토큰을 재사용하고 어느 것을 재계산할 것인가"를 결정하는 역할 분리가 완성된 분산 멀티-라운드 비연속 KV 재사용 스택이 가능하다.
- PrefillShare(A)가 공유 프리필 KV를 분산 서빙 레이어에서 배포하고, ICaRus(B)가 아키텍처 수준에서 동일 KV를 공유하는 모델 파인튜닝을 제공하면, "아키텍처(ICaRus) + 시스템(PrefillShare)" 이중 레이어에서 크로스-모델 비연속 KV 공유를 강화하는 통합 설계가 가능하다.

**A + C (SGLang DP Attention + 압축)**
- SGLang v0.5.11의 DP Attention이 멀티-GPU에서 KV 중복을 제거하는 방식과 Activity C의 KV 압축(FP8/FibQuant/SP-KV)이 결합되면, KV 중복 제거(A)와 개별 KV 크기 감소(C)를 동시에 적용하는 "중복 제거 + 압축" 이중 메모리 절감이 가능하다. DP Attention이 N-GPU 환경에서 KV 복사본을 1/N으로 줄이고, SP-KV write-time 희소화가 각 KV의 크기를 3~10× 추가 절감하면, 총 메모리 절감은 두 요소의 곱으로 증폭된다.

**A + B + C (통합 분산 멀티-에이전트 KV 스택)**
- PrefillShare 공유 프리필(A) → ICaRus 크로스-모델 아키텍처 공유(B) → SP-KV write-time 희소화(C) → FibQuant 랜덤-접근 VQ 압축(C) → AMPD 지연 읽기 멀티-라운드 재사용(A) → vLLM x Mooncake Store 분산 KV 풀 보존(A). PrefillShare가 공통 KV를 한 번만 생성하고, ICaRus가 여러 모델에서 이를 공유하며, SP-KV와 FibQuant가 공유 KV를 압축하고, AMPD가 멀티-라운드에서 이 압축 KV를 지연 읽기로 재사용하며, Mooncake Store가 클러스터 전체에 분산 보존하는 완전한 엔터프라이즈 KV 재사용 스택이다.
