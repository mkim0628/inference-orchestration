## Activity A — KV Cache-aware Scheduling / Orchestration

### 새로운 논문·구현체

#### 1. [CONCUR: High-Throughput Agentic Batch Inference of LLM via Congestion-Based Concurrency Control](https://arxiv.org/abs/2601.22705)
- **출처**: arXiv 2601.22705 (2026-01-30, Yizhi Zheng 외 — 여러 소속, 저자 논문 페이지 yezhisheng.me)
- **핵심 아이디어**: 에이전틱 배치 추론에서 GPU KV 캐시가 메모리 용량이 소진되기 훨씬 전에 처리율이 급격히 저하되는 현상 — 저자들이 "중간 단계 스래싱(middle-phase thrashing)"으로 명명하는 — 을 해결한다. 근본 원인은 KV 캐시를 요청 레벨이 아닌 에이전트 레벨에서 관리하지 않아 장기 실행 에이전트들이 시간에 따라 KV 상태를 누적하면서 캐시 효율이 붕괴되는 것이다. CONCUR는 세 가지 원칙으로 이를 해결한다: (1) **에이전트-레벨 입장 제어(agent-level admission control)** — 요청이 아닌 에이전트를 스케줄링 단위로 삼아 활성 에이전트 수를 동적으로 제한, (2) **혼잡 기반 조절(congestion-based regulation)** — 분산 네트워크의 혼잡 제어에서 영감을 받아 KV 캐시 점유율 피드백 신호로 입장 가능한 에이전트 수를 실시간 조절, (3) **연속성 보존(continuity preservation)** — 이미 활성화된 에이전트의 실행 연속성을 보장하면서 신규 에이전트의 입장을 제한하여 일시 중단 없이 스래싱 방지. 기존 LLM 서빙 시스템 위에 경량 제어 미들웨어로 투명하게 추가 가능하다.
- **연구 목표와의 관련성**: high
- **주목할 점**: Qwen3-32B에서 **4.09× 처리량**, DeepSeek-V3에서 **1.9× 처리량** 향상 — 기존 요청 레벨 캐시 관리 대비. "중간 단계 스래싱"이라는 새로운 에이전틱 KV 병리를 최초로 명명하고 정형화한 점이 학술적 기여다. Activity A의 "캐시 퇴거율 최소화" 목표에서 퇴거가 발생하기 전에 에이전트 수를 제한해 KV 풀을 혼잡 이하로 유지하는 선제적(proactive) 접근이라는 점이 기존 반응적(reactive) 퇴거 방법과 근본적으로 다르다. PBKV(이미 보고, 2605.06472)가 미래 에이전트 호출을 예측해 KV 보존 결정을 사전에 내린다면, CONCUR는 예측 없이 혼잡 피드백만으로 에이전트 입장을 제어한다 — "예측 기반(PBKV) + 혼잡 기반(CONCUR)" 이중 레이어 에이전틱 KV 스케줄링이 최고 성능 조합이 될 수 있다. ThunderAgent(이미 보고, 2602.13692)가 LLM Program 추상화로 워크플로우 전체를 정적 분석해 스케줄링하는 반면, CONCUR는 런타임 혼잡 신호에만 의존하는 완전 동적 접근이다 — 두 방법의 하이브리드(Program 분석으로 구조 예측 + CONCUR로 런타임 혼잡 대응)가 에이전틱 KV 관리의 이론-실용 최전선이 된다.

### 새로운 논문·구현체

CONCUR(Activity A 등재)의 에이전트-레벨 입장 제어가 Activity B와 맞닿는 지점이 있다. CONCUR가 활성 에이전트 수를 제한해 KV 풀을 혼잡 이하로 유지하면, Activity B의 비연속 세그먼트 재사용 스케줄러가 풀에 충분한 여유 공간을 확보해 더 많은 비연속 세그먼트를 GPU HBM에 적재·유지할 수 있다. "CONCUR 입장 제어 → 비연속 세그먼트 HBM 적재 여유 확보 → Louver/AdapShot 비연속 재사용"의 3단계 파이프라인이 A+B 시너지의 실용 방향이다.

SpecAttn(Activity C 등재)의 검증-가이드 희소 어텐션에서 중요 KV 항목을 드래프트 단계에 재사용하는 원리가 Activity B의 비연속 재사용과 접근 방향을 공유한다. SpecAttn이 "검증 단계의 전체 KV에서 중요 KV 부분집합을 선별 → 드래프트에 재사용"한다면, Activity B는 "이전 요청·에이전트의 비연속 KV 부분집합 → 현재 요청에 재사용"한다 — "선별 KV 재사용"이라는 동일 패러다임의 투기적 디코딩 내부(SpecAttn)와 크로스-요청(Activity B) 두 적용 변형.

## Activity C — KV Cache Compression

### 새로운 논문·구현체

#### 1. [SpecAttn: Co-Designing Sparse Attention with Self-Speculative Decoding](https://arxiv.org/abs/2602.07223)
- **출처**: arXiv 2602.07223 (2026-02-06, Yikang Yue 외 2인)
- **핵심 아이디어**: 기존 희소 어텐션(sparse attention) 방법들이 KV 항목 선택과 투기적 디코딩의 드래프팅 단계를 독립 설계해 두 과제의 중간 결과를 공유하지 못하는 비효율을 해결한다. SpecAttn은 두 단계를 공동 설계(co-design)한다: (1) **검증-가이드 KV 선별(verification-guided KV selection)** — 자기-투기적 디코딩의 검증 단계에서 계산된 완전 어텐션 로짓(full attention logits)을 부산물로 활용해 중요 KV 항목을 식별. 검증 단계는 어차피 전체 KV를 참조하므로 추가 비용 없이 중요 KV 마스크를 생성, (2) **희소 드래프팅(sparse drafting)** — 검증에서 식별된 중요 KV만 로드해 드래프트 토큰을 생성. 이 "Collect-2-Query" 메커니즘은 첫 번째 드래프트 토큰과 보너스 토큰에서만 로짓을 추출해 GPU 메모리 덤프 병목을 방지. 훈련 없는 플러그-앤-플레이 솔루션으로 vLLM에 통합 구현됐다.
- **연구 목표와의 관련성**: high
- **주목할 점**: 바닐라 자기회귀 디코딩 대비 **2.81× 처리량** 향상, 최신 희소성 기반 자기-투기적 디코딩 방법 대비 **1.29× 향상**. vLLM 플러그인으로 구현되어 Activity C의 6단계 vLLM 이식 경로가 이미 존재한다는 점이 실용화 관점에서 중요하다. 기존 Activity C의 토큰 퇴거 방법들(LaProx, LKV, SP-KV)이 프리필 단계에서 KV를 선별하는 반면, SpecAttn은 "디코딩 중 검증이 생성하는 완전 어텐션 정보를 실시간으로 희소화에 활용"하는 디코딩-시간 온라인 KV 선별이라는 새로운 시간축 차원을 추가한다. AB-Sparse(이미 보고, 2605.12110)의 헤드별 적응 블록 크기와 결합 시, "SpecAttn이 중요 KV를 검증-가이드로 선별 → AB-Sparse의 헤드별 최적 블록 크기로 해당 KV에 희소 어텐션 적용"하는 정확도-처리량 최적화 2단계 파이프라인이 가능하다. Fluxion(이미 보고, 2605.07719)의 CPU-GPU 하이브리드 스파스 어텐션이 CPU 상주 KV 전체 중 스파스 선택을 다루는 반면, SpecAttn은 GPU 내 KV에서 검증 로짓으로 실시간 선별을 수행하는 상보적 적용 도메인이다.

#### 2. [Low-Rank Key Value Attention (LRKV)](https://arxiv.org/abs/2601.11471)
- **출처**: arXiv 2601.11471 (2026-01-16, James O'Neill, Robert Clancy, Mariia Matskevichus, Fergal Reid — fin.ai 및 관련 소속)
- **핵심 아이디어**: 기존 KV 캐시 공유 메커니즘(MQA, GQA, MLA)이 헤드 간 완전 공유 또는 완전 독립이라는 이진적 선택에 한정되거나(MQA/GQA), 잠재 공간 압축으로 완전 표현력을 희생하는(MLA) 구조적 한계를 해결한다. LRKV는 **레이어당 하나의 공유 풀-랭크 KV 투영**에 **헤드별 저랭크 잔차(head-specific low-rank residuals)**를 더하는 구조를 사전훈련 중 공동 학습한다. 세 가지 설계 특성: (1) **연속적 공유-독립 트레이드오프** — 저랭크 잔차의 랭크 r을 0(완전 공유, MQA와 동등)부터 최대(완전 독립, MHA와 동등)까지 연속적으로 조절 가능, (2) **풀-랭크 공유 기반 보존** — 공유 기반이 풀-랭크이므로 MQA/GQA와 달리 표현 용량이 손실 없이 유지됨, (3) **기존 방법 포함 관계** — LRKV는 MQA와 GQA를 특수 케이스로 포함하고 MLA와는 직교적 방향(잠재 압축 vs 헤드-잔차)으로 다른 설계.
- **연구 목표와의 관련성**: high
- **주목할 점**: 128M~6.3B 파라미터 모델에서 사전훈련 후 **MHA·MQA/GQA·MLA 전 구성 대비 최저 테스트 손실** 달성. MHA KV 캐시 대비 **45~53% KV 메모리 절감**, 동일 품질 도달 속도 **18~25% 가속(훈련 스텝 기준)**. ARC, MMLU, GSM8K, HumanEval 다운스트림 태스크에서 최고 성능. LRKV는 활성화 레벨(inference-time) 압축이 아닌 **훈련 시 구조적(architectural) KV 절감**이라는 점에서 eOptShrinkQ(이미 보고)·MELT(이미 보고) 계열과 구별된다. Activity C의 "KV 캐시 메모리 −30% 이상" 목표를 정확도 보존 하에 사전훈련 레벨에서 45~53% 절감으로 달성하는 가장 완전한 구조적 접근이다. 현재 보고된 MLA(DeepSeek 계열에서 널리 채택)의 대안으로 LRKV가 비교 가능한 효율을 더 간단한 구조로 달성한다는 점이 주목된다. Activity C의 vLLM 이식(6단계)에서 훈련-시간 방법(LRKV)과 추론-시간 방법(LaProx, SP-KV, FibQuant)을 "어떤 방법이 동일 메모리 예산에서 가장 높은 정확도를 보존하는가" 기준으로 비교하는 실험이 필요하다.

**C (SpecAttn + LRKV)**
- SpecAttn(훈련-프리 추론-시간 KV 희소화)과 LRKV(훈련-시간 아키텍처 수준 KV 절감)는 직교적 방향에서 KV 메모리를 절감한다. LRKV로 사전훈련된 모델에 SpecAttn을 투기적 디코딩 플러그인으로 추가하면, 구조적 45~53% KV 절감 위에 디코딩 단계 추가 희소화가 결합되어 총 KV 접근량이 두 방법의 곱셈으로 절감된다. Activity C의 "KV 캐시 메모리 −30% 이상" 목표를 구조(LRKV) + 연산(SpecAttn) 두 레벨에서 동시 달성하는 가장 강력한 조합이다.

**A + B + C (CONCUR + 비연속 재사용 + SpecAttn + LRKV)**

SpecAttn(C)이 디코딩 중 KV 접근을 희소화해 KV 대역폭을 추가 절약한다. 에이전틱 배치 추론에서 "CONCUR 입장 제어(A) → LRKV 구조적 KV 절감(C) → 비연속 세그먼트 재사용(B) → SpecAttn 희소 KV 접근(C)"의 4단계 통합 파이프라인.
