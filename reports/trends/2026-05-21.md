# 트렌드 리포트 — 2026-05-21

## 전체 요약

이번 주(2026-05-15~2026-05-21)는 NeurIPS 2026 제출 마감 직후 아르카이브 활동이 소강 국면에 접어든 가운데, 두 편의 유의미한 신규 논문이 등장했다. Activity A에서는 P/D 분리 환경의 KV 전송 병목을 정면으로 다루는 서비스 인식 적응형 압축 프레임워크 KVServe가 발표되었고, Activity C에서는 청크드 프리필(chunked-prefill) 어텐션에서 블록-유니온 KV 선택으로 2.72×의 어텐션 속도 향상을 달성한 CompactAttention이 공개되었다. Activity B(비연속 KV 재사용)는 이번 기간 신규 논문이 없었으며, 기존 연구 성과의 구현 안정화 단계로 보인다. 전반적으로 "압축+스케줄링의 공동 최적화" 방향이 강화되는 추세이며, Pareto 효율 탐색과 Bayesian 프로파일링이 새로운 설계 축으로 부상하고 있다.

## Activity A — KV Cache-aware Scheduling / Orchestration

### 새로운 논문·구현체

#### 1. [KVServe: Service-Aware Adaptive KV Cache Compression for Disaggregated LLM Serving](https://arxiv.org/abs/2605.13734)
- **출처**: arXiv 2605.13734 (2026-05-13 제출)
- **핵심 아이디어**: P/D 분리 아키텍처에서 KV 캐시 자체가 네트워크 병목이 된다는 문제를 다룬다. 워크로드 혼합(온라인/오프라인), 인터커넥트 대역폭(InfiniBand/RDMA), SLO 예산을 동시에 고려하여 압축 전략을 동적으로 선택하는 서비스 인식(service-aware) 프레임워크를 제안한다. 핵심 구성 요소는 Bayesian Profiling Engine으로, 모듈형 압축 전략 공간(양자화 × 퇴거 × 근사)을 탐색하고 3차원 Pareto 후보 집합을 생성한다. 오프라인 탐색 오버헤드를 기존 대비 50× 감소시킨다고 보고한다.
- **연구 목표와의 관련성**: high
- **주목할 점**: Activity A(스케줄링/오케스트레이션)와 Activity C(압축)의 교차점에 위치한다. P/D 분리 환경에서 KV 전송 비용을 스케줄러 수준에서 압축으로 흡수하는 접근은 본 연구의 "멀티 노드 KV 라우팅 + 정확도 보존 압축" 조합과 직접적으로 맞닿아 있다. Bayesian 기반 전략 탐색은 아이디어 생성 에이전트에게 참고할 만한 메타-최적화 기법이다.

## Activity C — KV Cache Compression

### 새로운 논문·구현체

#### 1. [CompactAttention: Efficient Chunked-Prefill Attention via Block-Union KV Selection](https://arxiv.org/abs/2605.16839)
- **출처**: arXiv 2605.16839 (2026-05-16 제출)
- **핵심 아이디어**: 청크드 프리필(chunked-prefill) 환경에서 2D 블록-스파스 어텐션 마스크를 KV 선택 신호로 변환하는 Block-Union KV Selection 기법을 제안한다. 직접적인 스파스 커널이 아닌 GQA 인식 per-group KV 블록 테이블로 변환(Q-block union + intra-group union)함으로써 표준 어텐션 커널 재사용이 가능하다. LLaMA-3.1-8B-Instruct 기준 128K 컨텍스트에서 최대 2.72× 어텐션 속도 향상을 달성하면서 RULER 벤치마크에서 밀집 어텐션에 근접한 정확도를 유지한다고 보고한다.
- **연구 목표와의 관련성**: high
- **주목할 점**: 본 연구의 "정확도 보존(accuracy-preserving)" 제약을 명시적으로 만족시키는 희소 어텐션 KV 선택 접근이다. 청크드 프리필과의 통합은 Activity A의 스케줄링 단위(청크)와 자연스럽게 연동된다. Block-Union 추상화는 비연속 세그먼트(Activity B)의 KV를 물리적 블록 테이블로 표현하는 방식과 구조적으로 유사하여 A+B+C 시너지 탐색의 출발점이 될 수 있다.

### 동향 변화 감지

## Activity 간 시너지 포인트

**A + C (KVServe 주도 수렴)**
KVServe는 스케줄러가 네트워크 대역폭과 SLO 예산을 입력으로 받아 압축 전략을 실시간 조정한다. 이는 Activity A(스케줄링 오버헤드 최소화)와 Activity C(압축 정확도 보존)를 단일 최적화 루프로 연결하는 실증 사례다. 본 연구에서 스케줄러가 캐시 히트율 예측에 더해 "압축 가능 예산(compressible budget)"을 관리하는 설계를 탐색할 수 있다.

**B + C (CompactAttention Block-Union)**
CompactAttention의 Block-Union KV Selection은 비연속 KV 세그먼트(Activity B)를 GQA-aware 블록 테이블로 표현하는 방식과 구조적으로 동형이다. 비연속 세그먼트를 block-union으로 집약한 뒤 sparse → dense 변환을 거쳐 표준 커널로 처리하는 파이프라인은 B+C 공동 설계의 직접적 청사진이 된다.

**A + B + C (3-Way)**
KVServe(A+C) + CompactAttention(B+C)의 조합은 "비연속 세그먼트를 block-union으로 선택 → 압축 전략 공간을 Bayesian 탐색으로 축소 → 스케줄러가 전송 비용 인식"의 3단 파이프라인으로 확장 가능하다. 특히 P/D 분리 환경에서 prefill 노드가 비연속 KV 세그먼트를 block-union으로 압축하여 decode 노드로 전송하는 시나리오는 세 Activity의 자연스러운 교차점이다.

## idea-generator에게 전달할 포인트

**Activity A**
- KVServe의 Bayesian Profiling Engine 설계를 참조하여, 단순 LRU 퇴거 정책 대신 워크로드 혼합(온라인/오프라인 비율)과 인터커넥트 대역폭을 동적 입력으로 받는 "압축 예산 인식 스케줄러(compression-budget-aware scheduler)"를 설계하는 방향을 검토할 것.
- 3D Pareto 후보 집합(지연 × 정확도 × 대역폭)을 런타임에 유지하고, 요청 큐 상태 변화에 따라 Pareto 프런트를 이동하는 온라인 적응 알고리즘이 새로운 아이디어 방향이다.

**Activity B**
- 이번 주 신규 논문은 없으나, CompactAttention의 Block-Union 추상화가 비연속 세그먼트 재사용의 물리적 표현 문제(비연속 PA 블록을 연속 KV 텐서로 변환하는 비용)를 우회하는 힌트를 제공한다. "비연속 세그먼트 → block-union 인덱스 → GQA-aware 블록 테이블" 변환 파이프라인을 Activity B의 구현 후보로 검토할 것.

**Activity C**
- CompactAttention이 제시한 "스파스 마스크를 직접 사용하지 않고 KV 선택 블록 테이블로 변환하여 표준 커널 재사용"하는 패턴은 구현 복잡도를 낮추면서 정확도를 보존하는 실용적 압축 접근이다. 이 패턴을 Activity C의 "accuracy-preserving" 제약 하 구현에 적용하는 방향을 탐색할 것.
- KVServe의 Bayesian 기반 전략 탐색은 단일 압축 기법에 의존하지 않고 혼합 전략(양자화 + 퇴거 + 근사)을 자동 선택하는 메타-프레임워크로 확장 가능하다.
