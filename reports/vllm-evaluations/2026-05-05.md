## 식별된 이슈

### WARNING 1 — compression_ratio() 이론/실제 불일치
- **위치**: `nqkv_codec_patch.py:211`
- **문제**: `compression_ratio()` 수식 `n_qblocks * block_size * 0.5`는 packed INT4를 가정하지만, 실제 인덱스는 `torch.uint8`(1B/elem)로 저장돼 실제 비율이 1.88×임
- **영향**: `adjusted_num_gpu_blocks()` 호출 시 GPU 블록 풀을 실제보다 ~2× 과대 추정할 수 있음
- **권고**: packed INT4 구현(nibble 패킹)을 추가하거나, `compression_ratio()` 수식을 uint8 실측 기준으로 수정. 플래닝 함수에 "theoretical (packed INT4)" 주석 강화

### WARNING 2 — FireQAttentionPatch 단순화 API 부재
- **위치**: `fireq_attention_patch.py:229–249`
- **문제**: README 예제 및 평가 태스크 스크립트가 가정하는 `FireQAttentionPatch(num_heads, head_dim, num_kv_heads)` 생성자 및 `patch.calibrate(sample)`, `patch.encode_key(key)` 메서드가 없음
- **영향**: 외부 사용자가 문서 예제를 그대로 실행 불가 (TypeError)
- **권고**: `FireQAttentionPatch`에 `num_heads/head_dim/num_kv_heads` 키워드 생성자 오버로드 추가, 또는 `calibrate()`/`encode_key()` 편의 메서드를 `FireQAttentionPatch`에 위임으로 추가

### INFO — CUDA 없는 환경에서 처리량/TTFT 실측 불가
- 현재 테스트 환경에 GPU/CUDA가 없어 실제 처리량 및 TTFT 측정 불가
- 모든 성능 수치는 CPU 기반 또는 이론치임
- vLLM 0.20.1의 libcuda.so 로드 실패는 graceful fallback으로 처리됨 (vLLM 내부 WARNING)
