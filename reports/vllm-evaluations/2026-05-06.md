## 미해결 이슈

아래 항목들은 구현 결함이 아닌 실험 환경 미구성(GPU 없음) 문제다.

1. **Perplexity delta 실측 미완** — `experiments/run_triattention_accuracy.py`가 구현됐으나 GPU + transformers + datasets 환경에서만 실행 가능. 설계 수준 보장(pre-RoPE 기반, 보존 위치 무손실)은 확인됨.
2. **TTFT p50/p99 실측 미완** — GPU 환경 없이 실측 불가. CPU-only 측정에서 스케줄링 오버헤드 14ms (100 requests) — GPU 환경에서는 코사인 유사도 연산이 GPU 가속되므로 수 ms 수준으로 낮아질 것으로 예상.
3. **처리량 회귀 실측 미완** — GPU 없는 환경에서 tokens/sec 측정 불가.
4. **스케줄링 오버헤드 GPU 재측정 필요** — CPU-only에서 14ms로 측정됐으나 GPU + 실제 vLLM 런타임에서 5ms 이내 달성 가능성 높음 (Python CPU 연산, GPU I/O 없음).

## 다음 사이클 제언

1. **GPU 환경에서 Perplexity 자동 검증**: CI 파이프라인에 `run_triattention_accuracy.py`를 통합해 ±1% 기준을 매 사이클 자동 검증. GPT-2 small + WikiText-2로 빠른 smoke-level 검증 가능.
2. **compression_ratio 스윕**: 0.05/0.10/0.20/0.30에서 perplexity 측정 후 최적 비율 선정.
3. **Activity A 통합**: `make_qcrc_aware_scheduler_class(make_dag_aware_scheduler_class(Scheduler))`를 사용해 A+B+C 삼중 조합 스케줄러 완성. 캐시 히트율 추가 +5%p 달성 가능성.
4. **GPU 환경 TTFT 실측**: vLLM 서버 모드에서 prefill 오버헤드를 측정해 스케줄링 TTFT +5% 예산 충족 확인.
5. **Python 종료 segfault 원인 분석**: 3개 모듈 동시 임포트 후 Python finalize segfault는 CUDA/flashinfer teardown 경쟁 조건일 가능성. vLLM 이슈 트래커 확인 권장.
