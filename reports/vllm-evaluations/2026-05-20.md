## 미해결 이슈

1. **KV Memory Reduction 목표 미달 (Partial Pass)**: retention_ratio=0.70 기준 실측 22.5%로 목표 −30% 미달. 이 이슈는 Report ①에서 이미 식별된 알고리즘 설계 수준 문제이며 이식 코드 자체의 버그가 아님. 다음 사이클에서 retention_ratio 0.60 이하 또는 INT4 압축 범위 확대 검토 필요.
2. **Python 인터프리터 종료 시 segfault**: `-W error::DeprecationWarning` 모드에서 Py_FinalizeEx 시점 segfault 발생. 코드 실행 중(runtime)에는 영향 없으며 torch/vLLM C 확장 teardown 순서 문제로 추정. 기능 오류 아님.
3. **GPU 환경 없음**: FlashAttentionImpl GPU 패치 및 실 vLLM 엔진 처리량 측정 불가. CPU-only 환경에서 알고리즘 정확성 및 API 호환성만 검증됨.

## 다음 사이클 제언

- **retention_ratio 하한 조정**: 0.60 이하에서 −30% 메모리 절감 목표 달성 가능. CongestionDual 훅의 baseline을 0.65로 낮추어 CONGESTED 시 0.55로 감소하는 전략 고려.
- **GPU 벤치마크 추가**: 처리량·TTFT 측정이 현재 Report ①의 CPU 시뮬레이션 수치 의존. vLLM 엔진 기반 실측 벤치마크 코드 추가 권장.
- **vLLM 0.21.0 → 0.22.x 업그레이드 대비**: v1 Scheduler API (`vllm.v1.core.sched.scheduler`) 및 KVCacheManager 인터페이스가 안정적이나, 0.22.x 시 `schedule()` 시그니처 변경 여부 주시 필요.
- **Activity B와의 통합**: CONCURCongestionAdmissionSchedulerMixin을 비연속 KV 세그먼트 재사용(Activity B)와 결합하면 캐시 히트율 +15%p 이상 달성 가능성 있음.
